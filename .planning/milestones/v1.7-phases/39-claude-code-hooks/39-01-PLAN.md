Create `.claude/hooks/pre-commit.sh` with these exact behaviors (per D-01, D-02, D-04, D-05, D-06):

1. **Emergency bypass (D-06):** At the top, check `if [ "${SKIP_HOOKS:-0}" = "1" ]; then exit 0; fi`

2. **Consume stdin:** Read stdin with `INPUT=$(cat)` to prevent pipe buffering hangs (Research pitfall 4).

3. **Set PROJECT_DIR:** `PROJECT_DIR="${CLAUDE_PROJECT_DIR:-.}"`

4. **Detect staged files by stack:** Use `git diff --cached --name-only --diff-filter=ACM` to get staged files. Store TS files (`*.ts`, `*.tsx`) in `STAGED_TS` and Rust files (`*.rs`) in `STAGED_RS`. This enables stack-scoping -- skip cargo commands entirely if no Rust files are staged.

5. **Format staged TS files (D-02, D-04, D-05):** If `STAGED_TS` is non-empty, run `echo "$STAGED_TS" | xargs npx biome format --write` then `echo "$STAGED_TS" | xargs git add` to re-stage (Research pitfall 3).

6. **Format staged Rust files (D-05):** If `STAGED_RS` is non-empty, run `echo "$STAGED_RS" | xargs rustfmt` then `echo "$STAGED_RS" | xargs git add` to re-stage.

7. **Lint TS (D-01):** If `STAGED_TS` is non-empty, run `npx biome check src/`. On failure, append `"TypeScript lint failed (biome check)\n"` to an ERRORS variable.

8. **Lint Rust (D-01):** If `STAGED_RS` is non-empty, run `cargo clippy --manifest-path "$PROJECT_DIR/src-tauri/Cargo.toml" -- -D warnings`. On failure, append `"Rust lint failed (cargo clippy)\n"` to ERRORS. Always use `--manifest-path` (Research pitfall 6).

9. **Test TS (D-01):** If `STAGED_TS` is non-empty, run `npx vitest run`. On failure, append `"TypeScript tests failed (vitest run)\n"` to ERRORS.

10. **Test Rust (D-01):** If `STAGED_RS` is non-empty, run `cargo test --manifest-path "$PROJECT_DIR/src-tauri/Cargo.toml"`. On failure, append `"Rust tests failed (cargo test)\n"` to ERRORS.

11. **Gate:** If ERRORS is non-empty, print to stderr: `"Pre-commit blocked. Fix these issues:\n$ERRORS"` and `exit 2`. Otherwise `exit 0`.

Script must use `#!/usr/bin/env bash` and `set -uo pipefail` (not `set -e` since we check individual command exit codes).

Make the script executable: `chmod +x .claude/hooks/pre-commit.sh`.

Create `.claude/settings.json` with this exact structure:

```json
            "type": "command",
            "if": "Bash(git commit*)",

            "timeout": 300,
            "statusMessage": "Running pre-commit checks (lint, test, format)..."
          }
        ]
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {

            "timeout": 300,
            "statusMessage": "Running related tests..."
          }
        ]
      }
    ]
  }
}
```

- PreToolUse `if` uses `Bash(git commit*)` with wildcard to match all commit variations (Research pitfall 1)
- Both hooks use `timeout: 300` for HOOK-04 compliance (cargo builds can take 2-5 min on cold cache)
- PostToolUse matcher is `Edit|Write` to catch both edit and write tool operations
- Commands use `"$CLAUDE_PROJECT_DIR"` variable for portability

Create `.claude/hooks/test-on-save.sh` with these exact behaviors (per D-03, HOOK-02):

1. **Shebang and options:** `#!/usr/bin/env bash` and `set -uo pipefail`

2. **Emergency bypass (D-06):** `if [ "${SKIP_HOOKS:-0}" = "1" ]; then exit 0; fi`

3. **Read stdin JSON:** `INPUT=$(cat)` -- Claude Code pipes tool context as JSON. Must consume to avoid pipe buffering hang (Research pitfall 4).

4. **Extract file path:** Parse `file_path` from the JSON input using bash builtins:
   ```bash
   FILE_PATH=$(echo "$INPUT" | grep -o '"file_path":"[^"]*"' | head -1 | cut -d'"' -f4)
   ```
   If `FILE_PATH` is empty (e.g., tool didn't have a file_path field), `exit 0` silently.

5. **Route by file extension:** Use a `case` statement on `$FILE_PATH`:
   - `*.ts|*.tsx)` -- Run `npx vitest run --reporter=dot 2>&1` to keep output concise. Print results to stderr so Claude Code sees them.
   - `*.rs)` -- Run `cargo test --manifest-path "${CLAUDE_PROJECT_DIR:-.}/src-tauri/Cargo.toml" 2>&1`. Print results to stderr.
   - `*)` -- Exit 0 silently for non-code files (CSS, JSON, MD, etc.).

6. **Always exit 0:** PostToolUse hooks MUST exit 0 even when tests fail. Exit 2 in PostToolUse would make Claude think the edit itself failed and retry (Research pitfall 5). The test results are communicated via stderr output which Claude Code displays.

7. **D-03 compliance:** "Blocking" means synchronous execution (the default -- `async` is not set in settings.json). Claude Code waits for this hook to complete before proceeding. This is different from exit-code blocking.

Make the script executable: `chmod +x .claude/hooks/test-on-save.sh`.
