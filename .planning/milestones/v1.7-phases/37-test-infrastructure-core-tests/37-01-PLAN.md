  <action>
1. Install `@vitest/coverage-v8` as a dev dependency:
   ```bash
   npm install -D @vitest/coverage-v8@^4.1.0
   ```

2. Create `vitest.config.ts` at project root with this exact content:
   ```typescript
   import { defineConfig, mergeConfig } from 'vitest/config';
   import viteConfig from './vite.config';

   export default mergeConfig(viteConfig, defineConfig({
     test: {
       coverage: {
         provider: 'v8',
         reporter: ['text', 'text-summary', 'json-summary'],
         reportsDirectory: './coverage',
         include: ['src/lib/**/*.ts'],
         exclude: ['src/lib/**/*.test.ts', 'src/lib/**/*.test.tsx'],
       },
     },
   }));
   ```

   Key decisions (per D-03):
   - Uses `mergeConfig` to inherit environment, globals, setupFiles, and path aliases from vite.config.ts
   - Coverage scoped to `src/lib/` only (per D-01 -- utility functions only, no components)
   - No coverage thresholds (per D-02 -- reporting only, Phase 39 will enforce)
   - Three reporters: `text` for terminal output, `text-summary` for quick overview, `json-summary` for Phase 40 MCP server consumption
   - `reportsDirectory: './coverage'` for consistent output location

3. Add `test:coverage` script to package.json scripts section:
   ```json
   "test:coverage": "vitest run --coverage"
   ```
   Place it after the existing `"test:watch": "vitest"` line.

4. Run `npx vitest run --coverage` to verify coverage output is generated. The existing src/lib/ tests (date-utils, actionRegistry, shellAllowlist) should produce coverage numbers. Some component tests may fail -- that is expected and acceptable per D-01 (component tests are out of scope).

5. Add `coverage/` to `.gitignore` if not already present (coverage reports should not be committed).
