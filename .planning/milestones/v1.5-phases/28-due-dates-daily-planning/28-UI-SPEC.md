### Urgency Color System (D-06)

Three-tier urgency treatment applied consistently across goals tree and task views:

| Tier | Condition | Badge variant | ProgressDot color | Text treatment |
|------|-----------|---------------|-------------------|----------------|
| Overdue | Past due date | `destructive` (existing) | `bg-destructive-foreground` oklch(0.637 0.237 25.331) | Foreground uses `--color-destructive-foreground` |
| Due soon | Within 2 days | `warning` (new variant) | `bg-amber` oklch(0.75 0.15 85) | Foreground uses oklch(0.25 0.05 85) for contrast |
| Normal | 3+ days or no due date | `outline` (existing) | Unchanged (existing status colors) | Standard foreground |

Accent reserved for: selected date in calendar popover, "Confirm" action buttons on due date suggestions, primary CTA buttons.

## Interaction Contracts

### D-01/D-03: Date Picker Popover in TaskDetail

- Location: TaskDetail > Scheduling accordion section, above SchedulingBadges
- When no due date is set: render a `ghost` variant Button with text "+ Add due date" and CalendarDays icon (12px label weight 400)
- When due date is set: render the existing SchedulingBadges due date badge (clickable to reopen picker)
- Click opens a Popover anchored below the trigger button
- Popover contains quick-select shortcuts above the Calendar grid

### D-04: Quick-Select Shortcuts

- Rendered as a horizontal row of `outline` variant Buttons above the calendar grid inside the popover
- Labels and date calculations:
  - "Today" -- sets to current date
  - "Tomorrow" -- sets to current date + 1 day
  - "Next week" -- sets to next Monday
  - "+1 month" -- sets to current date + 1 calendar month
- Each button: 12px semibold text, 48px min height (grid-aligned, exceeds WCAG 44px touch target minimum), 8px horizontal gap between buttons
- Clicking a shortcut sets the due date, closes the popover, and persists via `updateTask`
- Below shortcuts: standard shadcn Calendar (react-day-picker) in single-select mode
- Selecting a date on the calendar grid also sets the due date and closes the popover
- "Clear" link (12px, muted-foreground) at bottom-right of popover to remove the due date

### D-05: Overdue Count Badge on Goals Tree

- When a project or phase node has overdue child tasks, display a small count badge to the right of the ProgressDot
- Badge: 12px semibold, destructive-foreground text on destructive background, rounded-full, min-width 16px, height 16px, centered text
- Count bubbles up: phase shows count of its overdue tasks; project shows sum across all its phases
- Backlog phases (sortOrder >= 999) are excluded from count calculation (D-08/D-09)
- Badge hidden when count is 0

### D-05/D-06: ProgressDot Overdue Variant

- Add new status value "overdue" to ProgressDot
- ProgressDot with status="overdue": solid fill with `--color-destructive-foreground` oklch(0.637 0.237 25.331)
- The ProgressDot on a phase/project node shows "overdue" status when any non-backlog child task is overdue, taking priority over other status values
- Due-soon state does NOT change the ProgressDot (only overdue does) to avoid visual noise

### D-07: Warning Badge Variant

- Add `warning` variant to Badge component CVA config:
  - `border-transparent` background `oklch(0.75 0.15 85)` at 20% opacity, text `oklch(0.75 0.15 85)` at full opacity
- SchedulingBadges logic update:
  - If due date is in the past: use `destructive` variant (existing)
  - If due date is within 2 days (today or tomorrow): use `warning` variant (new)
  - Otherwise: use `outline` variant (existing)
- Badge text for warning state: "Due {day}" (e.g., "Due today", "Due tomorrow", "Due Wed")

### D-09: Backlog Exemption

- Tasks belonging to phases with `sortOrder >= 999` never show overdue or due-soon badge variants
- These tasks always render due date badge with `outline` variant regardless of date
- GoalsTreeNode skips overdue count for phases with `sortOrder >= 999`
- No schema changes required -- uses existing Phase.sortOrder field

### D-10: Daily Plan in Briefing

- "Today's Plan" section appears inside BriefingContent after the existing briefing markdown
- Section heading: "Today's Plan" (16px semibold), left-aligned
- Below heading: list of DailyPlanTaskRow components ordered by priority ranking
- Each row shows: task title (14px regular, truncated), duration badge (12px secondary), due date badge (12px, urgency-colored), project name (12px caption muted-foreground)
- Rows separated by 16px vertical gap
- Rows are read-only in the briefing (not clickable to navigate -- that is the calendar view's job)

### D-11: Out of Time Divider

- Horizontal separator with centered text label
- Label text: "Won't fit today" (12px regular, muted-foreground)
- Line: 1px solid `--color-border`
- Tasks below this divider are shown with 60% opacity to visually de-emphasize
- Only rendered when the scheduling algorithm determines some tasks exceed available calendar time

### D-12: Due Date Suggestion Cards

- Rendered inline in BriefingContent when the bot suggests due dates for undated tasks
- Card layout: task title (14px semibold) + suggested date (14px regular) on one line, two action buttons on the right
- "Confirm" button: `default` variant Button, 12px semibold label
- "Skip" button: `ghost` variant Button, 12px regular label, muted-foreground
- Buttons are 28px height (compact), 8px gap between them
- On confirm: updates the task's due date via `updateTask`, card collapses with a brief "Set" confirmation text (12px, muted-foreground)
- On skip: card collapses, no change persisted
- Follows suggest-never-auto-apply pattern (D-12 context decision)

### D-13: Rescheduling via Hub Chat

- No new UI components needed -- uses existing HubChat natural language input
- Bot responses for rescheduling follow the same DailyPlanTaskRow format for the updated plan
- Bot never auto-applies schedule changes (out of scope per REQUIREMENTS.md)

### Scroll Behavior

- DatePickerPopover: no scroll needed (compact layout)
- DailyPlanSection within BriefingPanel: scrolls with the rest of the briefing content (existing CardContent overflow-auto)

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Date picker keyboard navigation | Calendar grid supports arrow key navigation (react-day-picker built-in); Enter selects date; Escape closes popover |
| Quick-select buttons | Each button has explicit text label (no icon-only buttons); 48px height meets WCAG touch target minimum |
| Overdue badges | Badge text includes "Overdue" prefix -- color is not sole indicator |
| Due soon badges | Badge text includes "Due" prefix -- color is not sole indicator |
| ProgressDot overdue | aria-label updated to "overdue" when in overdue state |
| Overdue count badge | aria-label="{N} overdue tasks" on the badge element |
| Out-of-time divider | role="separator" with aria-label="Tasks that won't fit today" |
| Daily plan task rows | Each row is a list item within role="list" container |
| Confirm/Skip buttons | aria-label="Confirm due date {date} for {task title}" and "Skip due date suggestion for {task title}" |
| Minimum contrast | Amber warning text oklch(0.75 0.15 85) on 20% opacity background meets 4.5:1 against dark surface |
