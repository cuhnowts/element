  <action>
    Create 4 leaf-level components. All use Tailwind only (no inline styles except dynamic positioning).

    **NowLine.tsx:**
    Props: `{ pixelOffset: number }`. Renders an absolutely-positioned div at `top: {pixelOffset}px`. Contains:
    - An 8px diameter circle at the left edge (positioned at left: -4px to overlap time gutter), background `oklch(0.585 0.156 272)`, rounded-full.
    - A 2px tall line spanning full width, background `oklch(0.585 0.156 272)`.
    - `aria-hidden="true"` on the container (decorative per UI-SPEC accessibility).
    - `style={{ top: pixelOffset }}` with `position: absolute`, `left: 0`, `right: 0`, `z-10`.

    **CalendarEventBlock.tsx:**
    Props: `{ event: PositionedEvent; gridStartMinutes: number; accountColorIndex?: number; onClick?: () => void }`.
    - Computes `top` via `timeToPixelOffset(event.event.startMinutes, gridStartMinutes)`.
    - Computes `height` via `eventHeight(event.event.startMinutes, event.event.endMinutes)`.
    - Computes `left` and `width` from `event.column` and `event.totalColumns`: `left = (column / totalColumns) * 100%`, `width = (1 / totalColumns) * 100%`.
    - For meetings (type === "meeting"): 3px left border at full account color (use CALENDAR_COLORS[accountColorIndex ?? 0]), fill at 40% opacity. Cursor `default`.
    - For work/buffer blocks (type === "work" | "buffer"): 1px solid border at WORK_BLOCK_COLOR, fill at 15% opacity. Cursor `pointer`.
    - Content: title (14px, truncated with text-ellipsis overflow-hidden whitespace-nowrap) + time range formatted as "9:00 - 10:00 AM" (12px, text-muted-foreground). Use `format(date, "h:mm")` from date-fns, or manual formatting from minutes.
    - Click handler: if work/buffer with taskId, call `useWorkspaceStore.getState().selectTask(taskId)` then `useStore.getState().setActiveView('task')`. If meeting, do nothing here (popover handled in Plan 03).
    - `aria-label="{title}, {startTime} to {endTime}"` per UI-SPEC accessibility.
    - `role="button"` and `tabIndex={0}` for work blocks, `tabIndex={0}` for meetings.
    - Min height clamped to EVENT_MIN_HEIGHT (20px).
    - Hover: meetings increase opacity from 40% to 55%. Work blocks increase from 15% to 25%. Use Tailwind `hover:` or inline style transition.

    **AllDayBanner.tsx:**
    Props: `{ events: MergedEvent[] }`. Renders above the time grid. Each all-day event is a horizontal bar, 28px tall (ALLDAY_BANNER_HEIGHT). Shows title truncated with ellipsis. Uses account color for meetings, WORK_BLOCK_COLOR for work blocks. Stacks vertically if multiple. If no all-day events, render nothing (null). Background uses `--color-card`.

    **OverflowIndicator.tsx:**
    Props: `{ count: number; direction: "earlier" | "later"; onClick: () => void }`. Shows "{count} earlier" or "{count} later" as a clickable text label. 12px font, muted-foreground color. Centered horizontally. Uses a ChevronUp icon for "earlier" and ChevronDown for "later" (from lucide-react). On click, calls the onClick handler (which will expand the grid range).
  </action>

    **CalendarHeader.tsx:**
    Props: none (reads from Zustand store directly).
    - Reads `hubSelectedDate`, `hubViewMode`, `setHubSelectedDate`, `setHubViewMode` from `useStore`.
    - Layout: flex row, items-center, justify-between, h-[40px] (HEADER_BAR_HEIGHT), px-4, bg-card border-b.
    - Left section: date display.
      - Day view format: "Wed, Apr 3" -- use `format(new Date(hubSelectedDate + "T00:00:00"), "EEE, MMM d")` from date-fns.
      - Week view format: "Mar 31 - Apr 4" -- compute startOfWeek (Monday, { weekStartsOn: 1 }) and endOfWeek, format as "MMM d - MMM d".
      - Text: 16px semibold (text-base font-semibold).
    - Center section: date navigation arrows + Today button.
      - `<` button (ChevronLeft icon): aria-label="Previous day" (day) or "Previous week" (week). Calls setHubSelectedDate with previous day/week using date-fns `addDays(date, -1)` or `addWeeks(date, -1)`. Format result as "YYYY-MM-DD" ISO string.
      - "Today" button (text "Today"): calls `setHubSelectedDate(new Date().toISOString().split("T")[0])`. Disabled when hubSelectedDate === today's ISO string. Use shadcn Button variant="ghost" size="sm".
      - `>` button (ChevronRight icon): aria-label="Next day" or "Next week".
    - Right section: segmented control (Day | Week).
      - Two buttons side by side in a flex container with rounded border.
      - Active segment: bg-secondary text-foreground.
      - Inactive segment: bg-transparent text-muted-foreground.
      - Role="tablist" on container, role="tab" aria-selected on each button per UI-SPEC accessibility.
      - Clicking "Day" calls `setHubViewMode("day")`. Clicking "Week" calls `setHubViewMode("week")`.
      - Switching preserves hubSelectedDate (D-06 contract).

    **CalendarDayGrid.tsx:**
    Props: `{ dateStr: string }`.
    - Reads workHours from `useStore((s) => s.workHours)`. If null, call `loadWorkHours()` in a useEffect and show Skeleton.
    - Computes gridStartMinutes and gridEndMinutes from workHours.startTime/endTime using normalizeToMinutes.
    - Calls `useCalendarEvents(dateStr)` for event data.
    - Calls `useNowLine(gridStartMinutes)` for now-line.
    - Calls `assignOverlapColumns(events)` to get positioned events.
    - Fetches calendar events for the date: `useEffect(() => { fetchCalendarEvents(dateStr + "T00:00:00", dateStr + "T23:59:59") }, [dateStr])`.
    - Listens for "calendar-synced" Tauri event to refetch (same pattern as MiniCalendar).
    - Grid layout:
      - Outer container: relative, uses shadcn ScrollArea for vertical scrolling. Store a ref to the scroll viewport.
      - Time gutter: absolute left, 56px wide (TIME_GUTTER_WIDTH), shows hour labels ("9 AM", "10 AM", etc.) at each hour boundary. Label style: 12px regular, text-muted-foreground, right-aligned within gutter.
      - Grid lines: for each hour from gridStart to gridEnd, a 1px solid border-border line. For each half-hour, a 1px dashed line at 50% opacity.
      - Event area: left margin of TIME_GUTTER_WIDTH, relative positioned. Contains CalendarEventBlock for each positioned event. Each block absolutely positioned.
      - NowLine: rendered inside event area at nowPixelOffset (only if today's date matches dateStr AND isVisible).
      - Grid total height: ((gridEndMinutes - gridStartMinutes) / MINUTES_PER_SLOT) * SLOT_HEIGHT.
    - Overflow handling (per D-01):
      - Count events that start before gridStartMinutes. If > 0, render OverflowIndicator with direction="earlier" above the grid.
      - Count events that end after gridEndMinutes. If > 0, render OverflowIndicator with direction="later" below the grid.
      - onClick for overflow: temporarily expand grid range to 0:00 or 23:59 (use useState for expanded range).
    - Auto-scroll on load: useEffect that scrolls now-line to ~1/3 from top of viewport. Use `scrollRef.current?.scrollTo({ top: nowPixelOffset - viewportHeight / 3 })`. Only on initial render for today's date.
    - Loading state: if isLoading, show Skeleton shimmer (3 blocks at varied heights per UI-SPEC copywriting contract).
    - Empty state: if no events and not loading, show "No events today" heading with body text per UI-SPEC copy.
    - Error state: read calendarError from store. If set, show "Couldn't load your calendar" with "Try Again" button per UI-SPEC copy.
