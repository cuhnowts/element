## Context File Content Contract

This is the core deliverable of Phase 13 -- the structure and copy of the generated `.element/context.md` file. This file is the "UI" between Element and the AI tool.

### Three-Section Structure (D-11)

```
# {Project Name}

{Description -- verbatim or summarized if long}

**Tier:** {Quick | Medium | GSD}
**State:** {No plan | Planned | In progress | Complete}
**Progress:** {X/Y tasks complete across Z phases}

## Instructions

{Tier+state-specific instructions -- see Instruction Matrix below}

## What Needs Attention

{Up to 5 items -- preserved from Phase 11}

## Current Work

{Phase/task data formatted per token budget rules}

## Output Contract

{Only rendered for Quick/Medium tiers -- D-10, D-13}
```

### Instruction Matrix (D-07, D-08)

Each cell contains the exact copy for the Instructions section.

#### NO_PLAN State

| Tier | Instructions Copy |
|------|-------------------|
| Quick | "Describe what you need done. I'll create a simple task list." |
| Medium | "Let's break this into phases. What are the major deliverables?" |
| GSD | "This project uses GSD workflow. Run /gsd:new-project to begin." |

#### PLANNED State (has tasks, none started)

| Tier | Instructions Copy |
|------|-------------------|
| Quick | "You have a task list below. Pick one and start working on it." |
| Medium | "Review the phases below. Start with the first incomplete phase. Ask if you need clarification." |
| GSD | "This project uses GSD workflow. Run /gsd:progress to see status. Run /gsd:next to continue." |

#### IN_PROGRESS State

| Tier | Instructions Copy |
|------|-------------------|
| Quick | "You have a simple task list. Check off items as you complete them." |
| Medium | "Review the phases below. Focus on the current phase. Ask if you need clarification." |
| GSD | "This project uses GSD workflow. Run /gsd:progress to see status. Run /gsd:next to continue." |

#### COMPLETE State (D-03)

| Tier | Instructions Copy |
|------|-------------------|
| All tiers | "All tasks are complete. Consider marking this project as done, or add follow-up work if needed." |

### State Detection Rules (D-01, D-02)

| State | Condition |
|-------|-----------|
| `NO_PLAN` | No tasks AND no phases exist |
| `PLANNED` | Has tasks, none have status `in_progress` or `complete` |
| `IN_PROGRESS` | At least one task has status `in_progress` |
| `COMPLETE` | All tasks have status `complete` |

### Token Budget Rules (D-04, D-05, D-06)

| Rule | Behavior |
|------|----------|
| Completed phases | Collapse to one-line summary: "Phase N: {name} [{done}/{total} complete]" |
| Active phase | Show individual task titles with status icons (checkbox format) |
| Future phases | Show phase name + task count only: "Phase N: {name} [{total} tasks]" |
| Soft budget | Target ~2000 tokens. If rollup exceeds target, show only the in-progress phase |
| Task detail | Titles only -- no descriptions (D-06) |

### Status Icons (existing, reuse)

| Status | Icon |
|--------|------|
| Todo | `[ ]` |
| In progress | `[~]` |
| Complete | `[x]` |

### Default Tier (D-09)

When no tier is set on the project: **Quick**. This ensures graceful behavior before Phase 14 adds the tier selection UI.

### Output Contract Section (D-10, D-13)

Rendered ONLY for Quick and Medium tiers. GSD tier manages its own planning via `.planning/`.

```
## Output Contract

When the user confirms the plan, write a JSON file to `.element/plan-output.json` with this exact schema:

{existing JSON schema -- preserved from current implementation}
```
