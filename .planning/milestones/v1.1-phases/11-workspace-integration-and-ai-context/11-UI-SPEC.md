Container: `max-w-2xl mx-auto` (matches existing ProjectDetail). AI cards section at top with `space-y-3` between cards. Gap of 24px (`mb-6`) between AI cards section and project name.

### ContextSummaryCard Layout

- Container: `rounded-lg bg-card p-4` with `border-l-2` using `--color-chart-2` (matches existing AiSuggestionPanel border pattern)
- Row 1: Icon (Bot, 16px, `text-muted-foreground`) + "Where was I?" text (16px semibold) + flex spacer + Dismiss button (X icon, ghost variant, `size-icon-xs`)
- Row 2: Unordered bullet list, 14px regular, `text-foreground`, `space-y-1`, `pl-6` (indented under header)
- Max bullets: 5 (enforced by AI prompt, not UI truncation)
- Appear animation: `animate-in fade-in slide-in-from-top-2 duration-300`
- Dismiss animation: `animate-out fade-out slide-out-to-top-2 duration-200`

### AiSuggestionCard Layout

- Container: `rounded-lg bg-card p-4` with `border-l-2` using `--color-chart-2`
- Row 1: Icon (Lightbulb, 16px, `text-chart-4`) + suggestion text (14px regular, flex-1) + action buttons
- Action buttons: ghost variant, 12px, specific to suggestion type:
  - Task-focused: "Go to task" (navigates to task in sidebar)
  - Scaffold-focused: "Scaffold with AI" (triggers existing AI assist flow)
- Dismiss button: X icon, ghost variant, `size-icon-xs`, positioned top-right
- Appear animation: staggered `animate-in fade-in slide-in-from-top-2` with 100ms delay per card

### FallbackSummaryCard Layout

- Same visual structure as ContextSummaryCard
- Icon: BarChart3 (16px, `text-muted-foreground`) instead of Bot
- Header: "Project Status" instead of "Where was I?"
- Content: simple template-generated lines:
  - "[N] tasks completed, [M] remaining"
  - "Current phase: [phase name] ([X]% complete)"
  - "Last activity: [relative time]"
- Footer: 12px `text-muted-foreground` italic: "Configure an AI provider in Settings for smarter summaries."
- No dismiss button (static informational card)

### Loading State

- While AI generates: show `ContextSummaryCard` container with 3 Skeleton lines (matches existing AiSuggestionPanel loading pattern)
- Skeleton widths: `w-3/4`, `w-1/2`, `w-2/3`
- No loading state for suggestion cards (they appear after summary completes)

### On-demand Mode (D-11)

- No summary card rendered
- No suggestion cards rendered
- Project detail view shows phases and progress only (clean view)

## Interaction Contract

### Project Switch with AI Summary (D-04, D-07, AIAS-02)

1. User selects a different project in the sidebar
2. System checks: is this the first open in session, OR has 30+ minutes elapsed since last view of this project?
3. If YES and AI mode is Track+Suggest or Track+Auto-execute:
   a. System checks for configured AI provider
   b. If provider configured: show loading skeleton, call `generate_context_summary` Tauri command, render ContextSummaryCard on response
   c. If no provider: render FallbackSummaryCard immediately (no loading state)
4. If NO (recently viewed): no summary card, show project detail directly
5. Update last-viewed timestamp for this project in Zustand in-memory state

### Summary Dismissal (D-07)

1. User clicks X button on ContextSummaryCard
2. Card animates out (fade-out + slide-up, 200ms)
3. Card removed from DOM after animation completes
4. Dismissed state stored per-project in Zustand (in-memory). Does not reappear until next qualifying project switch.

### Suggestion Card Actions (D-09, AIAS-03)

1. **"Go to task"**: navigates sidebar to select the referenced task. Task detail opens in the right panel (existing flow).
2. **"Scaffold with AI"**: triggers `requestAiAssist(taskId)` from existing aiSlice. Opens task detail with AiSuggestionPanel (existing flow).
3. **Dismiss (X)**: removes individual suggestion card with fade-out animation. Dismissed suggestions stored in Zustand per-project set. Do not reappear until next project switch.

### Suggestion Generation (D-10)

1. Suggestions generated alongside the context summary in a single AI call (or sequential pair)
2. Summary card appears first, suggestion cards appear with staggered animation after summary renders
3. Maximum 3 suggestion cards displayed at once. AI prompt enforces this limit.
4. No mid-session regeneration. No polling. Suggestions are snapshot-at-switch-time.

### Per-Project Tab Memory (D-02)

1. When user switches center panel tab (Detail vs Files), store the active tab keyed by project ID in Zustand in-memory map
2. When switching to a project, restore the last-active tab for that project
3. Default tab for first-ever visit: Detail
4. Tab memory resets on app restart (in-memory only per D-14)

### Per-Project Drawer State (D-03)

1. When user toggles drawer or switches drawer tab, store state keyed by project ID in Zustand in-memory map
2. When switching to a project, restore drawer open/closed and active tab
3. Default drawer state for first-ever visit: use global default from useWorkspaceStore
4. Drawer state resets on app restart (in-memory only per D-14)

## State Management Contract

### Zustand Store Extensions

| Store | New State | Actions |
|-------|-----------|---------|
| useWorkspaceStore (in-memory map) | `projectStates: Record<string, ProjectWorkspaceState>` | `setProjectTab(projectId, tab)`, `getProjectTab(projectId)`, `setProjectDrawer(projectId, state)`, `getProjectDrawer(projectId)` |
| aiSlice (extend) | `contextSummary: ContextSummary \| null`, `contextSuggestions: AiSuggestion[]`, `dismissedSuggestionIds: Set<string>`, `isGeneratingSummary: boolean`, `summaryError: string \| null` | `generateContextSummary(projectId)`, `dismissSummary(projectId)`, `dismissSuggestion(projectId, suggestionId)` |
| In-memory (not persisted) | `lastViewedTimestamps: Record<string, number>` | `recordProjectView(projectId)`, `shouldShowSummary(projectId): boolean` |

### ProjectWorkspaceState Type

```typescript
interface ProjectWorkspaceState {
  centerTab: "detail" | "files";
  drawerOpen: boolean;
  drawerTab: "logs" | "history" | "terminal";
}
```

### ContextSummary Type

```typescript
interface ContextSummary {
  projectId: string;
  bullets: string[];       // 3-5 items
  generatedAt: number;     // timestamp
  dismissed: boolean;
}
```

### AiSuggestion Type

```typescript
interface AiSuggestion {
  id: string;
  projectId: string;
  type: "go-to-task" | "scaffold";
  taskId: string;
  text: string;
}
```

### Optimistic Updates

- Summary dismissal: immediate removal from UI, no backend call needed (in-memory state)
- Suggestion dismissal: immediate removal from UI, add to dismissed set
- Tab/drawer memory: immediate state update on user action, no backend call

## Accessibility Notes

- Summary card: `role="status"` with `aria-live="polite"` so screen readers announce it on project switch
- Dismiss buttons: `aria-label="Dismiss summary"` / `aria-label="Dismiss suggestion"` (icon-only buttons)
- Suggestion action buttons: visible text serves as label, no additional aria needed
- Loading skeleton: `aria-busy="true"` on card container, `aria-label="Loading project summary"`
- Fallback card: static content, no special ARIA needed beyond semantic HTML
- Lightbulb icon: `aria-hidden="true"` (decorative)
- Bot icon: `aria-hidden="true"` (decorative)
