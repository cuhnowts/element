## Interaction Contract

### Directory Picker (D-03, PROJ-01)

1. User clicks "Link Directory" button (ghost variant, FolderOpen icon + text)
2. System opens Tauri native `dialog.open({ directory: true })` — OS-level picker
3. On selection: path stored, button changes to path display + "Change" button
4. Path display: truncated with `text-ellipsis overflow-hidden`, max-width 300px, 12px `text-muted-foreground`
5. "Change" button: ghost variant, 12px

### Phase Creation (D-04, PROJ-02)

1. User clicks "+ Add phase" at bottom of phase list
2. New phase row appears with empty auto-focused Input for name
3. User types name, presses Enter or blurs — phase saved
4. If name is empty on blur, phase row removed (cancelled)
5. New phase gets sort_order = max existing + 1

### Phase Reorder (D-05)

1. User grabs drag handle (GripVertical icon) on phase row
2. Drag visual: phase row lifts with subtle shadow, 1px `border-border` outline
3. Drop: phase snaps to new position, sort_order updated for all affected phases
4. Library: use `@dnd-kit/core` + `@dnd-kit/sortable` (already established pattern in similar apps, or evaluate lightweight alternative)

### Phase Context Menu (D-06)

1. Right-click on phase row opens ContextMenu
2. Items: "Rename" and "Delete"
3. "Rename": replaces phase name with auto-focused Input (same as creation flow)
4. "Delete": opens confirmation Dialog

### Task-to-Phase Assignment (D-07, PROJ-03)

1. In TaskDetail view: new "Phase" field with Select dropdown
2. Options: all phases for the current project + "Unassigned" option
3. Selecting a phase moves the task; UI updates immediately (optimistic)

### In-Phase Task Creation (D-08)

1. Inside expanded phase, user clicks "+ Add task"
2. Inline Input appears for task title, auto-focused
3. On Enter: task created with `phase_id` set to parent phase
4. On empty blur: cancelled

### Progress Updates (D-10, D-11, PROJ-04, PROJ-05)

1. Completing/uncompleting a task recalculates phase and project progress
2. Progress bars animate width change with `transition-all duration-300` (matches existing ProgressBar)
3. Fraction text updates immediately (no animation)

## State Management Contract

### Zustand Store Extensions

| Slice | New State | Actions |
|-------|-----------|---------|
| Project | `phases: Phase[]` | `loadPhases(projectId)`, `createPhase(projectId, name)`, `updatePhase(id, updates)`, `deletePhase(id)`, `reorderPhases(projectId, orderedIds)` |

| Task | `phase_id: string \| null` on Task type | (existing `updateTask` extended) |

### Optimistic Updates

- Phase reorder: update local sort_order immediately, revert on API failure
- Task status toggle: update progress counts immediately, revert on failure
- Phase creation: append to list immediately with temp ID, replace on API response

## Accessibility Notes

- Progress bars: `role="progressbar"` with `aria-valuenow`, `aria-valuemin`, `aria-valuemax`, `aria-label` (matches existing ProgressBar pattern)
- Phase collapse/expand: Collapsible from shadcn provides `aria-expanded` automatically

- Context menu: ContextMenu from shadcn/radix provides keyboard navigation and focus management
- Directory path: truncated text has `title` attribute showing full path on hover
- Phase fraction in sidebar: `aria-label="3 of 12 tasks complete"` on the fraction text
