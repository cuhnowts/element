  <action>
    1. **Add terminal focus guard** at the top of the `handleKeyDown` function, BEFORE any other shortcut checks:

       ```typescript
       // When terminal has focus, only allow Ctrl+` through to global handler.
       // All other keystrokes must flow to xterm.js unintercepted (per D-02, pitfall 4).
       const terminalFocused = document.activeElement?.closest(".xterm") !== null;
       if (terminalFocused && !(e.ctrlKey && e.key === "`")) {
         return; // Let xterm.js handle the keystroke
       }
       ```

       The `.xterm` class is set by xterm.js on its container element. Checking `.closest(".xterm")` detects if the focused element (xterm's hidden textarea) is inside an xterm instance.

    2. **Add Ctrl+` shortcut** after the existing `Cmd+B` handler block:

       ```typescript
       // Ctrl+`: toggle terminal (per D-02)
       if (e.ctrlKey && e.key === "`") {
         e.preventDefault();
         const currentTab = useWorkspaceStore.getState().activeDrawerTab;
         const isDrawerOpen = useWorkspaceStore.getState().drawerOpen;
         if (isDrawerOpen && currentTab === "terminal") {
           // Drawer is open on terminal tab -> close drawer
           toggleDrawer();
         } else {
           // Drawer is closed or on different tab -> open to terminal
           openTerminal();
         }
         return;
       }
       ```

       Note: Use `useWorkspaceStore.getState()` for reading current state inside the event handler (avoids stale closure issues). This is a standard Zustand pattern for reading state outside of React render.

    3. **Add `openTerminal` to the hook's dependencies:**
       - Import `openTerminal` from `useWorkspaceStore`: `const openTerminal = useWorkspaceStore((s) => s.openTerminal);`
       - Add `openTerminal` to the useEffect dependency array

    4. **Important:** The `Ctrl+`backtick shortcut uses `e.ctrlKey` specifically (NOT `meta`). This is intentional -- Ctrl+` is the standard terminal toggle shortcut (VS Code convention). On macOS, this means Ctrl key, not Cmd key. The existing `meta` variable (`e.metaKey || e.ctrlKey`) should NOT be used for this shortcut.
  </action>
