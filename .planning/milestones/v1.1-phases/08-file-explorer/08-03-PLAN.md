  <how-to-verify>
**Prerequisites:** You need a project with a linked directory. If Phase 7 is not yet complete, the `directoryPath` field may need to be manually set on a project in the database, or the plan should have added it to the Project type.

**Start the app:** `npm run tauri dev`

**1. Tab Bar (FILE-01, D-01, D-02)**
- Select a project that has a linked directory
- Verify: "Detail" and "Files" tabs appear above the content
- Click "Files" tab -- file tree appears
- Click "Detail" tab -- project detail view appears
- Select a project WITHOUT a directory -- verify NO tab bar shows

**2. File Tree Browsing (FILE-01, D-03, D-04, D-09)**
- In Files tab, verify root node shows directory name with "/" suffix (e.g. "element/")
- Click root node -- expands to show children with brief loading indicator on first expand
- Verify folders appear before files, alphabetically sorted
- Click a subfolder -- expands lazily
- Click expanded folder again -- collapses

**3. Gitignore Filtering (FILE-03, D-10)**
- In a project with a .gitignore file, verify:
  - `node_modules/`, `.git/`, `target/`, `__pycache__/`, `.DS_Store` are NOT visible
  - Files matching .gitignore patterns are NOT visible
  - Only clean, relevant files are shown

**4. Show-Hidden Toggle (D-11, D-12)**
- Click the eye icon in the Files tab header
- Verify: previously-hidden files appear with dimmed/greyed styling (lower opacity)
- Click again -- hidden files disappear
- Verify toggle persists when switching between Detail and Files tabs

**5. File Interactions (FILE-02, D-06, D-07)**
- Single-click a file -- highlight only (no info panel)
- Double-click a file -- opens in default editor (e.g. VS Code)
- Right-click a file -- context menu: "Open in Editor", "Copy Path", "Reveal in Finder"
- Click "Copy Path" -- verify path is in clipboard (paste somewhere)
- Click "Reveal in Finder" -- Finder opens at file location

**6. Folder Context Menu (D-08)**
- Right-click a folder -- context menu: "Copy Path", "Reveal in Finder"
- Verify both work

**7. Live Updates (FILE-04, D-14, D-15, D-16, D-17)**
- While Files tab is open, create a new file in the project directory via terminal: `touch test-live-update.txt`
- Verify: file appears in tree automatically (within ~1 second after debounce)
- Delete the file: `rm test-live-update.txt`
- Verify: file disappears from tree automatically
- Switch to Detail tab, create another file, switch back to Files -- verify new file is there (D-17)

**8. Expand State Persistence (D-05)**
- Expand several folders
- Switch to Detail tab and back to Files -- expanded folders stay expanded
- Close and reopen the app (if feasible) -- expanded folders should still be expanded
