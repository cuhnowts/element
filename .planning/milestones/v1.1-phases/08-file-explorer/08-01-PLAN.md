**Step 2: Create `src-tauri/src/commands/file_explorer_commands.rs`**

This file contains all 5 Tauri commands plus the `FileWatcherState` managed state struct and the `FileEntry` response type.

**Struct: `FileEntry`** (serde Serialize, Clone):
- `name: String` -- file/folder name
- `path: String` -- absolute path
- `is_dir: bool` -- directory flag
- `is_hidden: bool` -- true if gitignored or in hardcoded excludes (frontend uses for dimming)

**Struct: `FileWatcherState`** (managed Tauri state):
- `watcher: std::sync::Mutex<Option<notify_debouncer_mini::Debouncer<notify::RecommendedWatcher>>>`

**Command 1: `list_directory(dir_path: String, show_hidden: bool) -> Result<Vec<FileEntry>, String>`**
- Per D-04, D-10, D-12, D-13: Lazy-loads one directory level with .gitignore filtering.
- Validate `dir_path` is a directory, return error if not.
- Hardcoded excludes: `["node_modules", ".git", "target", "__pycache__", ".DS_Store"]`
- When `show_hidden == false`: Use `ignore::WalkBuilder::new(&dir_path).max_depth(Some(1)).hidden(false).git_ignore(true).git_global(true).git_exclude(true).build()`. Skip root entry (`entry.path() == path`). Skip entries whose name is in hardcoded_excludes. Push remaining as `FileEntry` with `is_hidden: false`.
- When `show_hidden == true`: Use `std::fs::read_dir` to get ALL entries. For each entry, check if name is in hardcoded_excludes list. Also check gitignore status by building an `ignore::gitignore::Gitignore` from the dir_path (use `ignore::gitignore::GitignoreBuilder` to load `.gitignore` from dir_path and parents). Set `is_hidden = true` if the entry matches gitignore or hardcoded excludes. Include all entries.
- CRITICAL: Wrap synchronous `ignore` crate calls in `tokio::task::spawn_blocking` to avoid blocking the async runtime (Pitfall 5 from RESEARCH.md).
- Sort result: folders first (`b.is_dir.cmp(&a.is_dir)`), then case-insensitive alphabetical (`a.name.to_lowercase().cmp(&b.name.to_lowercase())`).

**Command 2: `open_file_in_editor(file_path: String) -> Result<(), String>`**
- Per D-06, D-07: Opens file in OS default editor.
- Call `open::that_detached(&file_path).map_err(|e| format!("Failed to open file: {}", e))`

**Command 3: `reveal_in_file_manager(path: String) -> Result<(), String>`**
- Per D-07, D-08: Opens Finder/Explorer showing the file.
- macOS (`#[cfg(target_os = "macos")]`): `std::process::Command::new("open").arg("-R").arg(&path).spawn()`
- Windows (`#[cfg(target_os = "windows")]`): `std::process::Command::new("explorer").arg(format!("/select,{}", path)).spawn()`
- Linux (`#[cfg(target_os = "linux")]`): get parent directory, `open::that_detached(&parent)`

**Command 4: `start_file_watcher(app: AppHandle, state: tauri::State<'_, FileWatcherState>, dir_path: String) -> Result<(), String>`**
- Per D-14, D-15, D-17: Starts recursive file watcher with debouncing.
- Stop existing watcher first: `*guard = None`
- Create debouncer with `notify_debouncer_mini::new_debouncer(Duration::from_millis(500), callback)`
- Callback: collect unique parent directories from events into a `HashSet<String>`, then `app_clone.emit("file-system-changed", changed_dirs_vec)`
- Call `debouncer.watcher().watch(Path::new(&dir_path), notify::RecursiveMode::Recursive)`
- Store debouncer in state

**Command 5: `stop_file_watcher(state: tauri::State<'_, FileWatcherState>) -> Result<(), String>`**
- Lock mutex, set to None (dropping the debouncer stops watching).
