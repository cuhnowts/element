## Interaction Contract

### Center Panel Tab System

When a project with a linked directory is selected, the center panel shows a tab bar:

1. **ProjectTabBar** renders above content area with two tabs: "Detail" and "Files"
2. **"Detail" tab**: renders ProjectDetail (existing, from Phase 7)
3. **"Files" tab**: renders FileExplorer
4. Default active tab: "Detail" (user can switch freely)
5. When project has no linked directory: no tab bar, existing behavior unchanged

### Tab Bar Layout

- Horizontal row at top of center panel content area
- Two text tabs: "Detail" | "Files"
- Active tab: `text-primary` with 2px bottom border using `border-primary`
- Inactive tab: `text-muted-foreground` with no bottom border
- Tab row has `border-b border-border` separating it from content below
- Tabs use 14px/400 body text, padding `px-4 py-2` (16px horizontal, 8px vertical)

### Files Tab Header Row

- Fixed row at top of FileExplorer, does not scroll with tree
- Left side: root directory name as label (e.g. "element/") in 12px/600 `text-muted-foreground`
- Right side: show-hidden toggle button (ghost, icon-only, size `h-7 w-7`)
  - Hidden (default): `EyeOff` icon (lucide, size-4, `text-muted-foreground`)
  - Showing: `Eye` icon (lucide, size-4, `text-foreground`)
- Header has `px-4 py-2` (16px horizontal, 8px vertical) padding

### File Tree Layout

- Tree renders inside ScrollArea below the header
- Each tree row is a single line: indent + icon + name
- Row height: 28px (`h-7`), consistent clickable area
- Row padding: `px-2` (8px) base, plus `pl-{level * 8px}` additional indent per nesting level
- Nesting indent: 8px per level (e.g. level 0 = 0px extra, level 1 = 8px, level 2 = 16px)
- Root node (D-03): project directory name, rendered as the first expandable folder node at level 0

### Tree Node Rendering

| Element | Icon | Size | Color |
|---------|------|------|-------|
| Folder (collapsed) | `ChevronRight` + `Folder` | size-4 each | `text-muted-foreground` (chevron), `text-foreground` (folder) |
| Folder (expanded) | `ChevronDown` + `FolderOpen` | size-4 each | `text-muted-foreground` (chevron), `text-foreground` (folder) |
| File | `File` | size-4 | `text-muted-foreground` |

- Gap between chevron and folder icon: 0px (chevron is the expand trigger, flush with folder icon)
- Gap between icon and name: 4px (`gap-1`)
- File rows have no chevron -- 16px left offset to align name with folder names (indent accounts for missing chevron)

Source: RESEARCH.md discretion recommendation (generic icons from lucide-react, no file-type-specific icons).

### Sort Order

Folders first, then files. Alphabetical (case-insensitive) within each group. Backend handles sorting.

Source: RESEARCH.md discretion recommendation.

### Tree Node Interactions

| Trigger | Target | Action |
|---------|--------|--------|
| Single-click | Folder row or chevron | Toggle expand/collapse (D-09). If not yet loaded, fetch children with loading indicator (D-04). |
| Single-click | File row | Select file -- visual highlight only (D-06) |
| Double-click | File row | Open file in default external editor via `open_file_in_editor` command (D-06) |
| Right-click | File row | Context menu: "Open in Editor", "Copy Path", "Reveal in Finder" (D-07) |
| Right-click | Folder row | Context menu: "Copy Path", "Reveal in Finder" (D-08) |

### Context Menu Items

**File context menu:**

| Item | Icon | Action |
|------|------|--------|
| Open in Editor | `ExternalLink` size-4 | Invoke `open_file_in_editor` |
| Copy Path | `Copy` size-4 | Copy absolute file path to clipboard |
| Reveal in Finder | `FolderOpen` size-4 | Invoke `reveal_in_file_manager` |

**Folder context menu:**

| Item | Icon | Action |
|------|------|--------|
| Copy Path | `Copy` size-4 | Copy absolute folder path to clipboard |
| Reveal in Finder | `FolderOpen` size-4 | Invoke `reveal_in_file_manager` |

Context menu uses 14px/400 body text. Icons are `text-muted-foreground`, labels are `text-foreground`.

### Show-Hidden Toggle Behavior

1. Default: OFF -- gitignored and hardcoded-excluded files are hidden entirely (D-10)
2. User clicks toggle: ON -- hidden files appear in the tree with `opacity-40` styling (D-12)
3. Toggle change triggers re-fetch of all currently expanded directories with the new `show_hidden` parameter
4. Toggle state persists per project in Zustand store

### Loading State (D-04)

- When a folder is expanded for the first time, a brief loading indicator shows
- Indicator: single row below the folder with a `Skeleton` component (shadcn, already installed), width 60%, height 16px, at the child indent level
- Loading indicator replaced by actual children when data arrives
- Subsequent collapses/re-expands of the same folder use cached data (no loading indicator)

### Live Update Behavior (D-14 through D-17)

- File watcher runs continuously while project is open
- `file-system-changed` events from backend carry list of changed directory paths
- Frontend refreshes only expanded directories that overlap with changed paths
- No highlight or animation for new/changed files (D-16) -- tree re-renders with updated sort order
- Expand state preserved across live updates (expand state stored separately from tree data)

## States Inventory

### Tree Row States

| State | Visual Treatment |
|-------|-----------------|
| Default | Transparent background, normal text |
| Hover | `hover:bg-muted` row background |
| Selected | `bg-muted text-primary` row background and text |
| Context menu open | `bg-muted` background persists while menu is open |
| Hidden file (show-hidden ON) | `opacity-40` on entire row |
| Loading children | Skeleton placeholder at child indent |

### Folder States

| State | Visual Treatment |
|-------|-----------------|
| Collapsed | `ChevronRight` + `Folder` icon |
| Expanded (loaded) | `ChevronDown` + `FolderOpen` icon, children rendered below |
| Expanded (loading) | `ChevronDown` + `FolderOpen` icon, skeleton placeholder below |
| Empty (no visible children) | Expanded state, no children rendered (folder simply has nothing in it) |

### Files Tab States

| State | Visual Treatment |
|-------|-----------------|
| Normal | Tree rendered with root node |
| No linked directory | Files tab does not appear (tab bar hidden entirely per D-02) |
| Directory deleted/moved | Error state -- see Copywriting Contract |
| File watcher active | No visual indicator (silent background, D-14) |

### Tab Bar States

| State | Visual Treatment |
|-------|-----------------|
| Detail tab active | "Detail" has `text-primary border-b-2 border-primary`, "Files" has `text-muted-foreground` |
| Files tab active | "Files" has `text-primary border-b-2 border-primary`, "Detail" has `text-muted-foreground` |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Tab label: detail | "Detail" |
| Tab label: files | "Files" |
| Root node label | "{directoryName}/" (e.g. "element/") |
| Show-hidden toggle tooltip | "Show hidden files" (when OFF) / "Hide hidden files" (when ON) |
| Context menu: open in editor | "Open in Editor" |
| Context menu: copy path | "Copy Path" |
| Context menu: reveal | "Reveal in Finder" (macOS) / "Reveal in Explorer" (Windows) |
| Empty state heading | "No files found" |
| Empty state body | "This directory appears to be empty. Files will appear here as they are added." |
| Error: directory not found | "Directory not found. The linked directory may have been moved or deleted. Re-link the project directory in project settings." |
| Error: permission denied | "Unable to read directory. Check that Element has permission to access this folder." |
| Error: generic backend | "Couldn't load files -- please try again or restart the app." |
| Loading state (first expand) | No text -- skeleton placeholder only |
| Clipboard success toast | "Path copied" (via Sonner, already installed) |

No destructive actions in this phase -- file explorer is read-only (browse + open externally).
