## Interaction Patterns

### Workflow Builder

- **Insert step:** Click "+" button between any two steps. A new step appears inline with name field focused and executor picker visible.
- **Reorder steps:** Step context menu (three-dot icon) provides "Move Up" and "Move Down" actions. No drag-and-drop for v1.
- **Delete step:** Step context menu "Delete Step" action. If step has configuration, show confirmation dialog.
- **Expand/collapse step:** Click step row to expand inline editor. Only one step expanded at a time. Clicking another collapses the current.
- **Step type selection:** Executor picker dropdown in expanded step. Selecting a type reveals the appropriate editor (ShellEditor, HttpStepForm, or manual description).

### Cron Scheduling

- **Enable recurring:** Toggle "Recurring" switch in task/workflow detail. Reveals CronScheduler inline.
- **Quick presets:** Select from "Hourly", "Daily", "Weekly", "Monthly". Selection populates cron expression and updates CronPreview.
- **Advanced mode:** Toggle "Advanced" to reveal raw cron expression input. Typing updates CronPreview in real time.
- **Pause/resume:** Toggle schedule switch off to pause without deleting. Cron config preserved, greyed out. Toggle back on to resume.
- **CronPreview:** Always shows next 3 run times below the cron input. Updates live as expression changes.

### Execution and Feedback

- **Run Now:** Button in workflow detail header. Disabled during active execution. Click starts execution and switches to execution view.
- **Step progress:** During execution, the execution diagram highlights the current step with `animate-pulse` border. Completed steps show a checkmark icon overlay. Failed steps show an X icon overlay.
- **Failure handling:** On step failure, workflow halts. The failed step shows a "Retry Step" button inline. Clicking retries from that step forward.
- **Run history:** Tab or section in workflow detail showing past runs. Each run row shows: timestamp, duration, step count, pass/fail status. Click to expand per-step breakdown.

### Promote Task to Workflow

- **From task detail:** "Automate" button in task header (Button default variant). Clicking creates a new workflow with the task as step 1 and navigates to the workflow builder.
- **From task list:** Right-click context menu "Convert to Workflow" option. Same behavior as "Automate" button.

### Keyboard Shortcuts

| Shortcut | Action | Context |
|----------|--------|---------|
| `Cmd+Enter` | Run workflow now | Workflow detail focused |
| `Cmd+N` | Add new step at end | Workflow builder focused |
| `Escape` | Collapse expanded step | Step editor open |
| `Cmd+S` | Save workflow changes | Workflow builder focused |
| `Cmd+Backspace` | Delete selected step (with confirmation) | Step selected |

## Layout Integration

Phase 3 UI integrates into the existing Phase 2 multi-panel layout:

| Panel | Phase 3 Addition |
|-------|-----------------|
| Left sidebar — Workflow List | Already exists. Add "+" button to create new workflow. Context menu gains "Convert to Workflow" for task items. |
| Center panel — Task/Workflow Detail | Workflow builder (editable step list) replaces static execution diagram when in edit mode. CronScheduler appears below workflow metadata. "Run Now" and "Automate" buttons in header. |
| Center panel — Execution Diagram | Enhanced with step progress indicators during execution. Retry button on failed steps. Remains read-only visualization. |
| Bottom drawer — Output | Shows live execution output during workflow runs. Per-step output tabs or sections. Run history accessible as separate tab. |

**Primary focal point:** The center panel workflow builder step list. The "Run Now" accent-colored button in the detail header and the step list with its insert-anywhere "+" buttons form the visual anchor. During execution, the focal point shifts to the execution diagram with its pulsing active-step indicator.
