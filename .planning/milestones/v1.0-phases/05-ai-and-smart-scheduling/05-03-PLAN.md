1. **time_blocks.rs** -- Implement open block detection:

   ```rust
   use chrono::NaiveTime;
   use crate::scheduling::types::{CalendarEvent, WorkHoursConfig};

   pub struct OpenBlock {
       pub start: NaiveTime,
       pub end: NaiveTime,
       pub duration_minutes: i32,
   }

   pub fn find_open_blocks(
       work_hours: &WorkHoursConfig,
       events: &[CalendarEvent],
       buffer_minutes: i32,
       min_block_minutes: i32,
   ) -> Vec<OpenBlock>
   ```

   Algorithm:
   1. Parse work_hours.start_time and end_time as NaiveTime (format "HH:MM")
   2. Collect events, parse their start/end times as NaiveTime
   3. Clamp events to work hours window (ignore events fully outside, trim events that partially overlap)
   4. Sort events by start_time
   5. Merge overlapping events into consolidated occupied ranges
   6. Apply buffer_minutes before and after each event (clamp to work hours bounds)
   7. Walk the timeline from work_start to work_end, collect gaps between occupied+buffered ranges
   8. Filter gaps to only include those >= min_block_minutes
   9. Return as OpenBlock vec

2. **assignment.rs** -- Implement task scoring and greedy assignment:

   ```rust
   use chrono::NaiveDate;
   use crate::scheduling::types::{ScheduleBlock, BlockType, TaskWithPriority};
   use crate::scheduling::time_blocks::OpenBlock;
   use crate::models::task::TaskPriority;

   pub fn score_task(task: &TaskWithPriority, today: NaiveDate) -> f64 {
       let priority_weight = match task.priority {
           TaskPriority::Urgent => 100.0,
           TaskPriority::High => 75.0,
           TaskPriority::Medium => 50.0,
           TaskPriority::Low => 25.0,
       };

       let due_date_urgency = task.due_date.map_or(0.0, |due| {
           let days_until = (due - today).num_days() as f64;
           if days_until <= 0.0 { 50.0 }
           else if days_until <= 1.0 { 40.0 }
           else if days_until <= 3.0 { 25.0 }
           else if days_until <= 7.0 { 10.0 }
           else { 0.0 }
       });

       priority_weight + due_date_urgency
   }

   pub fn assign_tasks_to_blocks(
       open_blocks: &[OpenBlock],
       tasks: &[TaskWithPriority],
       schedule_date: &str,
       today: NaiveDate,
   ) -> Vec<ScheduleBlock>
   ```

   Assignment algorithm:
   1. Score all tasks, sort descending by score
   2. For each task (in score order):
      a. Get estimated_minutes (default 30 if None)
      b. Find first open block with remaining capacity >= min(estimated, remaining_task_minutes)
      c. If task fits in one block: create one ScheduleBlock (block_type=Work, is_continuation=false)
      d. If task exceeds block capacity: fill block, mark remaining minutes, continue to next block with is_continuation=true
      e. Generate uuid for each ScheduleBlock.id
   3. Return all created ScheduleBlocks
