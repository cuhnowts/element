## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, sparkle button internal padding |
| sm | 8px | Compact element spacing, AI suggestion field gaps, schedule block padding |
| md | 16px | Default element spacing, card padding, settings section inset |
| lg | 24px | Section padding, AI suggestion panel inset, schedule day header margin |

## Color

Exact values from app.css oklch tokens (dark theme only in current implementation).

| Role | CSS Variable | Usage |
|------|-------------|-------|
| Dominant (60%) | `--background` oklch(0.145 0 0) | Settings page background, dialog backgrounds, schedule panel background |
| Secondary (30%) | `--card` oklch(0.205 0 0) / `--muted` oklch(0.269 0 0) | Provider cards, AI suggestion panel background, schedule block cards |
| Accent (10%) | `--primary` oklch(0.985 0 0) | AI Assist sparkle button, "Add Provider" CTA, "Accept All" button, active settings tab indicator |
| Destructive | `--destructive` oklch(0.396 0.141 25.723) | Remove provider action, dismiss all suggestions action |

Accent reserved for: AI Assist sparkle button (primary affordance), "Add Provider" button, "Accept All" suggestions button, "Apply Schedule" confirmation button, active settings navigation tab. Never applied to static text, borders, or decorative elements.

| Block Type | Color | Visual |
|-----------|-------|--------|

| AI Providers tab content | Custom layout | Lists configured providers. "Add Provider" button top-right |

## Interaction Patterns

### Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| `Cmd+,` | Open Settings page (existing Phase 4 shortcut) |
| `Escape` | Close Settings / close dialog / collapse AI suggestion panel |

No new global shortcuts introduced. AI Assist is intentionally mouse/touch only (on-demand, not a power-user shortcut) to avoid accidental AI invocations.

### States

| State | Visual Treatment |
|-------|-----------------|
| Loading (provider list) | Two skeleton cards: 64px height each, `--muted` background, pulsing |
| Loading (AI generation) | Three skeleton rows in suggestion panel, sparkle button spinning |
| Loading (schedule generation) | Schedule strip shows three skeleton blocks (varying heights), pulsing |
| Loading (connection test) | "Test Connection" button shows spinner, disabled during test |
| Provider card hover | Background transitions to `--muted` |
| Suggested field hover | Background tint intensifies to `--chart-2` at 15% opacity. Accept/dismiss icons become fully opaque |
| Work block hover (in schedule) | Border becomes solid (if dashed), slight scale 1.01 transform. Cursor: grab |
| Work block dragging | Opacity 0.7, elevated shadow. Drop targets highlight with dashed `--primary` border |
| AI Assist button (no provider configured) | Button hidden entirely via conditional rendering |
| AI Assist button (provider configured, idle) | Standard ghost button appearance |
| AI Assist button (generating) | Sparkle icon rotates (CSS animation: rotate 1s linear infinite) |
| Schedule strip (no data) | Empty state component centered in strip area |

### Transitions

| Transition | Duration | Easing |
|-----------|----------|--------|
| AI suggestion panel expand | 200ms | ease-out (height auto) |
| AI suggestion panel collapse | 150ms | ease-in |
| Suggested field accept/dismiss | 150ms | ease-out (opacity fade + height collapse) |
| Schedule block drag | 0ms (follows cursor) | -- |
| Schedule block drop | 200ms | ease-out (snap to position) |
| Connection test result appear | 100ms | fade-in |
| Provider status dot pulse | 1.5s | ease-in-out infinite (for testing state) |
| Sparkle button rotation | 1s | linear infinite (during generation) |
| Dialog open/close | 200ms | ease-out (shadcn default) |
| Hover state (cards/blocks) | 100ms | ease-in |
| Sonner toast | 200ms in, auto-dismiss 3s | shadcn default |

### Drag and Drop (Schedule Blocks)

- Only suggested and confirmed work blocks are draggable. Meeting blocks are read-only.
- Drag handle: entire block surface (cursor: grab).
- Valid drop targets: other work block positions (swap) or empty time slots.
- Invalid drops: meeting blocks, buffer zones, outside work hours. Block snaps back on invalid drop.
- Reordering updates the schedule assignment in-memory. Applied on "Apply Schedule" click.
