1. **anthropic.rs** -- Implement `AnthropicProvider` struct with fields `api_key: String`, `model: String`, `client: reqwest::Client`.
   - `complete`: POST to `https://api.anthropic.com/v1/messages` with headers `x-api-key: {key}`, `anthropic-version: 2023-06-01`, `content-type: application/json`. Body: `{"model": self.model, "max_tokens": req.max_tokens, "system": req.system_prompt, "messages": [{"role": "user", "content": req.user_message}]}`. Parse response `content[0].text` as content, `usage.input_tokens` and `usage.output_tokens`.
   - `complete_stream`: Same endpoint with `"stream": true` in body. Read SSE lines (`data: {...}`), parse each chunk for `content_block_delta` events, send `delta.text` through `event_sender`. Accumulate final response.
   - `test_connection`: Send minimal completion request (max_tokens=1, user_message="hi"). Return `Ok(true)` on success, map errors.
   - `list_models`: Return hardcoded list: `["claude-sonnet-4-20250514", "claude-haiku-35-20241022"]` as ModelInfo items. Anthropic has no list-models API.

2. **openai.rs** -- Implement `OpenAiProvider` struct with `api_key: String`, `model: String`, `client: reqwest::Client`.
   - `complete`: POST to `https://api.openai.com/v1/chat/completions` with header `Authorization: Bearer {key}`. Body: `{"model": self.model, "max_tokens": req.max_tokens, "messages": [{"role": "system", "content": req.system_prompt}, {"role": "user", "content": req.user_message}]}`. Parse `choices[0].message.content`, `usage.prompt_tokens`, `usage.completion_tokens`.
   - `complete_stream`: Same endpoint with `"stream": true`. SSE format, parse `choices[0].delta.content` from each chunk.
   - `test_connection`: Minimal completion.
   - `list_models`: GET `https://api.openai.com/v1/models`, parse `data[].id`, filter to chat models (ids containing "gpt" or "o1" or "o3").

3. **ollama.rs** -- Implement `OllamaProvider` struct with `base_url: String` (default `http://localhost:11434`), `model: String`, `client: reqwest::Client`.
   - `complete`: POST to `{base_url}/api/chat` with body `{"model": self.model, "messages": [{"role": "system", "content": req.system_prompt}, {"role": "user", "content": req.user_message}], "stream": false}`. Parse `message.content`.
   - `complete_stream`: Same endpoint with `"stream": true`. Ollama streams NDJSON (one JSON object per line), parse `message.content` from each line.
   - `test_connection`: GET `{base_url}/api/tags` with 2-second timeout. Return `Ok(true)` if reachable. Important: 2-second timeout to avoid hanging when Ollama is not running.
   - `list_models`: GET `{base_url}/api/tags`, parse `models[].name` as ModelInfo items.

4. **openai_compat.rs** -- Implement `OpenAiCompatProvider` struct with `base_url: String`, `api_key: Option<String>`, `model: String`, `client: reqwest::Client`.
   - Identical to OpenAI implementation but uses `{base_url}/v1/chat/completions` and `{base_url}/v1/models`. API key is optional (some local servers don't need auth). If `api_key` is Some, add `Authorization: Bearer {key}` header; otherwise omit.

5. **gateway.rs** -- Create `AiGateway` struct:
   ```rust
   pub struct AiGateway {
       client: reqwest::Client,
   }
   ```
   Methods (all take `&self` plus a `db: &Database` reference):
   - `create_provider(db, input: CreateProviderInput) -> Result<AiProviderConfig, String>`: Generate UUID for provider. If input.api_key is Some, call `credentials::credential_key_for_provider(&id)` to get a credential_key, then `credentials::store_api_key(&credential_key, &api_key)` to store in OS keychain. INSERT into ai_providers table with credential_key (not the raw api_key). Return config.
   - `list_providers(db) -> Result<Vec<AiProviderConfig>, String>`: SELECT all from ai_providers. NOTE: credential_key is the keychain reference, not the secret. Frontend receives credential_key (it's safe -- it's just a lookup name, not the actual secret).
   - `remove_provider(db, id: &str) -> Result<(), String>`: Get provider config first, if it has a credential_key call `credentials::delete_api_key(&credential_key)` to remove from keychain. Then DELETE from ai_providers.
   - `set_default(db, id: &str) -> Result<(), String>`: UPDATE all is_default=0, then UPDATE target is_default=1.
   - `get_default_provider(db) -> Result<Box<dyn AiProvider>, AiError>`: SELECT where is_default=1, construct the appropriate provider implementation based on provider_type. Retrieve the actual API key from OS keychain via `credentials::get_api_key(&credential_key)`.
   - `build_provider(config: &AiProviderConfig) -> Result<Box<dyn AiProvider>, AiError>`: Factory method that creates the right provider impl from config. For providers needing API keys, retrieves from keychain via credential_key.
   - `test_connection(db, id: &str) -> Result<bool, String>`: Get provider config, build provider (which retrieves API key from keychain), call test_connection.
   - `list_models_for_provider(db, id: &str) -> Result<Vec<ModelInfo>, String>`: Get provider config, build provider, call list_models. For Ollama, also used during add-provider flow to auto-discover models.

   ```rust
   pub fn build_scaffold_request(title: &str, project_name: Option<&str>, description: &str, context: &str) -> CompletionRequest {
       let system = r#"You are a task planning assistant. Given a task title and optional context, generate a structured scaffold. Respond with valid JSON only, no markdown wrapping.

   JSON schema:
   {
     "description": "detailed task description",
     "steps": ["step 1", "step 2", ...],
     "priority": "urgent|high|medium|low",
     "estimated_minutes": 30,
     "tags": ["tag1", "tag2"],
     "related_tasks": ["task name 1", "task name 2"]
   }

   Keep descriptions actionable and concise. Steps should be concrete actions.
   Priority should reflect complexity and urgency implied by the title.
   Estimate duration realistically (most tasks: 30-120 minutes).
   Related tasks are suggestions for tasks that would naturally precede, follow, or complement this one."#;

       let user = format!(
           "Task title: {}\nProject: {}\nExisting description: {}\nExisting context: {}",
           title,
           project_name.unwrap_or("(none)"),
           if description.is_empty() { "(none)" } else { description },
           if context.is_empty() { "(none)" } else { context },
       );

       CompletionRequest {
           system_prompt: system.to_string(),
           user_message: user,
           max_tokens: 1024,
           temperature: 0.3,
       }
   }
   ```
   Also add `parse_scaffold_response(content: &str) -> Result<TaskScaffold, AiError>` that: (1) tries `serde_json::from_str`, (2) tries extracting JSON from markdown code blocks via regex, (3) returns a TaskScaffold with just description set to the raw content as fallback.
