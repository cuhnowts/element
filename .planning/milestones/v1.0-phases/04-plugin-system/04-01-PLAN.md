    **Architecture decision (LOCKED): CredentialManager as Mutex-managed Tauri state**

    CredentialManager wraps `Arc<Mutex<Database>>` + `Box<dyn SecretStore>` and is managed as `Mutex<CredentialManager>` in Tauri state. Credential commands take `State<'_, Mutex<CredentialManager>>` and call methods on the locked manager. The manager accesses `db.conn()` for SQL and `self.secret_store` for keychain -- matching the key_link pattern `credential_manager.lock`.

    In `lib.rs` setup: wrap `Database` in `Arc<Mutex<Database>>`, manage it for existing task/project commands, and also pass the `Arc` clone into `CredentialManager::new(db_arc.clone(), Box::new(KeychainStore))`. Both `Arc<Mutex<Database>>` and `Mutex<CredentialManager>` are registered as separate Tauri managed state.

    **1. Create `src-tauri/src/commands/plugin_commands.rs`:**
    Follow the exact Tauri command pattern from task_commands.rs: async fn, AppHandle, State<'_, Mutex<T>>, return Result<T, String>.

    Commands:
    - `list_plugins(state: State<'_, Mutex<PluginHost>>) -> Result<Vec<PluginInfo>, String>` - Returns serializable plugin list with name, display_name, version, description, status, error_message, capabilities, enabled
    - `get_plugin(state: State<'_, Mutex<PluginHost>>, name: String) -> Result<PluginInfo, String>` - Get single plugin details
    - `enable_plugin(app: AppHandle, state: State<'_, Mutex<PluginHost>>, name: String) -> Result<(), String>` - Enable plugin, emit "plugin-updated"
    - `disable_plugin(app: AppHandle, state: State<'_, Mutex<PluginHost>>, name: String) -> Result<(), String>` - Disable plugin, emit "plugin-updated"
