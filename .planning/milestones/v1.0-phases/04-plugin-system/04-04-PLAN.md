    CalendarPlugin struct with methods:
    - `start_oauth_google(app_handle: &AppHandle) -> Result<(String, String), CalendarError>`:
      1. Generate PKCE: code_verifier (43-128 random chars), code_challenge = BASE64URL(SHA256(code_verifier))
      2. Start tauri-plugin-oauth localhost server to capture redirect
      3. Build Google OAuth URL: `https://accounts.google.com/o/oauth2/v2/auth` with params: client_id (embedded), redirect_uri=http://localhost:{port}/callback, response_type=code, scope=https://www.googleapis.com/auth/calendar.readonly, code_challenge, code_challenge_method=S256, access_type=offline, prompt=consent
      4. Open URL in system browser via tauri shell
      5. Wait for callback with authorization code
      6. Exchange code for tokens: POST `https://oauth2.googleapis.com/token` with grant_type=authorization_code, code, redirect_uri, client_id, code_verifier
      7. Store refresh_token in credential vault (as oauth_token type)
      8. Create CalendarAccount record in SQLite
      9. Return (account_id, email)

    - `start_oauth_outlook(app_handle: &AppHandle) -> Result<(String, String), CalendarError>`:
      Same pattern but Microsoft endpoints:
      - Auth URL: `https://login.microsoftonline.com/common/oauth2/v2.0/authorize`
      - Scope: `https://graph.microsoft.com/Calendars.Read offline_access`
      - Token URL: `https://login.microsoftonline.com/common/oauth2/v2.0/token`
      - User info from Microsoft Graph: `GET https://graph.microsoft.com/v1.0/me`

    - `sync_account(account: &CalendarAccount, db: &Connection, secret_store: &dyn SecretStore) -> Result<SyncResult, CalendarError>`:
      1. Retrieve refresh_token from credential vault
      2. Refresh access_token using refresh_token (POST to token endpoint)
      3. Serialize token refresh through a Mutex to prevent race condition (Pitfall 3 from RESEARCH.md)
      4. If Google: call `sync_google_calendar` with syncToken
      5. If Outlook: call `sync_outlook_calendar` with deltaLink
      6. Parse response events
      7. Upsert events into calendar_events table
      8. Update sync_token and last_synced_at in calendar_accounts
      9. Return SyncResult { events_updated: usize, next_sync_token: String }

    - `sync_google_calendar(client: &reqwest::Client, access_token: &str, calendar_id: &str, sync_token: Option<&str>) -> Result<GoogleSyncResponse, CalendarError>`:
      - URL: `https://www.googleapis.com/calendar/v3/calendars/{calendar_id}/events`
      - Params: singleEvents=true, orderBy=startTime
      - If sync_token: add syncToken param (incremental sync, Pitfall 4 from RESEARCH.md)
      - Else: add timeMin=now, timeMax=now+30days (full sync)
      - Parse response: items array -> Vec<CalendarEvent>, extract nextSyncToken

    - `sync_outlook_calendar(client: &reqwest::Client, access_token: &str, delta_link: Option<&str>) -> Result<OutlookSyncResponse, CalendarError>`:
      - URL: delta_link if available, else `https://graph.microsoft.com/v1.0/me/calendarView?startDateTime={now}&endDateTime={now+30days}`
      - Parse response: value array -> Vec<CalendarEvent>, extract @odata.deltaLink

    - `parse_google_events(json: &serde_json::Value) -> Vec<CalendarEvent>`: Extracts from Google Calendar API format (summary -> title, start.dateTime/start.date, end.dateTime/end.date, location, attendees[].email, htmlLink)

    - `parse_outlook_events(json: &serde_json::Value) -> Vec<CalendarEvent>`: Extracts from Microsoft Graph format (subject -> title, start.dateTime, end.dateTime, location.displayName, attendees[].emailAddress.address, isAllDay)

    DB methods (add to Database struct or as standalone functions taking &Connection):
    - `save_calendar_account(conn, account) -> Result<CalendarAccount>`
    - `list_calendar_accounts(conn) -> Result<Vec<CalendarAccount>>`
    - `delete_calendar_account(conn, account_id) -> Result<()>` (CASCADE deletes events)
    - `save_events(conn, events: &[CalendarEvent]) -> Result<usize>` -- UPSERT using INSERT OR REPLACE
    - `list_events_for_range(conn, start: &str, end: &str) -> Result<Vec<CalendarEvent>>` -- SELECT WHERE start_time >= ? AND end_time <= ?
    - `update_sync_token(conn, account_id, sync_token, last_synced_at) -> Result<()>`

    Background sync (in PluginHost or separate module):
