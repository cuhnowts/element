    ```rust
    #[derive(Debug, Serialize, Deserialize, Clone)]
    pub struct ShellStepInput {
        pub command: String,
        pub working_directory: Option<String>,
        pub timeout_seconds: Option<u64>,  // default 30
    }
    ```

    `ShellPlugin` struct with:
    - `async fn execute(input: ShellStepInput) -> Result<ShellStepOutput, PluginError>`
    - Uses `tokio::process::Command::new("sh").arg("-c").arg(&input.command)`
    - On Windows: use `cmd /C` instead of `sh -c`
    - Sets working directory if provided: `.current_dir(cwd)`
    - Timeout: wrap with `tokio::time::timeout(Duration::from_secs(timeout), child.wait_with_output())`
    - Captures stdout and stderr as UTF-8 strings
    - Returns exit code from process status
    - If timeout: set timed_out=true, kill process, return partial output

    **2. Create `src-tauri/src/plugins/core/http.rs`:**

    Input/output structs:
    ```rust
    #[derive(Debug, Serialize, Deserialize, Clone)]
    pub struct HttpStepInput {
        pub method: String,             // GET, POST, PUT, DELETE, PATCH
        pub url: String,
        pub headers: Option<HashMap<String, String>>,
        pub body: Option<String>,
        pub auth: Option<HttpAuth>,
        pub timeout_seconds: Option<u64>,  // default 30
    }

    #[derive(Debug, Serialize, Deserialize, Clone)]
    #[serde(tag = "type", rename_all = "snake_case")]
    pub enum HttpAuth {
        Bearer { token: String },
        Basic { username: String, password: String },
    }

    #[derive(Debug, Serialize, Deserialize, Clone)]
    pub struct HttpStepOutput {
        pub status: u16,
        pub headers: HashMap<String, String>,
        pub body: String,
        pub elapsed_ms: u64,
    }
    ```

    `HttpPlugin` struct with:
    - `async fn execute(input: HttpStepInput) -> Result<HttpStepOutput, PluginError>`
    - Creates `reqwest::Client::new()` with timeout
    - Builds request from method string: client.get(url) / client.post(url) / etc.
    - Adds headers from input.headers HashMap
    - Adds body for POST/PUT/PATCH methods
    - Applies auth: Bearer token or Basic auth
    - Sends request, captures status, response headers, body text, elapsed time
    - Returns HttpStepOutput

    `FilesystemPlugin` struct with:

    - `fn validate_path(&self, path: &Path) -> Result<(), PluginError>` -- checks path is within an allowed_paths entry (uses canonicalize + starts_with)
    - `async fn execute(input: FsStepInput) -> Result<FsStepOutput, PluginError>`
    - Read: validate path, tokio::fs::read_to_string
    - Write: validate path, tokio::fs::write
    - List: validate path, tokio::fs::read_dir, collect entries

    **4. Create `src-tauri/src/plugins/core/mod.rs`:**
    ```rust
    pub mod shell;
    pub mod http;
    pub mod filesystem;

    pub fn register_core_plugins(registry: &mut PluginRegistry) {
        // Shell plugin manifest
        let shell_manifest = PluginManifest {
            name: "core-shell".to_string(),
            version: env!("CARGO_PKG_VERSION").to_string(),
            display_name: "Shell Command".to_string(),
            description: "Execute shell commands and CLI tools".to_string(),
            author: Some("Element".to_string()),
            capabilities: vec![PluginCapability::Shell],
            credentials: vec![],
            entry: None,  // Core plugin, no entry file
            step_types: vec![StepTypeDefinition {
                id: "shell-command".to_string(),
                name: "Shell Command".to_string(),
                description: "Execute a shell command".to_string(),
                input_schema: serde_json::json!({...}),  // ShellStepInput schema
                output_schema: serde_json::json!({...}),
            }],
        };

        // HTTP plugin manifest -- same pattern, name "core-http", capability Network
        // step_type id "http-request", name "HTTP Request"

        // Filesystem plugin manifest -- same pattern, name "core-filesystem", capabilities FsRead + FsWrite
        // step_type id "file-operation", name "File Operation"

        // Register all three with status Active
        registry.register(LoadedPlugin { manifest: shell_manifest, status: PluginStatus::Active, ... });
        registry.register(LoadedPlugin { manifest: http_manifest, status: PluginStatus::Active, ... });
        registry.register(LoadedPlugin { manifest: fs_manifest, status: PluginStatus::Active, ... });
    }
    ```
