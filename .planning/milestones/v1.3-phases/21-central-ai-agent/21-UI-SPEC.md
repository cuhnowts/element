## Interaction Contracts

### Agent Panel Toggle

| Interaction | Behavior |
|-------------|----------|
| Click toggle button (panel closed) | Agent panel slides in from right edge (200ms ease-out). Panel width: 320px. |
| Click toggle button (panel open) | Agent panel slides out to right (200ms ease-in). Center panel reclaims width. |
| Keyboard shortcut | Cmd+Shift+A toggles panel open/closed. |
| Badge on toggle | Shows pending approval count (1-9 or "9+"). Hidden when 0 pending. Uses `--destructive` background to draw attention. |

### Agent Panel Sub-Tabs

| Interaction | Behavior |
|-------------|----------|
| Click "Activity" tab | Show activity log. Matches existing drawer tab styling (12px/600 uppercase tracking-wide). |
| Click "Terminal" tab | Show raw agent terminal output (xterm.js). |
| Default on open | Activity tab is selected by default. |

### Activity Tab

| Interaction | Behavior |
|-------------|----------|
| Scroll | Vertical scroll via ScrollArea. New entries appear at top (reverse chronological). |
| Approval "Approve" click | Send approval to agent via MCP. Entry transitions to "Approved" state (green check icon, dimmed). Toast confirmation: "Phase approved". |
| Approval "Reject" click | Send rejection to agent via MCP. Entry transitions to "Rejected" state (red x icon, dimmed). Toast confirmation: "Phase rejected". |
| Click project name in entry | Navigate to that project in center panel (`selectProject(id)`). |

### Agent Lifecycle States (D-01, D-04)

| State | Panel Header Visual | Behavior |
|-------|-------------------|----------|
| Starting | Amber dot, "Starting..." label | Agent is launching on app start. No activity yet. |
| Running | Green dot, "Running" label | Agent is active and processing. Activity entries stream in. |
| Idle | Muted dot, "Idle" label | Agent is alive but no work queued. |
| Crashed | Red dot, "Error" label | Agent exited unexpectedly. Auto-restart in progress (D-04). Toast fires. |
| Restart failed | Red dot, "Stopped" label | 3 restart attempts exhausted. Manual "Restart" button appears in header. |

### "Open AI" Integration (D-12)

| State | Behavior |
|-------|----------|
| User clicks "Open AI" on project | Agent receives request via MCP. Agent spawns named terminal session for that project (using Phase 19 multi-terminal infrastructure). Drawer opens to terminal tab. Agent seeds project context into the new session. |
| Agent panel closed when "Open AI" clicked | Session still created. Agent panel does not auto-open. Drawer opens to terminal as usual. |

- Agent panel is a fixed-width right sidebar: 320px (`w-80`).
- Panel is collapsible. When collapsed, center panel takes full remaining width.
- Panel has `border-l border-border` separator from center content.
- Panel spans full height (not affected by output drawer resize).
- Toggle button `[Ag]` sits in the drawer handle bar, right side, after bell icon (from Phase 20).

### Agent Panel Header

- Height: 40px, `px-4 py-2`.
- Left: "Agent" text (14px/600) + status dot (8px circle, 4px gap from text) + status label (12px/400 muted-foreground).
- Right: Sub-tab buttons ("Activity" / "Terminal") using same style as drawer tabs (12px/600 uppercase tracking-wide) + collapse chevron button (ChevronRight icon, ghost, size-sm).

### Activity Entry

- Layout: horizontal icon (left, 16px) + content (flex-1) + timestamp (right).
- Icon: Lucide icon matching entry type -- `Play` for execution start, `CheckCircle2` for completion, `AlertCircle` for approval needed, `XCircle` for failure, `MessageSquare` for context seeded.
- Title: 14px/600, single line, truncate with ellipsis.
- Description: 14px/400, `text-muted-foreground`, max 2 lines, `line-clamp-2`.
- Timestamp: 12px/400, `text-muted-foreground`, relative time ("2m ago", "1h ago").
- Padding: `px-4 py-3` (16px horizontal, 12px vertical).
- Hover: `hover:bg-muted/50` subtle highlight.

### Approval Request

- Same base layout as ActivityEntry but with visual distinction.
- Left border: 2px `border-l-2 border-chart-4` (amber) to indicate pending action.
- Background: `bg-muted/30` subtle tint to stand out from informational entries.
- Action buttons row below description: "Approve" (`variant="default" size="sm"`) and "Reject" (`variant="outline" size="sm"`), right-aligned with 8px gap.
- Approved state: Left border changes to green (`border-chart-2`), buttons replaced with "Approved" label, `opacity-60`.
- Rejected state: Left border changes to `border-destructive`, buttons replaced with "Rejected" label, `opacity-60`.

## State Contracts

### Agent Store Shape (Zustand, session-only, NOT persisted)

```typescript
type AgentStatus = "starting" | "running" | "idle" | "error" | "stopped";

interface AgentActivityEntry {
  id: string;                // crypto.randomUUID()
  type: "execution_start" | "execution_complete" | "planning_complete"
      | "context_seeded" | "human_needed" | "error" | "approval_request";
  title: string;
  description: string;
  projectId?: string;
  projectName?: string;
  phaseId?: string;
  phaseName?: string;
  timestamp: number;         // Date.now()
  // Approval-specific fields
  approvalStatus?: "pending" | "approved" | "rejected";
}

interface AgentState {
  // Panel
  panelOpen: boolean;
  activeTab: "activity" | "terminal";

  // Agent lifecycle
  status: AgentStatus;
  restartCount: number;

  // Activity
  entries: AgentActivityEntry[];

  // Computed
  pendingApprovalCount: () => number;

  // Actions
  togglePanel: () => void;
  setActiveTab: (tab: "activity" | "terminal") => void;
  setStatus: (status: AgentStatus) => void;
  addEntry: (entry: Omit<AgentActivityEntry, "id" | "timestamp">) => void;
  approveEntry: (entryId: string) => void;
  rejectEntry: (entryId: string) => void;
  clearEntries: () => void;
  incrementRestart: () => void;
  resetRestartCount: () => void;
}
```

Notes:
- This store is NOT persisted (agent state is ephemeral per app session).
- Activity entries have no cap in v1 -- if memory becomes a concern, introduce a 200-entry rolling window in a future iteration.
- The agent terminal session is managed through the Phase 19 `TerminalSessionState` store with a reserved `type: "agent"` session type.
