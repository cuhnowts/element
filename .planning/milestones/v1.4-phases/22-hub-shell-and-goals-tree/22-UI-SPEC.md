## Layout Contract

### Hub 3-Column Layout

```
+------------------+---------------------------+------------------+
|   Goals Tree     |      Center Column        | Calendar         |
|   (25% default)  |      (50% default)        | (25% default)    |
|                  |                           |                  |
|  [Project A]     |   Welcome back            |  Calendar        |
|    Phase 1  *    |                           |  (coming soon)   |
|    Phase 2  o    |   [placeholder for        |                  |
|    Phase 3  .    |    briefing - Phase 23]   |                  |
|  [Project B]     |                           |                  |
|    Phase 1  *    |                           |                  |
|                  |                           |                  |
|  --- Chores ---  |                           |                  |
|  To-Do's         |   [placeholder for        |                  |
|  [ ] Task A      |    chat - Phase 24]       |                  |
|  [ ] Task B      |                           |                  |
+------------------+---------------------------+------------------+
```

- Columns use `react-resizable-panels` via shadcn `ResizablePanelGroup` (direction="horizontal")
- Default sizes: `defaultSize={25}` / `defaultSize={50}` / `defaultSize={25}`
- Minimum sizes before collapse: `minSize={15}` for side columns, `minSize={30}` for center
- ResizableHandle between each column (no grip icon, just 1px border-colored divider)

### Minimized Column State

When a column is minimized:
- Panel collapses to `collapsedSize={0}` with `collapsible={true}` on the ResizablePanel
- A 40px-wide overlay strip renders with:
  - Vertical text label rotated 90deg (e.g., "Goals", "Calendar")
  - "+" button (Lucide `Plus` icon, 16px) at the top of the strip
  - Background: `--color-card`
  - Border on the inner edge: `--color-border`
- Clicking "+" calls `panel.expand()` on the react-resizable-panels API
- Minimize trigger: double-click on resize handle, or column reaches minimum size

### Column Persistence

- Column sizes and collapsed states persist in `useWorkspaceStore` (Zustand persist middleware)
- Shape: `hubLayout: { goalsPanelSize: number; centerPanelSize: number; calendarPanelSize: number; goalsCollapsed: boolean; calendarCollapsed: boolean }`
- Center column cannot be collapsed

## Goals Tree Contract

### Tree Structure

```
Goals (column header, Heading 16px/600)
---------------------------------------
[v] Project Alpha                    *   <- filled circle (complete)
      Phase 1: Foundation            *
      Phase 2: Features              o   <- hollow circle (in progress)
      Phase 3: Polish                .   <- empty circle (not started)
[>] Project Beta                     o
                                         
--- Chores ----------------------------- 
To-Do's (Label 12px/400)
[x] Task A                              
[ ] Task B                              
[ ] Task C                              
```

### Tree Node Interaction

| Action | Behavior |
|--------|----------|
| Click project name | Navigate: `activeView = 'project'`, `selectedProjectId = id` |
| Click phase name | Navigate: `activeView = 'project'`, `selectedProjectId = parentId` (scroll to phase) |
| Click chevron (project) | Toggle expand/collapse of phase children |
| Click checkbox (chore) | Toggle task completion via existing task store |
| Hover tree node | Background changes to `--color-secondary` |

### Progress Indicators

Three states rendered as 8px circles:

| State | Visual | CSS |
|-------|--------|-----|
| Complete | Filled circle | `w-2 h-2 rounded-full bg-primary` |
| In Progress | Hollow circle with border | `w-2 h-2 rounded-full border border-primary bg-transparent` |
| Not Started | Dim empty circle | `w-2 h-2 rounded-full border border-muted-foreground/40 bg-transparent` |

Project-level progress: derived from child phases. If all complete, filled. If any in progress, hollow. Otherwise empty.

### Tree Indentation

- Project nodes: 8px left padding (flush to column edge with small inset)
- Phase nodes: 28px left padding (8px base + 20px indent)
- Chevron icon: 16px, positioned inline before the project name
- Progress dot: right-aligned within the tree row, 8px right padding

## Navigation Contract

### Home Button

- Position: Top of ThemeSidebar, above theme list, below any window controls
- Icon: Lucide `Home`, 18px
- Label: "Home" (visible, not tooltip-only)
- Style: Same as sidebar section headers, with hover highlight
- Behavior: Sets `activeView = 'hub'`, clears `selectedProjectId`, `selectedTaskId`, `selectedThemeId`, `selectedWorkflowId`
- Active state: When `activeView === 'hub'`, icon and label use `--color-primary` (accent)

### activeView State Machine

```
hub (default on launch)
  -> project (click project in tree or sidebar)
  -> task (click task anywhere)
  -> theme (click theme in sidebar)
  -> workflow (click workflow)

project | task | theme | workflow
  -> hub (click Home button)
  -> any other view (click corresponding item)
```

- `activeView` defaults to `'hub'` on app launch (D-07)
- No back stack, no history (D-13)
- CenterPanel renders based on `activeView` value via switch statement

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Tree keyboard nav | Arrow keys navigate tree nodes, Enter/Space expands or activates |
| Tree ARIA roles | `role="tree"` on container, `role="treeitem"` on nodes, `aria-expanded` on expandable nodes |
| Home button | `aria-current="page"` when hub is active view |
| Minimized column expand | `aria-label="Expand Goals column"` / `"Expand Calendar column"` on "+" buttons |
| Checkbox | Native checkbox semantics via existing shadcn pattern, `aria-label` with task name |
| Resize handle | `aria-label="Resize column"` (already handled by react-resizable-panels) |
| Focus management | When navigating from hub to detail view, focus moves to detail view heading |
