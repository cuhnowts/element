## Interaction Contract

### Drawer Tab Bar (extended)

The existing DrawerHeader gains a third tab:

```
[ Logs ] [ History ] [ Terminal ]                    [ Clear Logs ] [ Hide Output ]
```

- Tab styling unchanged: 12px/600 uppercase tracking-wide, active = `text-foreground bg-muted`, inactive = `text-muted-foreground hover:text-foreground`
- "Clear Logs" button only shows when Logs tab is active (existing behavior)
- No additional action buttons for Terminal tab in v1

### Terminal Tab States

| State | What Renders | Behavior |
|-------|-------------|----------|
| Project selected, directory linked, tab active | TerminalTab with live xterm.js terminal | PTY spawns lazily on first focus (D-04). Full interactive terminal. |
| Project selected, directory linked, tab inactive | TerminalTab hidden via `display: none` | Terminal instance stays alive, scrollback preserved. PTY continues running. |
| Project selected, no directory linked | TerminalEmptyState | Message + CTA button to link directory. |
| No project selected | TerminalEmptyState (variant) | Message indicating a project must be selected. |
| PTY process exited | TerminalTab with exit message | Shows "Process exited with code {N}" in terminal. User can switch projects to get a new session. |
| Project switched (new project has directory) | TerminalTab reinitializes | Previous PTY killed (D-05). New PTY spawns on next tab focus. |

### Terminal Interaction

| Trigger | Action |
|---------|--------|
| Type in terminal | Keystrokes sent to PTY via `pty.write()` |
| Terminal output from shell | Data written to xterm.js via `term.write()` |
| Ctrl+\` (keyboard shortcut) | Toggle drawer open to Terminal tab. If drawer is open on Terminal tab, close drawer. |
| Click Terminal tab | Switch to Terminal tab, spawn PTY if first focus |
| Resize drawer (drag handle) | ResizeObserver triggers `fitAddon.fit()` then `pty.resize(cols, rows)` |
| Copy (Cmd+C with selection) | xterm.js native selection copy |
| Paste (Cmd+V) | xterm.js native paste to PTY input |
| Scroll up | xterm.js scrollback buffer (5000 lines) |
| Switch to different project | Kill current PTY session. Clear terminal. Next focus spawns new session in new project directory. |
| Close app / unmount | `pty.kill()` called in cleanup. No zombie processes. |

### Keyboard Shortcut Guard

When the terminal has focus (xterm.js textarea is `document.activeElement`), global keyboard shortcuts (Cmd+K, Cmd+B, etc.) must NOT fire. Only Ctrl+\` passes through to the global handler. All other keystrokes go to xterm.js.

### Auto-Open Behavior (D-03)

On first project selection in a session (project has a linked directory):
1. Drawer opens automatically
2. Active tab set to "Terminal"
3. PTY spawns (since terminal tab receives focus)
4. After this first auto-open, subsequent project switches do NOT auto-open the drawer

Implementation: Track `hasAutoOpenedTerminal` boolean in workspace store (session-only, not persisted). Set to `true` after first auto-open.

### Empty State: No Directory Linked (D-07)

Centered vertically and horizontally within the drawer content area:

```
[Terminal icon, size-8, text-muted-foreground]

Link a directory to enable terminal

This project doesn't have a linked directory.
Link one to open a terminal here.

[ Link Directory ]
```

The "Link Directory" button opens the native directory picker (reuses Phase 7 directory linking flow).

### Empty State: No Project Selected

Centered vertically and horizontally within the drawer content area:

```
[Terminal icon, size-8, text-muted-foreground]

Select a project to use the terminal

Choose a project from the sidebar to open
a terminal in its directory.
```

No CTA button -- user action is to click a project in the sidebar.
