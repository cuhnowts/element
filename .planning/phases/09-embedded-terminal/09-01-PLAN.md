    1. **Extend `src/stores/useWorkspaceStore.ts`** -- Add drawer tab tracking and terminal auto-open state. The updated interface should be:

       ```typescript
       type DrawerTab = "logs" | "history" | "runs" | "terminal";

       interface WorkspaceState {
         // existing fields unchanged
         drawerHeight: number;
         drawerOpen: boolean;
         calendarVisible: boolean;
         selectedTaskId: string | null;

         // NEW fields
         activeDrawerTab: DrawerTab;
         hasAutoOpenedTerminal: boolean;  // session-only, per D-03

         // existing actions unchanged
         setDrawerHeight: (height: number) => void;
         toggleDrawer: () => void;
         toggleCalendar: () => void;
         selectTask: (id: string | null) => void;

         // NEW actions
         setActiveDrawerTab: (tab: DrawerTab) => void;
         openTerminal: () => void;         // sets drawerOpen=true, activeDrawerTab="terminal"
         markTerminalAutoOpened: () => void; // sets hasAutoOpenedTerminal=true
       }
       ```

       Implementation details:
       - `activeDrawerTab` defaults to `"logs"`
       - `openTerminal` sets `{ drawerOpen: true, activeDrawerTab: "terminal" }`
       - `markTerminalAutoOpened` sets `{ hasAutoOpenedTerminal: true }`
       - `hasAutoOpenedTerminal` defaults to `false`
       - In the `partialize` function, DO NOT persist `activeDrawerTab` or `hasAutoOpenedTerminal` -- these are session-only state (per D-03, auto-open is per-session)
       - Export the `DrawerTab` type

    2. **Create `src/hooks/useTerminal.ts`** -- A custom hook managing the full PTY lifecycle. The hook signature:

       ```typescript
       export function useTerminal(
         containerRef: React.RefObject<HTMLDivElement | null>,
         cwd: string | null,
         isVisible: boolean
       ): { isReady: boolean; error: string | null }
       ```

       Implementation:
       - Import `Terminal` from `@xterm/xterm`, `FitAddon` from `@xterm/addon-fit`, `WebglAddon` from `@xterm/addon-webgl`, `spawn` from `tauri-pty`
       - Import `@xterm/xterm/css/xterm.css` at the top of the file (CRITICAL -- without this, terminal renders invisibly)
       - Use `useRef` for Terminal instance (`termRef`), FitAddon instance (`fitRef`), PTY handle (`ptyRef`), and ResizeObserver (`observerRef`)
       - Use `useState` for `isReady` (boolean, default false) and `error` (string | null, default null)

       **Terminal creation effect** (depends on `containerRef`, `cwd`):
       - If `containerRef.current` is null or `cwd` is null, return early (do not spawn)
       - Create Terminal with these options:
         ```typescript
         {
           cursorBlink: true,
           fontSize: 13,
           fontFamily: "'SF Mono', 'Fira Code', 'Cascadia Code', Menlo, monospace",
           scrollback: 5000,
           theme: {
             background: "#09090b",
             foreground: "#fafafa",
             cursor: "#fafafa",
             cursorAccent: "#09090b",
             selectionBackground: "#27272a80",
             selectionForeground: "#fafafa",
             black: "#09090b",
             red: "#ef4444",
             green: "#22c55e",
             yellow: "#eab308",
             blue: "#3b82f6",
             magenta: "#a855f7",
             cyan: "#06b6d4",
             white: "#fafafa",
             brightBlack: "#52525b",
             brightRed: "#f87171",
             brightGreen: "#4ade80",
             brightYellow: "#facc15",
             brightBlue: "#60a5fa",
             brightMagenta: "#c084fc",
             brightCyan: "#22d3ee",
             brightWhite: "#ffffff",
           }
         }
         ```
       - Create and load FitAddon
       - Call `term.open(containerRef.current)`
       - Load WebglAddon in try/catch (falls back to canvas/DOM automatically on failure)
       - Call `fitAddon.fit()`
       - Determine shell: use `"/bin/zsh"` as default (macOS). On Windows, would use `powershell.exe` -- but per D-08, read from environment. For now, hardcode `"/bin/zsh"` with a comment noting the Windows case.
       - Spawn PTY: `spawn(shell, ["-l"], { cols: term.cols, rows: term.rows, cwd })` (per D-09, -l flag for login shell)
       - Wire bidirectional data: `pty.onData((data) => term.write(data))` and `term.onData((data) => pty.write(data))`
       - Wire exit handler: `pty.onExit(({ exitCode }) => term.write(\`\r\nProcess exited with code ${exitCode}\r\n\`))`
       - Create ResizeObserver on `containerRef.current`: on resize, call `fitAddon.fit()` then `pty.resize(term.cols, term.rows)`
       - Set `isReady` to true after PTY spawn succeeds
       - Wrap PTY spawn in try/catch: on error, set `error` to the error message (e.g., "Could not start /bin/zsh. Ensure $SHELL is set or restart the app." per UI-SPEC copywriting)
       - Store refs: `termRef.current = term`, `fitRef.current = fitAddon`, `ptyRef.current = pty`, `observerRef.current = resizeObserver`

       **Cleanup function** (returned from the effect):
       - `observerRef.current?.disconnect()`
       - `ptyRef.current?.kill()`
       - `termRef.current?.dispose()`
       - Set all refs to null

       **Visibility effect** (depends on `isVisible`):
       - When `isVisible` becomes true and `fitRef.current` exists, call `fitRef.current.fit()` then if `ptyRef.current` and `termRef.current`, call `ptyRef.current.resize(termRef.current.cols, termRef.current.rows)`
       - This handles the case where the terminal was hidden via CSS `display: none` and needs re-fitting (per pitfall 2 from RESEARCH.md)
