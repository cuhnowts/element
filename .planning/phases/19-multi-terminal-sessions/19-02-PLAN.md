    **TerminalSession.tsx** — Per-session terminal wrapper (replaces direct TerminalTab usage):

    Props: `{ sessionId: string; projectId: string; cwd: string; isVisible: boolean; initialCommand?: { command: string; args: string[] } | null }`

    Implementation:
    - Uses `useRef<HTMLDivElement>(null)` for container
    - Calls `useTerminal(containerRef, cwd, isVisible, initialCommand)` — gets back `{ isReady, error, ptyRef, termRef }`
    - On PTY exit (detected via pty.onExit in useTerminal), calls `useTerminalSessionStore.getState().markExited(projectId, sessionId)`
    - After marking exited, sets a 3-second timeout then calls `useTerminalSessionStore.getState().removeSession(projectId, sessionId)` (per D-07)
    - On unmount (cleanup effect), if ptyRef.current exists, call `gracefulKillPty(ptyRef.current)` for clean PTY shutdown (per D-12)
    - Renders same layout as current TerminalTab: `<div className="h-full w-full flex flex-col"><div ref={containerRef} className="h-full w-full p-2" />{error && ...}</div>`
    - Error display matches TerminalTab: "Terminal failed to start -- check that your shell is configured correctly."

    To detect PTY exit, add a `useEffect` that watches `ptyRef.current` and calls `pty.onExit(callback)`:
    ```typescript
    useEffect(() => {
      const pty = ptyRef.current;
      if (!pty) return;
      pty.onExit(() => {
        useTerminalSessionStore.getState().markExited(projectId, sessionId);
        const timer = setTimeout(() => {
          useTerminalSessionStore.getState().removeSession(projectId, sessionId);
        }, 3000);
        return () => clearTimeout(timer);
      });
    }, [ptyRef.current, projectId, sessionId]);
    ```
