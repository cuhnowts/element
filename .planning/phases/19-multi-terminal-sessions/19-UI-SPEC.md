## Interaction Contracts

### Session Tab Bar

| Interaction | Behavior |
|-------------|----------|
| Click session tab | Switch visible terminal to that session. Apply active style. |
| Click "+" button | Create new shell session for current project. Auto-name "Shell N" (N = next available integer). Focus new session. |
| Click "x" on tab | Kill PTY immediately (SIGTERM then SIGKILL per D-12). Remove tab. If closed tab was active, focus the nearest remaining tab (prefer left neighbor). If no tabs remain, show empty state. |
| Horizontal scroll | Mouse wheel scrolls tab row horizontally. Subtle fade indicators on left/right edges when content overflows (D-08). |
| Tab name truncation | Names longer than 20 characters truncate with ellipsis. Full name shown in tooltip on hover. |

### "Open AI" Button Flow (D-01)

| State | Behavior |
|-------|----------|
| No AI session for project | Create new named session (name derived from phase or "AI Planning"). Open drawer to terminal. Focus new session. |
| AI session already exists | Show RefreshContextDialog. "Refresh Context" kills existing AI session, creates new one with fresh context. "Keep Existing" switches to the existing AI session tab. |
| No linked directory | Toast error (existing behavior, unchanged). |

### Session Lifecycle

| Event | Behavior |
|-------|----------|
| Shell process exits (`exit` command) | Show "Process exited" in muted text. Auto-remove tab after 3 seconds (D-07). Fade-out animation over 300ms. |
| PTY spawn failure | Show error inline in terminal area (existing pattern from TerminalTab). Toast notification. |
| Project switch | Tab bar swaps to new project's sessions. Previous project's PTYs continue running in background (D-09). |
| Project switch back | Tab bar restores previous project's sessions. Scroll history preserved. Previously active tab re-selected. |
| Project removed/unlinked | Kill all sessions for that project immediately (D-11). |
| App quit | Iterate all sessions across all projects. SIGTERM each, wait 3s, SIGKILL survivors (D-13). |

### Sidebar Session Indicator (D-10)

| State | Appearance |
|-------|------------|
| Project has running sessions | Small 6px circle, `#22c55e` at 50% opacity, positioned right of project name with 4px gap. |
| Project has no sessions | No indicator shown. |

## Layout Contract

### Drawer Terminal Area (when Terminal tab is active)

```
+------------------------------------------------------------------+
| [Logs] [History] [Terminal]          [Clear Logs] [Hide Output]  |  <- DrawerHeader (existing)
+------------------------------------------------------------------+
| [Phase 19: Multi-Terminal] [Shell 1] [Dev Server]  [+]          |  <- SessionTabBar (new, 32px height)
+------------------------------------------------------------------+
|                                                                  |
|  xterm.js terminal content for active session                    |
|  (full remaining height, p-2 padding)                            |
|                                                                  |
+------------------------------------------------------------------+
```

- SessionTabBar height: 32px (py-1 + tab content).
- Session tabs: `px-2 py-1`, 12px font, semibold, uppercase tracking-wide. Matches existing drawer tab styling for visual consistency.
- Active session tab: `text-foreground bg-muted` background with `rounded` corners.
- Inactive session tab: `text-muted-foreground hover:text-foreground` transition.
- "+" button: ghost button, 24px square, Lucide `Plus` icon at 14px (`size-3.5`). Right-aligned after last tab, sticky (does not scroll with tabs).
- Close "x" on each tab: Lucide `X` icon at 12px (`size-3`), `text-muted-foreground hover:text-foreground`, 4px left margin from tab name. Only visible on tab hover.

## State Contracts

### Session Store Shape (Zustand, session-only, NOT persisted)

```typescript
interface TerminalSession {
  id: string;              // crypto.randomUUID()
  projectId: string;
  name: string;            // "Phase 19: Multi-Terminal", "Shell 1", "Dev Server"
  type: "ai" | "shell";   // Distinguishes AI sessions from manual shells
  initialCommand?: { command: string; args: string[] } | null;
  status: "running" | "exited";
  createdAt: number;       // Date.now()
}

interface TerminalSessionState {
  sessions: Record<string, TerminalSession[]>;  // keyed by projectId
  activeSessionId: Record<string, string>;       // keyed by projectId -> session.id
  // Actions
  createSession: (projectId: string, name: string, type: "ai" | "shell", initialCommand?: {...}) => string;
  removeSession: (projectId: string, sessionId: string) => void;
  setActiveSession: (projectId: string, sessionId: string) => void;
  markExited: (projectId: string, sessionId: string) => void;
  getAiSession: (projectId: string) => TerminalSession | undefined;
  removeAllForProject: (projectId: string) => void;
  getAllSessions: () => TerminalSession[];
}
```

Notes:
- This store is NOT persisted (session-only, matching existing pattern from D-14 in workspace store).
- The `useWorkspaceStore` fields `terminalSessionKey` and `terminalInitialCommand` will be removed and replaced by this dedicated store.
- `launchTerminalCommand()` will be replaced by `createSession()` with type "ai".
