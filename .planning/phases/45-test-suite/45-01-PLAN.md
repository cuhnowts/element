Helper function to add:
```rust
fn create_skill_plugin(dir: &Path, name: &str, skills_json: &str) {
    let plugin_dir = dir.join(name);
    std::fs::create_dir_all(&plugin_dir).unwrap();
    std::fs::write(
        plugin_dir.join("plugin.json"),
        format!(r#"{{
            "name": "{}",
            "version": "1.0.0",
            "display_name": "Test",
            "description": "Test plugin",
            "manifest_version": 2,
            "skills": {}
        }}"#, name, skills_json),
    ).unwrap();
}
```

<action>
Add the following tests to the `#[cfg(test)] mod tests` block in `src-tauri/src/plugins/mod.rs`:

1. **`test_dispatch_skill_routes_to_registered`**: Create plugin with skill "query". Enable it. Call `host.dispatch_skill("plugin-name:query", serde_json::json!({"q": "test"}))`. Verify result is `Ok` and contains `"status": "dispatched"` and `"skill": "plugin-name:query"`.

2. **`test_dispatch_skill_errors_for_unregistered`**: Call `host.dispatch_skill("nonexistent:skill", serde_json::json!({}))`. Verify result is `Err` and error message contains "failed to execute".

3. **`test_namespace_collision_prevented`**: Create two plugins "plugin-a" and "plugin-b", both with a skill named "ingest". Enable both. Verify `host.list_skills()` contains both "plugin-a:ingest" and "plugin-b:ingest" (2 entries). Dispatch "plugin-a:ingest" succeeds. Dispatch "plugin-b:ingest" succeeds.

4. **`test_enable_with_create_dirs_hook`**: Create plugin with `on_enable: ["create_dirs"]` and `owned_directories: [{"path": ".test-data", "scope": "global", "description": "test"}]`. Call `host.set_enabled("plugin-name", true)`. Verify directory `.test-data` exists inside the app_data_dir.

5. **`test_enable_with_unknown_hook_does_not_error`**: Create plugin with `on_enable: ["unknown_hook"]`. Call `host.set_enabled("plugin-name", true)`. Verify it returns `Ok(())` (unknown hooks are logged but tolerated).
