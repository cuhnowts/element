## Interaction Contract

### "Plan with AI" Flow (D-01, D-02, AIOB-01)

1. User sees "Plan with AI" button in empty project detail view (zero phases)
2. Click opens the scope input form (replaces empty state in center panel)
3. User fills scope (required) and optional goals
4. "Start AI Planning" triggers: (a) Element writes `.element/onboard.md` to project directory, (b) drawer opens and Terminal tab activates (D-11), (c) configured CLI tool launched in terminal, (d) center panel shows waiting state card
5. If project has no linked directory: show Dialog prompting "Link a directory first" with directory picker -- directory is required for skill file placement

### File Watcher Detection (D-09)

1. After CLI launch, Element watches for `.element/plan-output.json` in project directory
2. When file appears: waiting card transitions -- spinner becomes CheckCircle2, heading changes to "Plan ready for review"
3. After 500ms pause: center panel transitions to review screen with parsed plan data
4. If file is malformed JSON: show error toast "Could not parse AI plan output. Check .element/plan-output.json format." Stay on waiting screen.

### Review Editing (D-13, AIOB-04)

1. Click phase name: inline edit with Input, auto-focus, save on Enter/blur
2. Click task name: inline edit with Input, auto-focus, save on Enter/blur
3. Click [x] on phase: remove phase and its tasks from review (no confirmation needed -- not yet saved)
4. Click [x] on task: remove task from review
5. Click "+ Add phase": append new phase with empty name Input, auto-focused
6. Click "+ Add task" within phase: append new task with empty name Input, auto-focused
7. Drag phase via GripVertical handle: reorder within the review list
8. Empty name on blur: item removed (cancelled)

### Confirm & Save (D-14)

1. User clicks "Confirm & Save"
2. Button shows loading state (spinner icon replaces text)
3. All phases and tasks batch-created via backend commands
4. On success: center panel transitions to populated project detail view, toast: "[N] phases and [M] tasks created"
5. On failure: toast with error, review screen preserved so user can retry

### Discard (D-15)

1. User clicks "Discard plan"
2. Confirmation Dialog: "Discard this plan? The AI-generated phases and tasks will not be saved."
3. Confirm ("Discard plan" button): return to empty project detail view, delete `.element/plan-output.json` if present
4. Cancel ("Keep reviewing" button): stay on review screen

### Cancel During Waiting (D-04, D-10)

1. User clicks "Stop AI setup" on waiting card
2. Return to empty project detail view
3. Terminal session stays open (user may want to see output)
4. `.element/plan-output.json` is NOT cleaned up (user can manually inspect)

### AI Mode Selection (D-16, D-17, D-18, AIAS-01)

1. AI mode Select dropdown visible in project detail header
2. Change triggers backend update: `update_project_ai_mode(project_id, mode)`
3. Optimistic update: dropdown reflects new value immediately
4. On failure: revert to previous value, show error toast
5. Default for new projects: "On-demand"

## State Management Contract

### Zustand Store Extensions

| Slice | New State | Actions |
|-------|-----------|---------|
| AI / Onboarding | `onboardingStep: 'idle' \| 'scope-input' \| 'waiting' \| 'review'` | `setOnboardingStep(step)` |
| AI / Onboarding | `pendingPlan: { phases: PendingPhase[] } \| null` | `setPendingPlan(plan)`, `updatePendingPhase(index, updates)`, `removePendingPhase(index)`, `addPendingPhase()`, `reorderPendingPhases(fromIndex, toIndex)`, `updatePendingTask(phaseIndex, taskIndex, updates)`, `removePendingTask(phaseIndex, taskIndex)`, `addPendingTask(phaseIndex)` |
| AI / Onboarding | `onboardingScope: string`, `onboardingGoals: string` | `setOnboardingScope(text)`, `setOnboardingGoals(text)` |
| Project | `ai_mode: 'on-demand' \| 'track-suggest' \| 'track-auto-execute'` on Project type | `updateProjectAiMode(projectId, mode)` |

### Optimistic Updates

- AI mode change: update Select value immediately, revert on API failure
- Review edits: all edits are local state (pendingPlan) until "Confirm & Save" -- no optimistic backend calls needed
- Confirm & Save: show loading state on button, transition on success, preserve state on failure

## Accessibility Notes

- Scope textarea: `aria-required="true"`, associated Label via `htmlFor`

- AI mode Select: `aria-label="AI assistance mode"`
- Discard confirmation Dialog: focus trapped, Escape to dismiss (shadcn Dialog default)
- Drag-and-drop in review: `@dnd-kit` provides `aria-roledescription="sortable"` and keyboard support (consistent with Phase 7)
