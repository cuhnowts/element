Fix the black screen crash in TaskDetail.tsx. The root causes are:

1. **Uncaught async error in useEffect**: Line 67-69 calls `loadTaskDetail(selectedTaskId)` which calls `api.getTask(taskId)`. If this throws (e.g., task was deleted, DB error), the error is uncaught and React crashes with no error boundary. Fix: wrap the `loadTaskDetail` and `fetchExecutionHistory` calls in the useEffect (lines 66-71) in a try/catch. On error, call `selectWorkspaceTask(null)` to deselect the broken task and return to hub view.

2. **Null coalescing for nullable fields**: Several fields on `TaskWithTags` are `string | null` or `number | null` but are passed to child components or used in state initialization. While the existing skeleton guard (lines 84-92) should prevent null renders, add defensive defaults in the useEffect that syncs local state (lines 74-80):
   - `setTitle(selectedTask.title)` — title is non-null, OK
   - `setDescription(selectedTask.description ?? "")` — description is `string` per type but DB might return null
   - `setContext(selectedTask.context ?? "")` — same defensive treatment

3. **Wrap the entire render return in a try pattern**: Add a top-level guard after the skeleton check. If `selectedTask` somehow becomes null between the guard and render (race condition with async state updates), the component should return the skeleton, not crash. Use optional chaining on any deep property access that could fail:
   - `selectedTask.tags?.map(...)` at line 251 (tags is `Tag[]` but guard against undefined)
   - `phases.find(...)?.name ?? "Unknown"` at line 328 (already correct)

Implementation:
```tsx
// In the useEffect at lines 66-71, wrap in try/catch:
useEffect(() => {
  if (selectedTaskId) {
    loadTaskDetail(selectedTaskId).catch(() => {
      selectWorkspaceTask(null);
    });
    fetchExecutionHistory(selectedTaskId).catch(() => {
      // Non-critical, silently ignore
    });
  }
}, [selectedTaskId, loadTaskDetail, fetchExecutionHistory, selectWorkspaceTask]);

// In the state sync useEffect (lines 74-80), add null coalescing:
useEffect(() => {
  if (selectedTask) {
    setTitle(selectedTask.title);
    setDescription(selectedTask.description ?? "");
    setContext(selectedTask.context ?? "");
  }
}, [selectedTask?.id, selectedTask?.context, selectedTask?.description, selectedTask]);

// In the tags render at line 251, add optional chaining:
{selectedTask.tags?.map((tag) => (
// ... existing code
))}
```

Do NOT add an error boundary component — that is overkill for this fix. The try/catch in useEffect is sufficient.

Both CalendarAccounts.tsx and PhaseRow.tsx already have backdrop click handlers and Escape key handlers on their modal overlays. The likely issue is that the backdrop `div` with `onKeyDown` never receives keyboard focus, so the Escape handler never fires. The backdrop div needs `tabIndex={-1}` and an `autoFocus`-like mechanism (or a `useEffect` to focus it) to receive keyboard events.

**CalendarAccounts.tsx** (lines 248-272):
The modal overlay div at line 250 has `onKeyDown` for Escape and `onClick` for backdrop dismiss. These ARE already present (line 251-252). The fix needed:
1. Add `tabIndex={-1}` to the backdrop div so it can receive focus
2. Add a `ref` and use `useEffect` or `autoFocus` to focus the backdrop div when it appears, OR simpler: add `tabIndex={0}` and rely on the click to focus, but this is fragile. Better approach: add a `useEffect` with a `ref` that calls `.focus()` when `disconnectTarget` becomes truthy.

Simplest reliable fix for CalendarAccounts.tsx:
```tsx
// Add useRef import (already imported via useState)
// Add ref for the backdrop:
// Inside the component, before the return:
const backdropRef = useRef<HTMLDivElement>(null);

// Add useEffect to focus backdrop when dialog opens:
useEffect(() => {
  if (disconnectTarget && backdropRef.current) {
    backdropRef.current.focus();
  }
}, [disconnectTarget]);

// On the backdrop div (line 250), add ref and tabIndex:
<div
  ref={backdropRef}
  className="fixed inset-0 z-50 flex items-center justify-center bg-black/50"
  onClick={() => setDisconnectTarget(null)}
  onKeyDown={(e) => { if (e.key === "Escape") setDisconnectTarget(null); }}
  tabIndex={-1}
  role="dialog"
>
```
Note: `useRef` must be imported — check if it already is. CalendarAccounts.tsx imports `useEffect, useState` from react (line 2). Add `useRef` to that import.
