## Focal Point: Tier Selection Dialog

**Trigger:** User clicks "Open AI" on a project with no stored tier and no existing tasks/phases (D-02).

### Layout

```
+------------------------------------------+
|  [X]                                      |
|  How should we plan this?                 |
|  Choose a planning approach for this      |
|  project.                                 |
|                                           |
|  ( ) Quick                                |
|      A simple task list. Best for small   |
|      projects or quick todos.             |
|                                           |
|  (*) Medium                               |
|      AI asks a few questions, then builds |
|      phases and tasks for your review.    |
|                                           |
|  ( ) GSD                                  |
|      Full GSD workflow with phases,       |
|      plans, and .planning/ directory.     |
|                                           |
|  What are you building?                   |
|  +--------------------------------------+ |
|  | Describe your project briefly...     | |
|  |                                      | |
|  +--------------------------------------+ |
|                                           |
| [                        ] [Start Planning] |
+------------------------------------------+
```

**Layout rules:**
- Dialog max-width: `sm:max-w-md` (28rem / 448px) -- wider than default `sm:max-w-sm` to fit descriptions
- 16px (`gap-4`) between each radio option
- 8px (`gap-2`) between radio label and its description line
- 24px (`gap-6`) between radio group and textarea section
- Textarea: 3 rows, full-width, placeholder "Describe your project briefly..."
- DialogFooter: "Start Planning" button right-aligned, default variant (accent color)
- Radio selection uses native `<input type="radio">` styled with Tailwind, or a custom radio group. Each option is a clickable row with a radio indicator on the left.

### Radio Option Styling

- Unselected: `border border-border rounded-lg p-3` -- transparent background
- Selected: `border border-primary bg-card rounded-lg p-3` -- card background with accent border
- Radio indicator: 16px circle, accent fill when selected, border-only when unselected
- Option label: 14px semibold (Label role)
- Option description: 14px regular, `text-muted-foreground` (Body role)

## Focal Point: Change Plan Warning Dialog

**Trigger:** User clicks "Change plan" on a project that already has tasks or phases (D-04).

```
+------------------------------------------+
|  [X]                                      |
|  Change planning approach?                |
|                                           |
|  This project already has tasks. Changing |
|  the tier will not delete existing work,  |
|  but new AI sessions will use the new     |
|  approach.                                |
|                                           |
| [Keep Current Plan   ] [Change Plan     ] |
+------------------------------------------+
```

- "Keep Current Plan" button: `variant="outline"`
- "Change Plan" button: `variant="default"` (accent color, not destructive -- this action does not delete data)
- After confirmation, opens the `TierSelectionDialog` with current tier pre-selected

## Interaction States

### Tier Selection Dialog

| State | Behavior |
|-------|----------|
| Open (no selection) | No radio selected. "Start Planning" button disabled. Textarea empty with placeholder. |
| Radio selected, no description | "Start Planning" enabled for Medium and GSD tiers. Quick tier requires description (shows inline validation "Describe what you're building so AI can create tasks."). |
| Radio selected, description entered | "Start Planning" enabled for all tiers. |
| Submitting | "Start Planning" button shows "Starting..." and is disabled. Dialog stays open until terminal launches successfully. |
| Submit success | Dialog closes. Terminal opens with tier-appropriate context file. For Quick/Medium, plan watcher starts. |
| Submit failure | Dialog stays open. `toast.error("Failed to launch AI: {error}")` -- existing pattern. |

### Open AI Button (updated flow)

| State | Behavior |
|-------|----------|
| Project has no tier and no tasks | Opens `TierSelectionDialog` instead of launching terminal directly |
| Project has stored tier | Skips dialog, generates context file with stored tier, launches terminal (existing flow) |
| Project has tasks but no stored tier | Treats as Quick tier (Phase 13 default), skips dialog, launches terminal |

### Change Plan Button

| State | Behavior |
|-------|----------|
| Project has no tasks/phases | Opens `TierSelectionDialog` directly (no warning needed) |
| Project has tasks or phases | Opens warning dialog first, then `TierSelectionDialog` on confirmation |

## Backend Changes (Non-UI, for planner reference)

These are context file generation changes with no direct React rendering, documented for completeness:

- `generate_context_file_content` in `onboarding.rs` -- Add tier-aware templates per Phase 13 Instruction Matrix. Add "What's next?" execution mode content (D-11 through D-14).
- `OpenAiButton` flow change -- After tier selection, call `set_planning_tier(project_id, tier)` (Phase 12 command) before generating context file.
- Quick tier context template -- Instructs AI to generate flat task list as `plan-output.json`. Includes user's project description.
- Medium tier context template -- Instructs AI to ask 3-5 questions, then generate phases + tasks as `plan-output.json`. Includes user's project description.
- GSD tier context template -- Instructs AI to run `/gsd:new-project`. No plan watcher needed.
- "What's next?" execution mode -- For projects with existing tasks, context file includes progress summary, suggested next task (first incomplete task in earliest phase), and blockers (phases with mixed complete/incomplete progress). Per D-12, no new UI widget; this lives entirely in the context file.
