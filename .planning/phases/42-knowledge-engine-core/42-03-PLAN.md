**Category 1: Stale Sources (Error severity)**
For each wiki article in `knowledge_dir/wiki/`:
1. Parse frontmatter to get `sources` vec.
2. For each `SourceRef`, find the matching raw file in `knowledge_dir/raw/` (match by source name in filename after the hash prefix).
3. If raw file found, compute `compute_content_hash` of its content. If hash differs from `source_ref.hash`, emit `LintIssue { category: StaleSources, severity: Error, article: slug, message: "Source '{file}' has changed since last compile (stored: {old_hash_prefix}..., current: {new_hash_prefix}...)" }`.
4. If raw file not found, emit issue: "Source '{file}' not found in raw/ directory".

**Category 2: Broken Wikilinks (Error severity)**
1. Collect all existing slugs from wiki/ directory (strip `.md` extension).
2. For each article, scan body for `[[slug]]` patterns using regex `\[\[([^\]]+)\]\]`.
3. Also check frontmatter `related` field entries.
4. For each referenced slug not in the slug set, emit `LintIssue { category: BrokenWikilinks, severity: Error, article: slug, message: "Wikilink [[{target}]] points to non-existent article" }`.

**Category 3: Thin Articles (Warning severity)**
For each article, parse body (after frontmatter). If body length < 200 characters (excluding whitespace), emit `LintIssue { category: ThinArticles, severity: Warning, article: slug, message: "Article body is only {len} characters (minimum recommended: 200)" }`.

**Category 4: Orphan Pages (Warning severity)**
1. Build a "referenced set": for each article, add all `related` slugs and all `[[slug]]` targets to the set.
2. For each article slug NOT in the referenced set (not mentioned by any other article), emit `LintIssue { category: OrphanPages, severity: Warning, article: slug, message: "Article is not referenced by any other article" }`.
3. Exception: if there's only 1 article total, skip orphan detection (single article is naturally unreferenced).

**Category 5: Contradictions (Info severity, LLM-powered)**
1. If fewer than 2 articles exist, skip contradiction check.
2. Collect all article summaries from frontmatter (title + summary pairs).
3. Call `provider.complete()` with system prompt:
```
You are a knowledge consistency checker. Given summaries of wiki articles, identify any contradictions or conflicting claims between articles.

Respond with ONLY a JSON array (no markdown fences). Each entry:
{"articles": ["slug-a", "slug-b"], "contradiction": "Description of the contradiction"}

If no contradictions found, respond with: []
```
User message: list of `- {slug}: {title} -- {summary}` entries.
Use `max_tokens: 2048`, `temperature: 0.2`.
4. Parse JSON response. For each contradiction, emit `LintIssue { category: Contradictions, severity: Info, article: articles[0], message: contradiction_description, details: Some(format!("Involves: {}", articles.join(", "))) }`.

**Build LintReport:**
After running all 5 categories, compile the `LintReport`:
- `timestamp`: `chrono::Utc::now().to_rfc3339()`
- `article_count`: number of articles in wiki/
- `issues`: all collected issues
- `summary`: `LintSummary { total_issues: issues.len(), by_category }` where `by_category` counts issues per category name

**knowledge_ingest command:**
```rust
#[tauri::command]
pub async fn knowledge_ingest(
    source_path: String,
    engine: State<'_, KnowledgeEngine>,
    db_state: State<'_, Arc<Mutex<Database>>>,
    gateway: State<'_, AiGateway>,
) -> Result<IngestResult, String> {
    let provider = {
        let db = db_state.lock().map_err(|e| e.to_string())?;
        gateway.get_default_provider(&db).map_err(|e| e.to_string())?
    };

    // Determine source type from path (per D-02)
    let (name, content, source_type) = if source_path.starts_with("http://") || source_path.starts_with("https://") {
        // URL source: name is the URL, content left empty for ingest pipeline to fetch
        (source_path.clone(), String::new(), SourceType::Url)
    } else {
        // File source: read content from disk
        let content = tokio::fs::read_to_string(&source_path)
            .await
            .map_err(|e| format!("Failed to read source file '{}': {}", source_path, e))?;
        let name = std::path::PathBuf::from(&source_path)
            .file_name()
            .unwrap_or_default()
            .to_string_lossy()
            .to_string();
        (name, content, SourceType::File)
    };

    let source = SourceInput {
        name,
        content,
        source_type,
    };

    engine.ingest(source, provider.as_ref())
        .await
        .map_err(|e| e.to_string())
}
```

```rust
#[cfg(test)]
mod tests {
    use super::*;
    use crate::knowledge::test_helpers::MockProvider;

    #[tokio::test]
    async fn test_concurrent_ingest_serializes() {
        // Setup: create a KnowledgeEngine with a tempdir
        let dir = tempfile::tempdir().unwrap();
        let engine = KnowledgeEngine::new(dir.path().to_path_buf());

        // Mock provider that returns valid JSON for ingest
        let mock_response_1 = serde_json::json!({
            "title": "Article One", "slug": "article-one",
            "tags": ["test"], "summary": "First article",
            "related": [], "content": "Content of article one with enough text to pass lint thresholds and be a real article."
        }).to_string();
        let mock_response_2 = serde_json::json!({
            "title": "Article Two", "slug": "article-two",
            "tags": ["test"], "summary": "Second article",
            "related": [], "content": "Content of article two with enough text to pass lint thresholds and be a real article."
        }).to_string();

        let provider1 = MockProvider { response: mock_response_1 };
        let provider2 = MockProvider { response: mock_response_2 };

        let source1 = SourceInput {
            name: "source1.txt".to_string(),
            content: "Raw content for source one".to_string(),
            source_type: SourceType::File,
        };
        let source2 = SourceInput {
            name: "source2.txt".to_string(),
            content: "Raw content for source two".to_string(),
            source_type: SourceType::File,
        };

        // Spawn two concurrent ingest operations
        let engine_ref = &engine;
        let (result1, result2) = tokio::join!(
            engine_ref.ingest(source1, &provider1),
            engine_ref.ingest(source2, &provider2),
        );

        // Both should succeed (not corrupt each other)
        assert!(result1.is_ok(), "First concurrent ingest failed: {:?}", result1.err());
        assert!(result2.is_ok(), "Second concurrent ingest failed: {:?}", result2.err());

        // Index.md should contain exactly 2 entries (not corrupted by concurrent writes)
        let entries = index::read_index(&engine.index_path()).unwrap();
        assert_eq!(entries.len(), 2, "Index should have exactly 2 entries after concurrent ingest, got {}", entries.len());

        // Both articles should exist on disk
        assert!(engine.wiki_dir().join("article-one.md").exists());
        assert!(engine.wiki_dir().join("article-two.md").exists());
    }
}
```

This test proves that two concurrent `engine.ingest()` calls both complete successfully and produce a consistent index.md with exactly 2 entries -- the tokio::Mutex ensures they serialize rather than interleave writes.
