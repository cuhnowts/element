Implement the ingest and query pipelines -- the two primary wiki operations. /// Fetch URL content via reqwest and strip HTML tags to plain text.
/// Used when source_type == SourceType::Url.
pub(crate) async fn fetch_url_content(url: &str) -> Result<String, KnowledgeError> {
    let response = reqwest::get(url)
        .await
        .map_err(|e| KnowledgeError::IoError(format!("Failed to fetch URL '{}': {}", url, e)))?;

    let status = response.status();
    if !status.is_success() {
        return Err(KnowledgeError::IoError(
            format!("URL '{}' returned HTTP {}", url, status)
        ));
    }

    let html = response.text()
        .await
        .map_err(|e| KnowledgeError::IoError(format!("Failed to read response body: {}", e)))?;

    Ok(strip_html_tags(&html))
}

Pipeline steps (per D-01, D-02, D-03, D-04, WIKI-01, WIKI-03, WIKI-06):

0. **Resolve content (per D-02):** Branch on `source.source_type`:
   - `SourceType::File` -> use `source.content` as-is (already loaded by the Tauri command layer).
   - `SourceType::Url` -> call `fetch_url_content(&source.name).await?` to fetch the page and strip HTML. Use the fetched+stripped text as the content for all subsequent steps. Store `source.name` (the URL) as the source name in frontmatter. If the caller already provided pre-fetched content in `source.content` (non-empty), use that instead of fetching (allows the Tauri command layer to pre-fetch if desired).

1. **Compute content hash** using `frontmatter::compute_content_hash(&resolved_content)`.

2. **Check for no-op re-ingest** (WIKI-06): Scan existing wiki articles in `knowledge_dir/wiki/`. For each `.md` file, call `frontmatter::parse_article`, check if any `source_ref.file == source.name`. If found and `source_ref.hash == new_hash`, return `IngestResult { was_noop: true, ... }`.

3. **Store raw source**: Write resolved content to `knowledge_dir/raw/{hash_prefix_8chars}-{source.name}`. Use first 8 chars of hash as prefix for dedup. For URL sources, sanitize the URL to a safe filename (replace `://`, `/`, `?`, `&` with `-`).

4. **LLM compilation** (D-03): Call `provider.complete()` with this system prompt:
```
You are a wiki compiler. Given raw source content, produce a structured wiki article.

Respond with ONLY a JSON object (no markdown fences):
{
  "title": "Human-readable article title",
  "slug": "lowercase-hyphenated-slug",
  "tags": ["tag1", "tag2"],
  "summary": "One-line summary under 100 chars",
  "related": [],
  "content": "Full compiled wiki article content in Markdown. Use [[slug]] for cross-references to other articles."
}

Rules:
- Slug must be lowercase, hyphen-separated, descriptive (e.g., "rust-error-handling")
- Tags should be 2-5 relevant topic keywords
- Content should be well-structured with headers, organized for reference
- Use [[slug]] wikilink syntax when referencing concepts that could be separate articles
- Do not include YAML frontmatter in the content field
```
User message: the resolved source content.
Use `max_tokens: 4096`, `temperature: 0.3`.

5. **Parse LLM response**: Parse the JSON response to extract title, slug, tags, summary, related, content. Handle JSON parse errors gracefully with `KnowledgeError::ParseError`.

6. **Handle slug collision**: Check if `knowledge_dir/wiki/{slug}.md` already exists AND is not the same source (different source name). If collision, append `-2`, `-3`, etc. until unique.

7. **Build WikiFrontmatter**: Create the frontmatter struct with title, slug, sources (vec with one SourceRef containing source.name and hash), created (chrono::Utc::now ISO 8601), updated (same), tags, summary, related. If re-ingesting a changed source (existing article found by source name), preserve the original `created` date and only update `updated`.

8. **Write wiki article**: Combine `frontmatter.to_yaml()` + `\n\n` + LLM content. Write to `knowledge_dir/wiki/{slug}.md`.

9. **Update index.md** (WIKI-03): Call `index::add_entry(index_path, IndexEntry { slug, title, tags, summary })`.

10. **Return IngestResult** with slug, title, source_hash, was_noop: false, article_path.
