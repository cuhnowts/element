**Update `src-tauri/src/commands/plugin_commands.rs`:**

Update the `dispatch_plugin_skill` Tauri command to actually execute handlers asynchronously. The current implementation just calls `host.dispatch_skill()` synchronously. Replace with:

```rust
#[tauri::command]
pub async fn dispatch_plugin_skill(
    state: State<'_, std::sync::Mutex<PluginHost>>,
    skill_name: String,
    input: serde_json::Value,
) -> Result<serde_json::Value, String> {
    // Extract handler and context while holding the lock briefly
    let dispatch_info = {
        let host = state.lock().map_err(|e| e.to_string())?;
        match host.get_skill_handler(&skill_name) {
            Ok((handler, local_name, ctx)) => Some((handler, local_name, ctx)),
            Err(_) => {
                // Fallback to sync dispatch for plugins without handlers
                return host.dispatch_skill(&skill_name, input);
            }
        }
    };
    // Lock is released here

    // Execute handler async (outside the Mutex lock)
    if let Some((handler, local_name, ctx)) = dispatch_info {
        handler.execute(&local_name, input, &ctx).await
    } else {
        Err(format!("No handler found for skill: {}", skill_name))
    }
}
```

This is critical: the `std::sync::Mutex` lock on PluginHost is dropped BEFORE the async handler executes. Otherwise the lock would be held across an await point, which is incorrect with `std::sync::Mutex`. The handler Arc and SkillContext Arc are cloned out while the lock is held, then execution happens after release.
