### Today Indicator Specification (FIX-01)

Only the actual current local day receives these visual treatments:

| Treatment | Today | Other days |
|-----------|-------|------------|
| Date circle fill | `oklch(0.585 0.156 272)` (blue) | none (transparent) |
| Date circle text | `text-primary-foreground` | `text-foreground` |
| Column background | `bg-card` | none |
| "No events today" text | Allowed (when date is actually today) | Must read "No events" (omit "today") |

### Overdue Indicator Specification (FIX-02)

| State | Visual | Source |
|-------|--------|--------|
| Overdue | `text-destructive-foreground` text color on due date label | Deterministic query: `due_date < today AND status != complete` |
| Not overdue | Default `text-muted-foreground` | Normal rendering |

No LLM involvement in overdue determination. Frontend must use a deterministic date comparison matching `heartbeat/risk.rs` logic.

## Interaction Contracts

### FIX-01: Calendar Today Label

| Interaction | Behavior |
|-------------|----------|
| Open calendar week view | Only the actual current local day column shows blue date circle, `bg-card` column background |
| Navigate to different week | No day shows today styling (current day is not in view) |
| Navigate back to current week | Today styling reappears on correct day |

### FIX-02: Overdue Detection

| Interaction | Behavior |
|-------------|----------|
| Task with `due_date < today` and `status != complete` | Rendered with `text-destructive-foreground` on due date, "Overdue" badge |
| Task with `due_date >= today` | Normal rendering, no overdue indicator |
| Task with `status == complete` | No overdue indicator regardless of date |
| No LLM call | Overdue state determined entirely by database query / frontend date comparison |

Dependency: FIX-02 consumes Phase 33 structured briefing JSON output. If Phase 33 is not yet complete, overdue detection falls back to the existing `heartbeat/risk.rs` deterministic engine and frontend `isOverdue()` utility in `src/lib/date-utils.ts`.

### FIX-03: Workflows Minimize

| Interaction | Behavior |
|-------------|----------|
| Click workflow section header | Toggle collapsed/expanded state |
| Chevron icon | `ChevronDown` when expanded, `ChevronRight` when collapsed |
| Default state (first visit) | Collapsed |
| Persistence | `localStorage` key `element:workflows-collapsed`, survives page reload and session restart |
| Expanded content | Existing workflow list with scroll area, create button, empty state |
| Collapsed content | Only section header row visible: "Workflows" label + plus button + chevron |
