# Phase 2 — UI Design Contract

> Visual and interaction contract for the multi-panel workspace: calendar sidebar, task detail center panel, and execution output drawer. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, compact list item gutter |
| sm | 8px | Compact element spacing, sidebar item padding, log line spacing |
| md | 16px | Default element spacing, card padding, panel content inset |
| lg | 24px | Section padding within panels, group separators |
| xl | 32px | Panel header height padding, layout gaps between major regions |
| 2xl | 48px | Welcome dashboard section breaks |
| 3xl | 64px | Not used in this phase |

Exceptions: Sidebar width fixed at 280px. Bottom drawer default height 30% of viewport. Resizable panel drag handle hit target 8px wide (4px visual + 2px each side).

---

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Task descriptions, log output, general content |
| Label | 12px | 500 (medium) | 1.4 | Sidebar section headers, metadata labels, status badges, timestamps |
| Heading | 18px | 600 (semibold) | 1.3 | Task title in detail view, panel titles, welcome dashboard headings |
| Display | 24px | 600 (semibold) | 1.2 | Welcome dashboard greeting only |

**Rationale:** 14px body (not 16px) matches Discord/Outlook information density for a desktop app. Two weights only: 400 for content, 600 for emphasis. Medium (500) used exclusively for small labels where semibold would look too heavy at 12px.

## Color

This phase uses shadcn's dark theme tokens. Exact hex values depend on the preset chosen in Phase 1. The contract declares semantic roles and their CSS variable mappings.

| Role | CSS Variable | Usage |
|------|-------------|-------|
| Dominant (60%) | `--background` | App background, panel backgrounds |
| Secondary (30%) | `--card` / `--muted` | Sidebar background, task cards, calendar cells, drawer background |
| Accent (10%) | `--primary` | Selected task highlight, calendar today indicator, active tab underline, primary CTA buttons |
| Destructive | `--destructive` | Delete task button, clear logs action |

Accent reserved for: selected task in sidebar list, today's date highlight in calendar, active panel tab indicator, "Create Task" CTA button, and running execution status dot. Never applied to static text, borders, or decorative elements.

### Semantic Status Colors

| Status | CSS Variable | Usage |
|--------|-------------|-------|
| Pending | `--muted-foreground` | Pending task icon and status badge |
| In Progress | `--primary` | Running task icon, active execution pulse |
| Complete | `--chart-2` (green tone) | Completed task checkmark, success log entries |
| Blocked | `--destructive` | Blocked task icon, error log entries |
| Scheduled | `--chart-4` (blue tone) | Scheduled workflow calendar markers |

## Component Inventory

### Left Sidebar

| Component | shadcn Base | Behavior |
|-----------|-------------|----------|
| Calendar toggle | `Button` variant="ghost" + `Switch` | Toggles calendar view visibility. Default: visible |
| Mini calendar | Custom component | Month grid, day/week/month tabs via `Tabs`. Today highlighted with accent. Clickable dates filter task list |
| Task list | `ScrollArea` + custom list items | Scrollable list of today's tasks. Each item: status dot + title + time. Click selects task in center panel |
| Workflow list | `ScrollArea` + custom list items | Upcoming scheduled workflows. Each item: icon + name + next run time |
| Section headers | Label typography (12px/500) | "CALENDAR", "TODAY'S TASKS", "WORKFLOWS" |

### Center Panel

| Component | shadcn Base | Behavior |
|-----------|-------------|----------|
| Task header | Custom | Task title (Heading), status badge (`Badge`), priority indicator |
| Task metadata | Custom | Project, tags, created/due dates. Grid layout with Label typography |
| Task description | Custom | Body text, markdown rendered |
| Execution diagram | Custom | Step list (not flowchart). Each step: number + name + status icon + duration. Vertical connector lines between steps |
| Agent/skill/tool chips | `Badge` variant="secondary" | Inline badges showing assigned agents, skills, tools per step |
| Welcome dashboard | Custom | Shown when no task selected. Greeting (Display), recent tasks list, upcoming workflows, quick-create button |

### Bottom Drawer

| Component | shadcn Base | Behavior |
|-----------|-------------|----------|
| Drawer header | Custom | "Output" label, collapse toggle (`Button` ghost), clear logs button |
| Log output | `ScrollArea` + monospace text | Terminal-style log display. Monospace font (JetBrains Mono or system monospace). Auto-scroll to bottom on new entries |
| Log entry | Custom | Timestamp (Label, muted) + level indicator (color-coded) + message (Body, monospace) |
| Empty state | Custom | Centered message when no execution history exists |

### Log Entry Formatting

```
[12:34:56] INFO  Step 1 "Fetch data" started
[12:34:57] INFO  Step 1 completed (1.2s)
[12:34:57] ERROR Step 2 "Process" failed: connection timeout
```

- Timestamps: `--muted-foreground`, monospace
- INFO level: default foreground
- WARN level: `--chart-5` (amber/yellow tone)
- ERROR level: `--destructive`
- Step names: quoted, regular weight
- Durations: `--muted-foreground`, parenthetical

## Interaction Patterns

### Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| `Cmd+N` | Create new task (opens task creation -- delegates to Phase 1 UI) |
| `Cmd+B` | Toggle bottom drawer open/closed |
| `Cmd+[` | Select previous task in sidebar list |
| `Cmd+]` | Select next task in sidebar list |
| `Escape` | Deselect task, return to welcome dashboard |
| `Cmd+K` | Quick-search tasks (future phase, register shortcut now as no-op) |

### States

| State | Visual Treatment |
|-------|-----------------|
| Loading (task detail) | Skeleton placeholders matching task header + metadata + description layout |
| Loading (logs) | Three pulsing skeleton lines in monospace height |
| Loading (calendar) | Skeleton grid matching calendar month layout |
| Selected task (sidebar) | Background: `--accent` at 10% opacity. Left border: 2px solid `--primary` |
| Hover task (sidebar) | Background: `--muted` |
| Running execution | Status dot pulses (CSS animation, 2s ease-in-out infinite). Log panel auto-scrolls |
| Drawer collapsed | Drawer shrinks to header bar only (40px). Drag handle hidden |
| Drawer expanded | Default 30% height. Drag handle visible |

### Transitions

| Transition | Duration | Easing |
|-----------|----------|--------|
| Drawer collapse/expand | 200ms | ease-out |
| Task selection (sidebar highlight) | 100ms | ease-in |
| Skeleton to content | 150ms | fade-in (opacity 0 to 1) |
| Calendar month change | 200ms | slide-left or slide-right |
