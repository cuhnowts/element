## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, credential field icon spacing |
| sm | 8px | Compact element spacing, plugin list item gutter, vault row padding |
| md | 16px | Default element spacing, card padding, settings section inset |
| lg | 24px | Section padding, settings group separators |
| xl | 32px | Settings page content margins, dialog body padding |
| 2xl | 48px | Empty state illustration spacing |
| 3xl | 64px | Not used in this phase |

Exceptions: Settings panel sidebar navigation width 200px. OAuth popup dimensions 480x640px. Credential reveal/copy button group has 4px internal gap.

## Color

Exact values from app.css oklch tokens (dark theme only in current implementation).

| Role | CSS Variable | Usage |
|------|-------------|-------|
| Dominant (60%) | `--background` oklch(0.145 0 0) | Settings page background, dialog backgrounds |
| Secondary (30%) | `--card` oklch(0.205 0 0) / `--muted` oklch(0.269 0 0) | Plugin cards, credential vault rows, step config panels |
| Accent (10%) | `--primary` oklch(0.985 0 0) | "Add Credential" CTA, "Install Plugin" confirmation, active settings tab indicator, OAuth "Connect" button |
| Destructive | `--destructive` oklch(0.396 0.141 25.723) | Delete credential button, remove plugin action, revoke calendar connection |

Accent reserved for: "Add Credential" button, "Connect Calendar" button, active settings navigation tab, plugin enable/disable toggle in active state. Never applied to static text, borders, or decorative elements.

### Plugin Status Colors

| Status | CSS Variable | Visual |
|--------|-------------|--------|
| Active | `--chart-2` (green tone) | Green dot next to plugin name |
| Error | `--destructive` | Red dot + error badge with details on hover/expand |
| Disabled | `--muted-foreground` | Muted text and icon, no status dot |

Source: CONTEXT.md -- multiple accounts, each color-coded.

| Account Slot | CSS Variable | Rationale |

## Component Inventory

### Settings Shell

| Component | shadcn Base | Behavior |
|-----------|-------------|----------|
| Settings page | Custom layout | Replaces center panel when settings active. Back button returns to task view |
| Settings nav | Custom vertical tabs | Left nav within settings. 200px wide. Active tab uses `--primary` left border (2px) |
| Settings back button | `Button` variant="ghost" | Top-left of settings content. Lucide `ArrowLeft` + "Back" label |

### Plugin Management (PLUG-01)

| Component | shadcn Base | Behavior |
|-----------|-------------|----------|
| Plugin list | `ScrollArea` + custom cards | Scrollable list of installed plugins. Each card shows: icon + name + version + status dot + enable toggle |
| Plugin card | `Card` (custom) | Background `--card`. 16px padding. Hover: background `--muted` |
| Plugin status dot | Custom | 8px circle. Color per status table above. Pulsing animation for loading state |
| Plugin error badge | `Badge` variant="destructive" | Shows "Error" text. Click/hover expands to show error message below card |
| Plugin error detail | Custom expandable | Appears below plugin card on error badge click. Monospace error text. "Reload" button to retry |
| Plugin enable toggle | `Switch` | Toggles plugin active/disabled. Fires reload on enable |
| Plugin capability badges | `Badge` variant="secondary" | Shows declared capabilities: "network", "fs:read", "credentials". Inline row below plugin description |
| Plugin empty state | Custom | Centered in content area when no plugins installed |
| Plugin directory button | `Button` variant="outline" | "Open Plugins Folder" -- opens OS file manager to plugins directory |

### Credential Vault (PLUG-02)

| Component | shadcn Base | Behavior |
|-----------|-------------|----------|
| Credential list | `ScrollArea` + custom rows | Vault-style list. Each row: lock icon + credential name + type label + actions |
| Credential row | Custom | Background `--card`. 12px vertical / 16px horizontal padding. Hover: `--muted` |
| Credential value (masked) | Custom | Shows "••••••••••••" by default. Monospace font |
| Reveal button | `Button` variant="ghost" size="icon" | Lucide `Eye` icon. Toggles to `EyeOff` when revealed. Auto-re-masks after 10 seconds |
| Copy button | `Button` variant="ghost" size="icon" | Lucide `Copy` icon. Copies value to clipboard. Shows `Check` icon for 2 seconds as confirmation |
| Delete credential button | `Button` variant="ghost" size="icon" | Lucide `Trash2` icon. `--destructive-foreground` color. Triggers confirmation dialog |
| Add credential dialog | `Dialog` | Title: "Add Credential". Fields: Name (`Input`), Type (`Select`: API Key / Token / Secret / OAuth Token), Value (`Input` type="password"), Notes (`Textarea` optional). CTA: "Save Credential" |
| Edit credential dialog | `Dialog` | Same as add but pre-filled. Title: "Edit Credential". CTA: "Update Credential" |
| Add credential button | `Button` variant="default" | "Add Credential" label. Top-right of credential vault content area |

### Core Plugin Step Config (PLUG-03)

These appear inline within the workflow step editor (Phase 3 establishes the step editor; Phase 4 adds these config panels).

| Component | shadcn Base | Behavior |
|-----------|-------------|----------|
| Shell command config | Custom panel | Fields: Command (`Input` monospace), Working Directory (`Input` with folder icon), Timeout (`Input` type="number" + "seconds" label) |
| HTTP request config | Custom panel | Fields: Method (`Select`: GET/POST/PUT/DELETE/PATCH), URL (`Input`), Headers (key-value pair list with add/remove), Body (`Textarea` monospace, shown for POST/PUT/PATCH), Auth (`Select`: None/Bearer/Basic + credential picker), Timeout (`Input` type="number") |
| File system config | Custom panel | Fields: Operation (`Select`: Read/Write/List/Watch), Path (`Input` with folder icon), Content (`Textarea`, shown for Write only) |
| Credential picker | `Select` with `Command` search | Dropdown listing vault credentials by name. Filtered by type. Shows lock icon per item |
| Key-value pair row | Custom | Two `Input` fields (key + value) side by side + `Button` ghost `X` to remove. "Add Header" button below list |
| Shell warning icon | Lucide `AlertTriangle` | Amber (`--chart-4`) icon next to shell command steps. Tooltip: "Shell commands run with full system access" |

### Calendar Integration (PLUG-04)

| Component | shadcn Base | Behavior |
|-----------|-------------|----------|
| Calendar accounts list | Custom | Shows connected calendar accounts. Each row: color dot + provider icon + account email + sync status + disconnect button |
| Connect Google button | `Button` variant="outline" | Google icon (inline SVG) + "Connect Google Calendar". Launches OAuth popup |
| Connect Outlook button | `Button` variant="outline" | Microsoft icon (inline SVG) + "Connect Outlook Calendar". Launches OAuth popup |
| OAuth popup | Tauri WebView | 480x640px. Shows provider OAuth consent screen. Closes on success/failure |
| Sync status label | Label typography | "Last synced 2 min ago" or "Syncing..." with spinner |
| Manual refresh button | `Button` variant="ghost" size="icon" | Lucide `RefreshCw` icon. Triggers immediate sync |
| Disconnect button | `Button` variant="ghost" size="icon" | Lucide `Unlink` icon. `--destructive-foreground` color. Triggers confirmation |
| Calendar event (in sidebar) | Custom | Renders in existing Phase 2 mini calendar. Color-coded dot per account. Full event details in tooltip on hover |
| Calendar event detail | `Tooltip` or popover | Shows: title, time range, location (if set), attendee count. Read-only |
| Calendar empty state | Custom | Shown in Calendars settings tab when no accounts connected |

## Interaction Patterns

### Keyboard Shortcuts

| Shortcut | Action |
|----------|--------|
| `Cmd+,` | Open Settings page (standard macOS convention) |
| `Escape` | Close Settings and return to task view (when in settings) |
| `Escape` | Close dialog (when dialog is open) |

### States

| State | Visual Treatment |
|-------|-----------------|
| Loading (plugin list) | Three skeleton cards: 64px height each, `--muted` background, pulsing |
| Loading (credential list) | Three skeleton rows: 48px height each, pulsing |
| Loading (calendar sync) | `RefreshCw` icon spins (CSS rotate animation, 1s linear infinite) |
| Plugin card hover | Background transitions to `--muted` |
| Credential row hover | Background transitions to `--muted`, action buttons become visible (hidden by default, visible on hover + focus-within) |
| Credential revealed | Value shown in monospace for 10 seconds, then auto-re-masks. Eye icon changes to EyeOff |
| OAuth in progress | Connect button shows spinner, text changes to "Connecting..." and button is disabled |
| Settings nav active tab | Left border: 2px solid `--primary`. Background: `--muted` |

### Transitions

| Transition | Duration | Easing |
|-----------|----------|--------|
| Settings page open/close | 150ms | ease-out |
| Plugin error detail expand | 200ms | ease-out (height auto transition) |
| Credential reveal/mask | 100ms | fade (opacity) |
| Dialog open/close | 200ms | ease-out (shadcn default) |
| Hover state (cards/rows) | 100ms | ease-in |
| Copy confirmation icon swap | 100ms | fade |
