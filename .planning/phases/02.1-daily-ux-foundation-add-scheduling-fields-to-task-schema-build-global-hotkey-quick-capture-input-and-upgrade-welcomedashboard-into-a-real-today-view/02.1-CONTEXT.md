# Phase 2.1: Daily UX Foundation - Context

**Gathered:** 2026-03-16
**Status:** Ready for planning

<domain>
## Phase Boundary

User can quickly capture tasks via global hotkey from anywhere on the system, see a time-aware today view replacing the WelcomeDashboard, and tasks carry duration/scheduling metadata (due date, scheduled date+time, duration estimate, recurrence) for future scheduling features. This phase adds scheduling fields to the data model, builds the quick-capture floating window, and upgrades the center panel default view.

</domain>

<decisions>
## Implementation Decisions

### Scheduling fields
- Four new optional fields on tasks: due date, scheduled date, scheduled time, duration estimate, recurrence rule
- Due date: date only (no time component)
- Scheduled date + time: specific clock time ("2:00 PM"), not loose time-of-day slots
- Duration estimate: preset chips (15m, 30m, 1h, 2h, 4h, custom)
- Recurrence: simple presets only (daily, weekdays, weekly, monthly) — no full RRULE support yet
- Recurrence is visual indicator only — no auto-creation of next instance (deferred to Phase 3's scheduling engine)
- All fields optional — tasks work fine without any scheduling metadata
- Fields appear as inline chips/badges in the task detail header area (visible at a glance)
- Task list rows get a small due date indicator badge (overdue flag) — keeping rows otherwise compact

### Quick-capture UX
- Floating mini-window triggered by Ctrl+Space global hotkey (system-wide, works even when app is minimized/background)
- Uses Tauri's global-shortcut plugin for system-wide registration
- Window appears center-screen (Spotlight/Raycast style), title field auto-focused
- Available fields at capture time: title (required), project selector, priority, due date, duration estimate
- Project defaults to currently selected project; priority defaults to medium
- Enter to save, Escape to dismiss
- Window disappears after submit

### Today view design
- Replaces WelcomeDashboard entirely — today view IS the new center panel default when no task is selected
- Time-grouped sections: Overdue, Morning (before 12pm), Afternoon (12-5pm), Evening (after 5pm), Unscheduled
- Fixed time boundaries (not configurable in this phase)
- Shows tasks across ALL projects (not filtered to selected project) — project name shown as subtle label per task
- Header: time-of-day greeting + progress stats ("4 of 9 tasks complete" with progress bar + "2 overdue · 3 upcoming")

### Time awareness
- Overdue tasks pinned in a dedicated section at top with warning color — cannot be missed
- Completed tasks fade (reduced opacity + strikethrough) but remain visible in their section
- Next upcoming incomplete task gets a subtle left border accent highlight
- Time groups use fixed boundaries: Morning < 12pm, Afternoon 12-5pm, Evening > 5pm

### Claude's Discretion
- Quick-capture window styling, animation, and exact dimensions
- Progress bar visual style and colors
- Time group section header styling
- Overdue warning color treatment
- "Next up" highlight accent color
- Duration chip visual design
- Recurrence indicator icon choice
- SQLite migration strategy for new fields
- Tauri command signatures for new scheduling CRUD

</decisions>
