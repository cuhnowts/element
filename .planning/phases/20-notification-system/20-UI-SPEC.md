## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding, inline spacing |
| sm | 8px | Notification item internal padding, gap between icon and text |
| md | 16px | Popover content padding, section spacing |
| lg | 24px | Popover header/footer padding |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: Unread badge is 16px (size-4) circle -- fits 4px grid. Bell button touch target is 32px (size-sm button).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 600 (semibold) | 1.33 |
| Heading | 14px | 600 (semibold) | 1.2 |
| Display | Not used | -- | -- |

Phase-specific usage:
- **Body (14px/400):** Notification body text, empty state body
- **Label (12px/600):** Priority badges, timestamp text, unread badge count (10px exception -- see below), tab labels in drawer handle
- **Heading (14px/600):** Notification title text, popover header "Notifications", bulk action button labels

Exception: Unread badge count rendered at 10px (`text-[10px]`) to fit inside the 16px badge circle. This is display-only, not a readable text size.

| Dominant (60%) | `oklch(0.145 0 0)` / `--background` | App background behind popover |

### Priority Badge Colors

| Priority | Badge Background | Badge Text | Sonner Variant |
|----------|-----------------|------------|----------------|
| Critical | `--destructive` (red) | `--destructive-foreground` | `toast.error()` |
| Informational | `--chart-2` (teal/blue) | `--foreground` | `toast.info()` |
| Silent | `--muted` (neutral gray) | `--muted-foreground` | No toast |

Accent reserved for: Unread notification badge on bell icon (uses `--destructive` red to draw attention). Priority badges on notification items. "Mark all as read" and "Clear all" interactive text buttons.

## Component Inventory

### NotificationBell (titlebar area -- drawer handle bar)

| Property | Value |
|----------|-------|
| Placement | Right side of ResizableHandle bar, after "Clear Logs" button area |
| Icon | `Bell` from lucide-react (size-4, 16px) |
| Button | `variant="ghost" size="sm"` -- matches existing drawer handle controls |
| Unread badge | 16px red circle (`bg-destructive`), positioned `absolute -top-1 -right-1`, shows count 1-9 or "9+" |
| Zero unread | No badge shown, plain bell icon |
| Click | Opens Popover anchored to bell, aligned `end` |

### NotificationPopover

| Property | Value |
|----------|-------|
| Width | 320px (`w-80`) |
| Max height | 400px with vertical scroll (`max-h-[400px] overflow-y-auto`) |
| Alignment | `align="end"` (right-aligned to bell icon) |
| Header | "Notifications" heading (14px/600) + unread count in parentheses |
| Bulk actions | "Mark all as read" and "Clear all" as ghost text buttons (12px/600), right-aligned in header |
| Empty state | Centered in popover body when no notifications exist |
| List | Flat reverse-chronological, no grouping, no tabs, no filters (per D-04) |
| Separator | `border-b border-border` between header and list, between each notification item |

### NotificationItem

| Property | Value |
|----------|-------|
| Layout | Horizontal: priority badge (left) + content (center, flex-1) + timestamp (right) |
| Priority badge | 6px dot or small pill with priority label text (12px/600) |
| Title | 14px/600, single line, truncate with ellipsis |
| Body | 14px/400, max 2 lines, truncate with ellipsis (`line-clamp-2`) |
| Timestamp | 12px/400, `text-muted-foreground`, relative time ("2m ago", "1h ago", "Yesterday") |
| Unread indicator | Left border 2px `border-l-2 border-destructive` on unread items, or subtle background tint `bg-muted/50` |
| Hover | `hover:bg-muted` background transition |
| Click | Navigate to `action_url` target (project/phase/task deep link), mark as read |
| Read state | No left border, no background tint, slightly dimmed text (`opacity-70`) |

### Sonner Toast Integration (existing component, no changes)

| Priority | Variant | Duration | Behavior |
|----------|---------|----------|----------|
| Critical | `toast.error()` | 8000ms | Red accent, OctagonX icon, persists longer |
| Informational | `toast.info()` | 4000ms | Default popover style, Info icon |
| Silent | No toast | -- | History only, no visual interruption |

## Interaction States

### Bell Icon States

| State | Visual | Badge |
|-------|--------|-------|
| No notifications | Bell icon, muted-foreground color | None |
| Has unread (1-9) | Bell icon, foreground color | Red circle with count |
| Has unread (10+) | Bell icon, foreground color | Red circle with "9+" |
| Popover open | Bell icon, bg-muted background (active state) | Badge remains visible |

### Notification Item States

| State | Visual |
|-------|--------|
| Unread | Left red border, subtle muted background tint |
| Read | No border, no tint, `opacity-70` on text |
| Hover | `bg-muted` background |
| Click/Active | Navigate + mark read |

### Popover Transitions

| Trigger | Behavior |
|---------|----------|
| Bell click (closed) | Popover opens with Radix default animation (fade + slight scale) |
| Bell click (open) | Popover closes |
| Click outside | Popover closes |
| Escape key | Popover closes |
| Notification click | Navigate to target, mark as read, popover stays open |
| "Mark all as read" | All items transition to read state, unread count resets to 0, badge disappears |
| "Clear all" | All items removed, empty state shown |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | N/A -- notification system is passive (no user-initiated creation) |
| Popover heading | "Notifications" |
| Bulk action: mark read | "Mark all as read" |
| Bulk action: clear | "Clear all" |
| Empty state heading | "No notifications" |
| Empty state body | "You're all caught up. Notifications from your projects will appear here." |
| Error state (OS permission denied) | Log warning to console: "Desktop notifications unavailable -- permission denied. Using in-app notifications only." No user-facing error -- graceful fallback. |

## Layout Integration

### Bell Icon Placement (in AppLayout.tsx ResizableHandle)

The bell icon mounts inside the existing `ResizableHandle` bar, in the right-side `div.flex.items-center.gap-2` container, after any conditional buttons (like "Clear Logs"). This ensures:
- Visible regardless of which panel is active
- Consistent with existing control placement pattern
- Does not interfere with drawer tab buttons on the left

```
[GripHorizontal] [Logs] [History] [Terminal]          [Clear Logs?] [BellIcon+Badge]
```

### Deep-Link Navigation Format

Notification `action_url` values use internal route-style strings parsed on click:

| Format | Navigation Action |
|--------|-------------------|
| `project/{id}` | `selectProject(id)` |
| `project/{id}/phase/{phaseId}` | `selectProject(id)` + expand phase |
| `project/{id}/task/{taskId}` | `selectProject(id)` + `setSelectedTaskId(taskId)` |

No actual router changes needed -- these map to existing Zustand store actions.
