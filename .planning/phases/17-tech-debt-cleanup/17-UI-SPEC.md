### DEBT-03 Error Toast Interaction Contract

The navigation bug fix (DEBT-03) is the sole UI-visible change in Phase 17. The contract:

1. **No navigation on error:** If `launchTerminalCommand` or any step in `handleOpenAi` fails or encounters invalid state, the user MUST remain on ProjectDetail. No `navigate()` call, no route change, no view switch.
2. **Toast on error:** Show `toast.error()` with a message following the established pattern: "[Problem description]. [What to do next]."
3. **Button state reset:** `isLaunching` MUST return to `false` in all error paths (already handled by `finally` block -- verify preservation).
4. **Existing toasts preserved:** The three existing error toasts in OpenAiButton (no directory, no CLI tool, CLI tool not found) remain unchanged.
5. **New error path:** Whatever state race or side effect causes navigation-to-home must be caught and converted to a toast + early return, matching the existing error handling pattern in lines 27-56 of OpenAiButton.tsx.

### Copy Guidelines for New Error Toasts

Follow the established pattern visible in the existing codebase:
- "Link a project directory first. The AI tool needs a directory to work in."
- "No AI tool configured. Set one up in Settings > AI."
- "${command} not found on your system. Check the command in Settings > AI."

Pattern: [State the problem plainly]. [Tell the user exactly what to do].
