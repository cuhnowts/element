```sql
-- calendar_events table (005_plugins_credentials_calendar.sql)
-- start_time/end_time are full ISO datetime strings (e.g., "2026-04-03T10:00:00")
CREATE TABLE calendar_events (
    id TEXT PRIMARY KEY,
    account_id TEXT NOT NULL,
    title TEXT NOT NULL,
    description TEXT NOT NULL DEFAULT '',
    location TEXT,
    start_time TEXT NOT NULL,    -- Full ISO datetime
    end_time TEXT NOT NULL,      -- Full ISO datetime
    all_day INTEGER NOT NULL DEFAULT 0,
    attendees TEXT NOT NULL DEFAULT '[]',
    status TEXT NOT NULL DEFAULT 'confirmed',
    updated_at TEXT NOT NULL
);

-- scheduled_blocks table (006_ai_scheduling.sql)
-- start_time/end_time are HH:mm format
CREATE TABLE scheduled_blocks (
    id TEXT PRIMARY KEY,
    schedule_date TEXT NOT NULL,              -- YYYY-MM-DD
    task_id TEXT REFERENCES tasks(id) ON DELETE CASCADE,
    block_type TEXT NOT NULL CHECK(block_type IN ('work', 'meeting', 'buffer')),
    start_time TEXT NOT NULL,                 -- HH:mm
    end_time TEXT NOT NULL,                   -- HH:mm
    is_confirmed INTEGER NOT NULL DEFAULT 0,
    source_event_id TEXT,
    created_at TEXT NOT NULL
);

-- work_hours table
-- SELECT start_time, end_time, work_days, buffer_minutes, min_block_minutes FROM work_hours WHERE id = 1
```

Rust find_open_blocks algorithm (to port to TypeScript):
1. Parse work_start and work_end from work_hours config
2. For each calendar event, clamp to work hours, apply buffer_minutes before and after
3. Sort occupied ranges by start time
4. Merge overlapping occupied ranges
5. Walk from work_start to work_end collecting gaps >= min_block_minutes
6. Return array of {start, end, duration_minutes}

CRITICAL difference in TypeScript port: Also treat existing scheduled_blocks (is_confirmed=1) as occupied time (no buffer on work blocks, only on calendar events). This prevents double-booking.

Create `mcp-server/src/tools/calendar-tools.ts` with these 5 exported functions:

**1. handleListCalendarEvents(db, args: {startDate: string, endDate: string})**
- Query: `SELECT id, title, description, location, start_time, end_time, all_day, status FROM calendar_events WHERE start_time >= ? AND end_time <= ? ORDER BY start_time`
- Params: `args.startDate + "T00:00:00"` and `args.endDate + "T23:59:59"` (per D-09, dates arrive as YYYY-MM-DD, DB stores full ISO datetimes)
- Return: `{content: [{type: "text", text: JSON.stringify(rows, null, 2)}]}`

**2. handleGetAvailableSlots(db, args: {date: string})**
- Read work_hours: `SELECT start_time, end_time, work_days, buffer_minutes, min_block_minutes FROM work_hours WHERE id = 1`
- Default if no row: `{start_time: "09:00", end_time: "17:00", work_days: "mon,tue,wed,thu,fri", buffer_minutes: 10, min_block_minutes: 30}`
- Read calendar events for date: `SELECT start_time, end_time FROM calendar_events WHERE start_time >= ? AND start_time < ?` using `date + "T00:00:00"` and date+1 day + "T00:00:00"`. Extract HH:mm from ISO datetime using `.slice(11, 16)`.
- Read existing confirmed work blocks: `SELECT start_time, end_time FROM scheduled_blocks WHERE schedule_date = ? AND is_confirmed = 1`
- Port the `find_open_blocks` algorithm from `time_blocks.rs`:
  - Parse work_start and work_end as minutes-since-midnight (e.g., "09:00" -> 540)
  - For calendar events: clamp to work hours, apply buffer_minutes before/after, collect as occupied ranges
  - For scheduled_blocks: clamp to work hours, NO buffer, collect as occupied ranges
  - Sort all occupied ranges by start
  - Merge overlapping ranges
  - Walk from work_start to work_end collecting gaps >= min_block_minutes
  - Return each gap as `{start: "HH:mm", end: "HH:mm", duration_minutes: number}` (per D-07, D-10)
- Return: `{content: [{type: "text", text: JSON.stringify(slots, null, 2)}]}`

**3. handleCreateWorkBlock(db, dbPath, args: {date: string, taskId: string, startTime: string, endTime: string})**
- Validate taskId exists: `SELECT id FROM tasks WHERE id = ?`. If not found, return `{content: [{type: "text", text: "Error: Task not found..."}], isError: true}`
- Generate id with `randomUUID()`
- Insert: `INSERT INTO scheduled_blocks (id, schedule_date, task_id, block_type, start_time, end_time, is_confirmed, created_at) VALUES (?, ?, ?, 'work', ?, ?, 1, ?)`
- Call `emitDataChanged(dbPath, "schedule-changed")`
- Return JSON with {id, date, taskId, startTime, endTime}

**4. handleMoveWorkBlock(db, dbPath, args: {blockId: string, startTime: string, endTime: string})**
- Check block exists: `SELECT id, schedule_date FROM scheduled_blocks WHERE id = ?`. If not found, return isError.
- Update: `UPDATE scheduled_blocks SET start_time = ?, end_time = ? WHERE id = ?`
- Call `emitDataChanged(dbPath, "schedule-changed")`
- Return JSON with {blockId, startTime, endTime}

**5. handleDeleteWorkBlock(db, dbPath, args: {blockId: string})**
- Check block exists: `SELECT id FROM scheduled_blocks WHERE id = ?`. If not found, return isError.
- Delete: `DELETE FROM scheduled_blocks WHERE id = ?`
- Call `emitDataChanged(dbPath, "schedule-changed")`
- Return JSON with {blockId, deleted: true}

Import `emitDataChanged` from `"./write-tools.js"` and `randomUUID` from `"node:crypto"`.
