# Phase 29 — UI Design Contract

> Visual and interaction contract for Calendar MCP Tools. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

Phase 29 is primarily a backend/API phase. It exposes five MCP tools for reading calendar events and managing work blocks. The UI surface is limited to approval confirmations for destructive tool actions rendered in the existing hub chat, and tool result messages. No new screens, components, or layouts are introduced.

## Interaction Contracts

### Tool Approval Flow (Destructive Tools)

The three write tools (`create_work_block`, `move_work_block`, `delete_work_block`) are flagged `destructive: true` in actionRegistry. When the bot calls one of these tools:

1. An ApprovalRequest card appears in the agent activity tab
2. Card shows the tool title and a human-readable description of what will happen
3. User clicks "Approve" or "Reject"
4. On approval: Tauri command executes, work block appears/moves/disappears in the Phase 27 calendar view
5. On rejection: tool call is skipped, bot is informed

No changes to the approval flow mechanics -- existing ApprovalRequest component handles all states (pending, approved, rejected).

### Read Tool Results (Non-Destructive Tools)

The two read tools (`list_calendar_events`, `get_available_slots`) execute immediately without approval. Results are returned as structured text to the LLM, which narrates them in chat. No special UI rendering for tool results beyond the bot's natural language response.

### Calendar View Reactivity

When a work block is created or moved via MCP tool approval, the Phase 27 calendar view must reactively update. This is handled by the existing Zustand store invalidation pattern -- the Tauri command response triggers a store refresh, and the calendar grid re-renders.

## Action Registry Entries

New entries to add to `src/lib/actionRegistry.ts`. These define the tool shapes for both MCP server and hub chat bot.

| Tool Name | Description (for LLM) | Destructive | Key Parameters |
|-----------|----------------------|-------------|----------------|
| list_calendar_events | "List calendar events for a date range. Returns event titles, times, and sources." | false | date_start (YYYY-MM-DD), date_end (YYYY-MM-DD) |
| get_available_slots | "Get available time slots for a given day. Returns gaps between meetings and work blocks." | false | date (YYYY-MM-DD) |
| create_work_block | "Schedule a work block for a task on the calendar. Requires user approval." | true | task_id, date (YYYY-MM-DD), start_time (HH:mm), end_time (HH:mm) |
| move_work_block | "Move an existing work block to a new time. Requires user approval." | true | block_id, date (YYYY-MM-DD), start_time (HH:mm), end_time (HH:mm) |
| delete_work_block | "Remove a work block from the calendar. Requires user approval." | true | block_id |
