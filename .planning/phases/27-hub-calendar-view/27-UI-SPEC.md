## Grid Dimensions

These are the pixel-level specifications for the time grid layout.

| Dimension | Value | Rationale |
|-----------|-------|-----------|
| Hour slot height | 48px (24px per 30-min) | Compact enough for 8+ hours visible; tall enough for event text |
| Time gutter width | 56px | Fits "12:00 PM" label with 8px horizontal padding |
| Day column min-width | 120px | Prevents text truncation on short titles in week view |
| Event block min-height | 20px | Even 15-min events remain clickable |
| Event block border-radius | 6px (var(--radius) minus 4px) | Slightly tighter than card radius for nested feel |
| All-day banner height | 28px per row | Stacks if multiple all-day events |
| Header bar height | 40px | Contains date nav + view toggle |
| Overlap column split | Equal width per overlapping event | Max 4 columns; 5+ events show "+N more" badge |

## Interaction Contracts

### D-06: View Toggle (Day | Week)

- Segmented control in header bar, right-aligned
- Two segments: "Day" and "Week"
- Active segment uses `--color-secondary` background with `--color-foreground` text
- Inactive segment uses transparent background with `--color-muted-foreground` text
- Switching view preserves the selected date (day view shows that day; week view shows the week containing that day)

### D-10: Work Block Click

- Cursor: `pointer`
- Hover: background opacity increases from 15% to 25%
- Click: navigates to task detail view via existing workspace store navigation
- The entire block is the click target

### D-11: External Meeting Click

- Cursor: `default`
- Hover: background opacity increases from 40% to 55%
- Click: opens Popover anchored to the block showing meeting details
- Popover contents: title (16px semibold), time range (12px regular), location (12px regular, muted), attendees list (12px regular, muted), separator, status badge

### D-12: Now-Line

- 2px tall horizontal line at the pixel position corresponding to current time
- Indigo color (`oklch(0.585 0.156 272)`)
- 6px diameter circle at the left edge (overlapping the time gutter)
- Updates position every 60 seconds
- On initial load, scroll position centers the now-line vertically in the viewport

### D-13: Date Navigation

- Left arrow: moves back 1 day (day view) or 1 week (week view)
- Right arrow: moves forward 1 day or 1 week
- "Today" button: returns to current date
- Today button disabled when already viewing today
- Date display format: "Wed, Apr 3" (day view) or "Mar 31 - Apr 4" (week view)
- Date display is 16px semibold

### D-14: MiniCalendar Coordination

- Clicking a date in the sidebar MiniCalendar updates the hub calendar grid to show that date
- Coordination via a shared selectedDate in the Zustand store (new state field)
- MiniCalendar already has `onDateSelect` prop -- wire this to the shared state

### Scroll Behavior

- Vertical scroll only (no horizontal scroll in day view; horizontal implicit in week column layout)
- On load: auto-scroll to place the now-line at roughly 1/3 from top of viewport
- Scroll position resets when changing dates
- Scroll position preserved when toggling day/week on the same date range

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Event blocks keyboard-navigable | Tab order flows through events chronologically; Enter activates click action |
| Screen reader announcements | Each event block has aria-label: "{title}, {start time} to {end time}" |
| Color not sole differentiator | Work blocks have dashed/lighter border pattern in addition to color difference from meetings |
| Now-line | aria-hidden="true" (decorative) |
| View toggle | Role="tablist" with aria-selected on active tab |
| Minimum contrast | All text on event blocks meets WCAG 2.1 AA (4.5:1 for 12px text, 3:1 for 14px+) |
