## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 24px | 600 (semibold) | 1.2 |
| Card Title | 16px | 500 (medium) | 1.4 |

Body is the default for card content text, blocker descriptions, deadline items, and win items. Label is used for muted metadata (timestamps, tag labels) -- distinguished by `--color-muted-foreground`, not weight. Heading is the greeting line only. Card Title is used for project card names and the summary card heading.

Source: existing `body { font-size: 14px; line-height: 1.5 }` in app.css. Heading matches `BriefingGreeting` at `text-2xl font-semibold`. Card Title matches existing `CardTitle` at `text-base font-medium`.

## Interaction Contract

### Briefing Generation Flow (BRIEF-01, D-03)

1. Hub loads: user sees greeting + contextual summary + action chips. No briefing auto-fire.
2. User clicks "Run Daily Briefing" chip.
3. Chip enters active/streaming state (spinner, "Generating...", disabled).
4. Backend builds manifest, runs scoring engine, sends scored data to LLM.
5. LLM streams structured JSON. Frontend parses and renders cards incrementally:
   - Summary card appears first.
   - Project cards appear one by one as JSON is parsed.
6. On stream complete: chip returns to default state. Cards remain in chat stream.

### Briefing Regeneration (D-11)

Clicking "Run Daily Briefing" again replaces the existing briefing cards in-place. Chat messages below the briefing are preserved. Only one briefing instance is visible at a time.

### Project Card Navigation (D-09)

Clicking a project card header navigates to that project's detail view. Uses existing app navigation (not a new route -- reuses sidebar project selection).

### Collapsible Sections (D-09)

| Interaction | Behavior |
|-------------|----------|
| Click section header | Toggle expand/collapse |
| Keyboard Enter on section header | Toggle expand/collapse |
| Keyboard focus | Section headers are focusable (`tabIndex={0}`) |

### Back-to-Top Button

| Property | Value |
|----------|-------|
| Visibility | Shown when scrolled past action chips area |
| Position | Fixed bottom-right of hub center, 16px from edges |
| Style | `Button variant="secondary" size="icon"` with `rounded-full` |
| Icon | `ChevronUp` 16px |
| Accessibility | `aria-label="Back to top"` |

## Data Contract (D-12, D-13)

### BriefingJSON Schema

LLM returns structured JSON (not markdown). Frontend renders each field as a distinct card.

```typescript
interface BriefingJSON {
  summary: string;          // 1-2 sentence LLM-narrated day overview
  projects: BriefingProject[];  // Ranked by priority (most urgent first)
}

interface BriefingProject {
  name: string;             // Project display name
  projectId?: number;       // Database ID for navigation
  tags: BriefingTag[];      // Scoring engine tags
  blockers: string[];       // Blocker descriptions
  deadlines: string[];      // Deadline descriptions
  wins: string[];           // Recent completions
}

type BriefingTag = "overdue" | "approaching-deadline" | "blocked" | "on-track" | "recently-completed";
```

The scoring engine in Rust produces the ranked data and tags. The LLM receives this data and adds narrative flavor to the `summary` field and each item description. The math is the source of truth; the LLM is the copywriter.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Greeting (morning) | "Good morning. Here's your day." |
| Greeting (afternoon) | "Good afternoon. Here's where things stand." |

## Accessibility Contract

| Concern | Implementation |
|---------|----------------|
| Briefing cards | `role="region"` with `aria-label="Daily briefing"` on the card container |
| Summary card | `aria-label="Briefing summary"` |
| Project cards | `aria-label="{project name} briefing"` |
| Collapsible sections | `aria-expanded` on toggle button, `aria-controls` pointing to content ID |
| Action chips | Standard `Button` semantics; disabled chips get `aria-disabled="true"` |
| Streaming state | `aria-live="polite"` on the briefing card container so screen readers announce new cards |
| Back-to-top | `aria-label="Back to top"` |
| Keyboard | All interactive elements reachable via Tab. Collapsible sections toggle on Enter/Space. |
