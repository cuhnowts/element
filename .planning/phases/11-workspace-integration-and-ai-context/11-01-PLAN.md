**In src-tauri/src/models/onboarding.rs**, add a new function `generate_context_file_content` that takes structured project data and returns a markdown string. The function signature:

```rust
pub struct ProjectContextData {
    pub project_name: String,
    pub project_description: String,
    pub phases: Vec<PhaseContextData>,
    pub unassigned_tasks: Vec<TaskContextData>,
    pub total_tasks: usize,
    pub completed_tasks: usize,
    pub in_progress_tasks: usize,
    pub is_empty: bool,
}

pub struct PhaseContextData {
    pub name: String,
    pub sort_order: i32,
    pub tasks: Vec<TaskContextData>,
    pub completed: usize,
    pub total: usize,
}

pub struct TaskContextData {
    pub title: String,
    pub status: String,
    pub description: String,
}
```

For **populated projects** (is_empty = false), generate markdown with these sections:
1. `# Project Context: {project_name}` header
2. `## Overview` — project description, overall progress (e.g., "12/20 tasks complete (60%)")
3. `## Phases` — each phase with name, progress bar text (e.g., "[3/5]"), and task list showing title + status
4. `## Unassigned Tasks` — tasks not in any phase (if any)
5. `## What Needs Attention` — list the first 5 incomplete tasks (status pending or in-progress), starting with in-progress ones
6. `## Output Contract` — same plan-output.json schema from existing `generate_skill_file_content`, so the AI can generate structured plans if the user asks

For **empty projects** (is_empty = true, no phases and no tasks), generate:
1. `# Project Onboarding: {project_name}` header
2. `## Project Context` — name and description
3. `## Your Task` — same onboarding instructions as existing `generate_skill_file_content` (ask about deliverables, constraints, priorities)
4. `## Output Contract` — same plan-output.json schema

**In src-tauri/src/commands/onboarding_commands.rs**, add a new command:

```rust
#[tauri::command]
pub async fn generate_context_file(
    state: State<'_, std::sync::Arc<std::sync::Mutex<Database>>>,
    project_id: String,
) -> Result<String, String> {
    // 1. Lock DB, get project
    // 2. Get phases via db.list_phases(&project_id)
    // 3. Get tasks via db.list_tasks(&project_id)
    // 4. Build ProjectContextData from the query results
    //    - Group tasks by phase_id
    //    - Separate unassigned (phase_id is None)
    //    - Count completed/in-progress
    //    - is_empty = phases.is_empty() && tasks.is_empty()
    // 5. Call generate_context_file_content(&data)
    // 6. Write to {project.directory_path}/.element/context.md
    //    - Create .element dir if needed (std::fs::create_dir_all)
    //    - Error if project has no directory_path
    // 7. Return the file path as String
}
```

The command must:
- Return `Err("Project has no linked directory".into())` if `project.directory_path` is None
- Create `.element/` directory if it doesn't exist
- Write the context file to `.element/context.md`
- Return the full path to the context file

**In src-tauri/src/lib.rs**, add `generate_context_file` to the `.invoke_handler(tauri::generate_handler![...])` list.

**In src/lib/tauri.ts**, add the API binding:
```typescript

  invoke<string>("generate_context_file", { projectId }),
```
Add it in the "// Onboarding" section, after `generateSkillFile`.

Note: The `generate_context_file` command only writes the context file and returns the path. It does NOT start the plan watcher or spawn the CLI tool. The plan watcher (`startPlanWatcher`) is called by OpenAiButton in Plan 11-03 after generating the context file, keeping concerns separated.
