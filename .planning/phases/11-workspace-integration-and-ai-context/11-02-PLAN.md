2. Add a `useEffect` that watches `selectedProjectId`. When it changes:
   - If there was a previous project, call `saveCurrentProjectState(prevProjectId)` to snapshot the current state
   - Call `restoreProjectState(newProjectId)` to apply the new project's state
   - Use a `useRef` to track the previous project ID
3. Replace the `activeProjectTab` usage: instead of reading from `fileExplorerSlice`, read from `useWorkspaceStore.getProjectState(selectedProjectId).centerTab`. When the user changes tabs (via `ProjectTabBar`), call `setProjectCenterTab(selectedProjectId, tab)` AND the existing `setActiveProjectTab(tab)` for backward compat.

Alternatively, keep the simpler approach: on project switch, read the per-project state and call `setActiveProjectTab(restoredTab)` to sync it into the existing global state. This minimizes changes to the tab bar.

**Implementation approach (simpler)**:
```typescript
const prevProjectRef = useRef<string | null>(null);

useEffect(() => {
  if (!selectedProjectId) return;

  // Save outgoing project state
  if (prevProjectRef.current && prevProjectRef.current !== selectedProjectId) {
    saveCurrentProjectState(prevProjectRef.current);
  }

  // Restore incoming project state
  restoreProjectState(selectedProjectId);
  const projectState = getProjectState(selectedProjectId);
  setActiveProjectTab(projectState.centerTab);

  prevProjectRef.current = selectedProjectId;
}, [selectedProjectId]);
```

Also, when `setActiveProjectTab` is called (tab change by user), also update the per-project state:
```typescript
const handleTabChange = (tab: "detail" | "files") => {
  setActiveProjectTab(tab);
  if (selectedProjectId) {
    setProjectCenterTab(selectedProjectId, tab);
  }
};
```
