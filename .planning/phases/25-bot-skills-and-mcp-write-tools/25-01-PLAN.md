    Create `src/lib/shellAllowlist.ts` with:

    1. `SHELL_METACHARACTERS` regex: `/[;|&`$()><]/` -- characters that indicate shell injection attempts

    2. `DEFAULT_ALLOWLIST: string[]` constant (per D-07):
       ```
       "git status", "git log", "git diff", "git branch",
       "npm test", "npm run", "npm build",
       "yarn test", "yarn run", "yarn build",
       "pnpm test", "pnpm run", "pnpm build",
       "ls", "cat", "head", "tail", "wc", "echo", "date", "pwd"
       ```

    3. `parseBaseCommand(command: string): string` function:
       - Trim whitespace
       - Split on whitespace
       - For known multi-word commands (git, npm, yarn, pnpm), return first 2 tokens joined by space
       - For single-word commands (ls, cat, etc.), return first token only

    4. `isCommandAllowed(command: string, allowlist: string[]): boolean` function:
       - Return false if command is empty after trimming
       - Return false if command contains any SHELL_METACHARACTERS (per Research Pitfall 4)
       - Parse base command using parseBaseCommand
       - Check if base command matches any entry in the allowlist (exact match)
       - Return true if match found, false otherwise
