## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 400 (regular) | 1.4 |
| Heading | 16px | 600 (semibold) | 1.3 |
| Code | 14px | 400 (regular, monospace) | 1.4 |

Usage mapping:
- **Body (14px/400):** Chat messages, action result descriptions, confirmation card body text, settings field descriptions
- **Label (12px/400):** Action status badges, shell output header ("Shell Output"), collapsed code block summary, allowlist item labels
- **Heading (16px/600):** Confirmation card title ("Confirm Action"), settings section headers ("Shell Allowlist")
- **Code (14px/400 monospace):** Shell command display, shell output content, code blocks in action results. Font: `"SF Mono", "Fira Code", "Cascadia Code", monospace`

## Color

All colors use existing CSS custom properties defined in `src/app.css` (oklch neutral palette, dark theme).

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-background` oklch(0.145 0 0) | Chat background, settings page background |
| Secondary (30%) | `--color-card` oklch(0.205 0 0) | Confirmation cards, action result cards, shell output blocks, settings fields |
| Accent (10%) | `--color-primary` oklch(0.985 0 0) | See reserved list below |
| Destructive | `--color-destructive` oklch(0.396 0.141 25.723) | Destructive confirmation card border, reject button, destructive action badge |

Additional semantic tokens used:
- `--color-muted-foreground` oklch(0.708 0 0): Shell output header text, collapsed code block summary, secondary info in action cards
- `--color-border` oklch(0.269 0 0): Confirmation card border (non-destructive), shell output block border, settings section dividers
- `--color-secondary` oklch(0.269 0 0): Hover backgrounds on allowlist items, code block background

Accent reserved for:
- Approve button in confirmation cards
- Success status indicator on completed actions
- Shell command name highlight in output header
- Active toggle state in settings allowlist

## Layout Contract

### Confirmation Card (inline in chat)

```
+----------------------------------------------------+
| [icon] Create Task                          [badge] |
|                                                     |
| Create task "Fix login bug" in Project Alpha?       |
|                                                     |
| -------------------------                           |
| [Don't Create]                        [Approve]     |
+----------------------------------------------------+
```

- Width: 100% of chat message area (same width as bot message bubbles)
- Card background: `--color-card`
- Border: 1px `--color-border` (non-destructive) or 1px `--color-destructive` (destructive)
- Border radius: `--radius` (0.625rem)
- Internal padding: 16px all sides
- Icon: Lucide icon matching action type (see Action Icon Map below), 16px, `--color-muted-foreground`
- Badge: "destructive" badge uses `--color-destructive` background with `--color-destructive-foreground` text (only for destructive actions)
- Title: Heading 16px/600
- Body: Body 14px/400
- Separator: full-width, `--color-border`, 8px vertical margin above and below
- Button row: flex, justify-between
- Reject button: variant="outline", size="sm" — label varies by action type (see Copywriting Contract)
- Approve button (non-destructive): variant="default", size="sm"
- Approve button (destructive): variant="destructive", size="sm" (uses `--color-destructive` background)

### Action Icon Map

| Action | Lucide Icon |
|--------|-------------|
| create_task | `Plus` |
| update_task | `Pencil` |
| update_task_status | `CircleCheck` |
| delete_task | `Trash2` |
| update_phase_status | `CircleCheck` |
| create_project | `FolderPlus` |
| create_theme | `Palette` |

### Action Result Card (inline in chat)

```
+----------------------------------------------------+
| [check-icon] Task created: Fix login bug            |
+----------------------------------------------------+
```

- Width: 100% of chat message area
- Background: `--color-card`
- Border: 1px `--color-border`
- Border radius: `--radius`
- Internal padding: 8px 16px
- Single line layout: flex, align-center, gap 8px
- Success icon: Lucide `CheckCircle2`, 16px, `--color-primary` (accent)
- Error icon: Lucide `XCircle`, 16px, `--color-destructive`
- Text: Body 14px/400

### Shell Output Block (inline in chat)

```
+----------------------------------------------------+
| $ git status                              [expand]  |
|----------------------------------------------------|
| On branch main                                      |
| Your branch is up to date with 'origin/main'.      |
|                                                     |
| nothing to commit, working tree clean               |
+----------------------------------------------------+
```

- Width: 100% of chat message area
- Background: `--color-secondary` (darker code block feel)
- Border: 1px `--color-border`
- Border radius: `--radius`
- Header row: flex, justify-between, padding 8px 16px, `--color-card` background
  - Command text: Code 14px/400 monospace, `--color-primary` (accent)
  - Expand/collapse trigger: Lucide `ChevronDown`/`ChevronUp`, 16px, `--color-muted-foreground`
- Output area: padding 8px 16px, Code 14px/400 monospace, `--color-foreground`
- Auto-expand: output under 20 lines shows expanded by default
- Auto-collapse: output 20+ lines shows collapsed with summary text
- Truncation: output over 200 lines truncated with "... {remaining} more lines" at bottom
- Max height when expanded: 400px with ScrollArea

### Shell Allowlist Settings

```
+----------------------------------------------------+
| Shell Allowlist                                      |
| Commands the AI bot is allowed to run. Default safe  |
| commands are always included.                        |
|                                                     |
| Default commands (always available):                 |
| [git] [npm] [yarn] [pnpm] [ls] [cat] [head] [tail] |
| [wc] [echo] [date] [pwd]                           |
|                                                     |
| Custom commands:                                     |
| [docker  x] [cargo  x] [python  x]                 |
|                                                     |
| [___________________] [Add]                          |
+----------------------------------------------------+
```

- Section header: Heading 16px/600
- Description: Body 14px/400, `--color-muted-foreground`
- Default commands: read-only Badge list, variant="secondary", size="sm"
- Custom commands: Badge with "x" remove button, variant="outline", size="sm"
- Input + Add button: flex row, Input takes remaining width, Button variant="outline" size="sm"
- Vertical spacing: 16px between description and command groups, 16px between groups

## Interaction States

### Confirmation Card

| State | Visual |
|-------|--------|
| Pending | Card visible, buttons enabled, pulsing left-border accent (2px `--color-primary` for non-destructive, `--color-destructive` for destructive) |
| Approved | Card fades to 50% opacity over 300ms, buttons disabled, "Approved" label replaces buttons |
| Rejected | Card fades to 50% opacity over 300ms, buttons disabled, "Dismissed" label replaces buttons |
| Expired (timeout) | Not applicable for interactive chat -- blocks indefinitely |

### Shell Output Block

| State | Visual |
|-------|--------|
| Running | Header shows animated ellipsis "Running..." after command, no output area yet |
| Complete (short) | Expanded by default, static |
| Complete (long) | Collapsed by default, click header to expand |
| Error (non-zero exit) | Border changes to `--color-destructive`, error prefix in output |
| Timed out | Border changes to `--color-destructive`, truncation message appended |

### Action Result Card

| State | Visual |
|-------|--------|
| Success | Green check icon (`--color-primary`), body text |
| Error | Red X icon (`--color-destructive`), error message in body, `--color-destructive` left border (2px) |

### Allowlist Badge (Settings)

| State | Visual |
|-------|--------|
| Default (read-only) | `--color-secondary` background, `--color-foreground` text |
| Custom | `--color-border` outline, `--color-foreground` text, "x" button visible |
| Hover (custom) | "x" button changes to `--color-destructive` |

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Confirmation card focus | When card appears, focus moves to Approve button (most common action) |
| Confirmation keyboard | Enter on Approve button confirms, Escape rejects (triggers reject action) |
| Confirmation ARIA | `role="alertdialog"`, `aria-labelledby` pointing to title, `aria-describedby` pointing to body |
| Shell output block | `aria-expanded` on collapse trigger, `aria-label="Shell output for {command}"` |
| Shell output content | Code block has `role="log"`, `aria-live="polite"` while running |
| Action result | `role="status"`, `aria-live="polite"` for screen reader announcement |
| Destructive badge | `aria-label="This action is destructive"` on destructive badge |
| Settings allowlist | Each badge has `aria-label="Remove {command} from allowlist"` on remove button |
| Settings input | `aria-label="Add command to shell allowlist"` |
