**3. Create `src-tauri/src/engine/shell.rs`** -- async shell command executor:

Define `execute_shell(command: &str, working_dir: Option<&str>, timeout_ms: Option<u64>, stdin_input: Option<&str>) -> Result<Document, EngineError>`:
- Use `tokio::process::Command::new("sh").arg("-c").arg(command)`
- Set stdout(Stdio::piped()), stderr(Stdio::piped())
- If working_dir is Some, set current_dir
- If stdin_input is Some, pipe it to stdin (use tokio::io::AsyncWriteExt, then drop stdin to signal EOF)
- Wrap child.wait_with_output() in `tokio::time::timeout` with default 30_000ms
- On timeout, return EngineError::Timeout
- On non-zero exit code, return EngineError::StepFailed { exit_code, stdout, stderr }
- On success, return Document { content: stdout, content_type: "text/plain", metadata: { exit_code, status_code: None } }
- Cap output at 1MB -- if stdout exceeds 1MB, truncate and append "\n[output truncated at 1MB]"

Add unit tests:
- `test_shell_echo` -- execute "echo hello", verify content contains "hello"
- `test_shell_exit_code` -- execute "exit 1", verify EngineError::StepFailed
- `test_shell_stdin_pipe` -- execute "cat", stdin_input "piped data", verify output = "piped data"
- `test_shell_timeout` -- execute "sleep 10" with timeout_ms 100, verify EngineError::Timeout

**4. Create `src-tauri/src/engine/http.rs`** -- async HTTP request executor:

Define `execute_http(method: &str, url: &str, headers: &Option<Vec<(String, String)>>, body: &Option<serde_json::Value>, timeout_ms: Option<u64>) -> Result<Document, EngineError>`:
- Create reqwest::Client::new()
- Match method.to_uppercase(): GET, POST, PUT, DELETE, PATCH -> corresponding client method. Invalid method -> EngineError::InvalidMethod
- Apply headers if provided
- Apply JSON body if provided
- Wrap in tokio::time::timeout with default 30_000ms
- Capture status code and response body text
- Detect content_type by trying serde_json::from_str on body -- if valid JSON, "application/json", else "text/plain"
- Return Document { content: body_text, content_type, metadata: { status_code: Some(status), exit_code: None } }

Add unit tests (these test the function signature and error handling, not real HTTP):
- `test_invalid_method` -- call with method "INVALID", verify EngineError::InvalidMethod

Implement PipelineExecutor:
```rust
pub struct PipelineExecutor {
    step_outputs: HashMap<String, Document>,
}
```

Methods:
- `new() -> Self`
- `async execute(workflow: &Workflow, app: &tauri::AppHandle, db: &Arc<Mutex<Database>>, trigger_type: &str) -> Result<Vec<StepResult>, EngineError>`:
  1. Lock DB briefly to create_workflow_run, then unlock
  2. For each step in workflow.steps:
     a. Lock DB briefly to create_step_result with status "pending", then unlock
     b. Emit "workflow-step-started" event with StepProgress
     c. Lock DB briefly to update_step_result to "running", then unlock
     d. Resolve template variables in command/URL fields using resolve_templates()
     e. Dispatch to shell::execute_shell or http::execute_http based on StepDefinition variant
     f. For Manual steps: emit "workflow-step-manual" event and return early (manual steps pause execution)
     g. On success: store output in step_outputs HashMap, update step_result to "completed", emit "workflow-step-completed"
     h. On failure: update step_result to "failed", emit "workflow-step-failed", complete_workflow_run with "failed", return error
  3. After all steps succeed: complete_workflow_run with "completed"
  CRITICAL: Never hold the Mutex<Database> lock during step execution. Lock -> read/write -> unlock -> execute -> lock -> write result -> unlock.

- `fn resolve_templates(&self, input: &str) -> String`:
  Use `regex::Regex::new(r"\{\{(\w+)\.output\}\}")` to find `{{step_name.output}}` patterns.
  Replace each match with the content from self.step_outputs.get(step_name), or empty string if not found.

- `async retry_from_step(workflow: &Workflow, app: &tauri::AppHandle, db: &Arc<Mutex<Database>>, run_id: &str, from_step_index: usize) -> Result<(), EngineError>`:
  1. Load existing step results for run_id
  2. Populate step_outputs from completed steps (those before from_step_index)
  3. Resume execution from from_step_index using same logic as execute()

Add unit tests:
- `test_resolve_templates` -- set step_outputs with key "fetch", resolve "echo {{fetch.output}}", verify replacement
- `test_resolve_templates_missing` -- resolve "{{missing.output}}", verify replaced with empty string

```rust
use std::sync::Arc;
use crate::engine::executor::PipelineExecutor;
```

- `run_workflow(app: AppHandle, state: State<'_, std::sync::Mutex<Database>>, workflow_id: String) -> Result<String, String>`:
  1. Lock DB, read workflow via db.get_workflow(&workflow_id), unlock
  2. Clone the AppHandle
  3. Create Arc<Mutex<Database>> from state (use state.inner().clone() or equivalent pattern)
  4. Spawn a tokio task: `tokio::spawn(async move { ... })`
  5. Inside the spawned task: create PipelineExecutor::new(), call execute(&workflow, &app, &db_arc, "manual")
  6. Return the run_id immediately (don't wait for completion -- frontend listens to events)
  Note: The actual run_id should be created before spawning so it can be returned. Create the workflow_run in the command, then pass run_id to executor.

- `retry_workflow_step(app: AppHandle, state: State<'_, std::sync::Mutex<Database>>, workflow_id: String, run_id: String, step_index: usize) -> Result<(), String>`:
  1. Lock DB, read workflow, unlock
  2. Spawn tokio task calling PipelineExecutor::retry_from_step(&workflow, &app, &db_arc, &run_id, step_index)
  3. Return Ok(()) immediately

**3. Register the two new commands in `src-tauri/src/lib.rs` invoke_handler:** `run_workflow, retry_workflow_step`

**Important pattern for Arc<Mutex<Database>>:** The Tauri State wraps Mutex<Database>. To pass to a spawned task, you need to get a reference that lives long enough. Use:
```rust
let db_arc = Arc::new(std::sync::Mutex::new(
    // This won't work -- can't move out of State
));
```
Instead, refactor the app state to use `Arc<Mutex<Database>>` directly. In lib.rs setup:
```rust
let db = Database::new(app.handle())?;
app.manage(Arc::new(Mutex::new(db)));
```
And update ALL existing command signatures from `State<'_, std::sync::Mutex<Database>>` to `State<'_, Arc<std::sync::Mutex<Database>>>`. This is necessary for spawning async tasks that need DB access. Update the imports in task_commands.rs, project_commands.rs, workflow_commands.rs, and schedule_commands.rs.
