```typescript
interface WorkflowBuilderProps {
  workflow: Workflow;
  onSave: (steps: StepDefinition[]) => void;
}
```

Internal state:
- `steps: StepDefinition[]` -- initialized from workflow.steps
- `expandedIndex: number | null` -- which step is currently expanded (only one at a time)
- `hasChanges: boolean` -- track if steps differ from workflow.steps

Render:
- If steps.length === 0: EmptyState with heading "No steps yet", body "Add your first step to start building this workflow.", action button "Add Step" that creates a new Manual step.
- If steps.length > 0:
  - StepInsertButton at the top (inserts at index 0)
  - For each step: StepEditor + StepInsertButton below it
  - "Save Changes" button (disabled when !hasChanges) calls onSave(steps)

Insert step logic: clicking StepInsertButton at index N inserts a new step at position N:
```typescript
const newStep: ManualStepConfig = { type: "manual", name: `Step ${steps.length + 1}`, description: "" };
const updated = [...steps.slice(0, insertIndex), newStep, ...steps.slice(insertIndex)];
setSteps(updated);
setExpandedIndex(insertIndex); // Auto-expand new step
```

Move up/down: swap step at index with index-1 / index+1. Update expandedIndex to follow the moved step.

Delete: remove step at index. If deleted step was expanded, set expandedIndex to null.

onChange for a step: replace step at index with updated step in the steps array.

Per UI-SPEC keyboard shortcuts:
- `Cmd+N` (in workflow builder context): add new step at end
- `Escape`: collapse expanded step
- `Cmd+S`: save workflow changes
- `Cmd+Backspace`: delete selected step with confirmation
