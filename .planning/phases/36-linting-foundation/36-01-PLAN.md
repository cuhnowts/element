After auto-fixes, approximately 48 violations remain requiring manual judgment. Fix them in this priority order:

**Category 1: noNonNullAssertion (~21 violations)**
For each `!` non-null assertion:
- If the value can genuinely be null at runtime: add a proper null check (`if (x != null)` guard or optional chaining)
- If the value is guaranteed non-null by Tauri/runtime context (e.g., store state after initialization): add per-line suppression:
  `// biome-ignore lint/style/noNonNullAssertion: <specific reason>`

**Category 2: noArrayIndexKey (~11 violations)**
For each array `.map((item, index) => <Component key={index} ...>)`:
- If the item has a unique `id` field: use `key={item.id}`
- If the list is static/never reordered: suppress with `// biome-ignore lint/suspicious/noArrayIndexKey: static list, never reordered`

**Category 3: noExplicitAny (~4 violations)**
Replace `any` with proper types. Common patterns:
- Event handlers: use `React.SyntheticEvent` or specific event type
- API responses: define an interface or use `unknown` with type guard
- If truly untyped (e.g., third-party lib without types): suppress with reason

**Category 4: useExhaustiveDependencies (~4 remaining)**
For each remaining violation, analyze the dependency:
- If the dep IS stable (primitive, ref, memoized): add it to the array
- If adding the dep would cause infinite loops (unstable ref): suppress with `// biome-ignore lint/correctness/useExhaustiveDependencies: <dep> is intentionally excluded to prevent re-render loop`

**Category 5: Other (~8 violations)**
- useIterableCallbackReturn (2): ensure `.map()` callbacks return a value
- noInvalidUseBeforeDeclaration (2): reorder variable/function declarations
- noImplicitAnyLet (1): add explicit type annotation
- noGlobalIsNan (1): replace `isNaN(x)` with `Number.isNaN(x)`
- noAssignInExpressions (1): extract assignment to separate statement
- noUselessFragments (1): replace `<>x</>` with just `x`
- noNonNullAssertedOptionalChain (1): remove `!` after `?.`
- noUnusedVariables (1): prefix with `_` or remove

Per D-01: Do NOT disable any recommended rule globally. All suppressions must be per-line with justification.
