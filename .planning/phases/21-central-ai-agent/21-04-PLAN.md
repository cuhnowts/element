  <action>
Create `src/hooks/useAgentQueue.ts` -- a hook that polls the agent-queue directory for new files written by the MCP server and syncs them to the agent store.

**Why polling over fs events:** The MCP server sidecar is a child process of the agent CLI, not of Tauri. Using Tauri's file watcher would work but adds complexity. A simple 2-second poll interval is sufficient for the approval flow (human-in-the-loop latency dwarfs poll interval).

```typescript
import { useEffect, useRef } from "react";
import { useAgentStore } from "@/stores/useAgentStore";
import { appDataDir } from "@tauri-apps/api/path";
import { readDir, readTextFile, writeTextFile, mkdir, exists } from "@tauri-apps/plugin-fs";
```

Implementation:

**Queue directory resolution:**
- `getQueueDir()`: returns `<appDataDir>/agent-queue`
- On first call, ensure all subdirectories exist (`approvals/`, `notifications/`, `status/`, `sessions/`) via `mkdir({ recursive: true })`

**Poll loop (2-second interval):**
- `useEffect` sets up `setInterval(pollQueue, 2000)`. Cleanup clears interval.
- Track processed file IDs in a `Set<string>` ref to avoid re-processing.

**pollQueue():**
1. **Scan approvals/**: Read each `.json` file. For files not in processed set:
   - Parse JSON. If `status === "pending"`, call `addEntry({ type: "approval_request", title: "Approve: " + phaseName, description: projectName + " is ready to auto-execute " + phaseName + ". No human blockers detected.", projectId, projectName, phaseName, approvalStatus: "pending" })` with the approval file's `id` as a custom mapping (store the mapping of approvalId -> entryId in a ref Map).
   - Add to processed set.

2. **Scan notifications/**: Read each `.json` file. For files not in processed set:
   - Parse JSON. Call `addEntry({ type: mapPriorityToEntryType(priority), title, description: body, projectId, projectName })`
   - Where `mapPriorityToEntryType`: "critical" -> "human_needed", "informational" -> "execution_complete", "silent" -> "context_seeded"
   - Add to processed set.

3. **Scan status/**: Read each `.json` file. For files not in processed set:
   - Parse JSON. Call `addEntry({ type: mapStatusToEntryType(status, action), title: action, description: details ?? "", projectId })`
   - Where: "completed" -> "execution_complete", "failed" -> "error", "blocked" -> "human_needed"
   - Add to processed set.

**Write-back for approvals:**
Export `writeApprovalDecision(approvalId: string, decision: "approved" | "rejected")`:
- Read `<queueDir>/approvals/<approvalId>.json`
- Update `status` to decision, add `decidedAt: new Date().toISOString()`
- Write back to same file via `writeTextFile`
- This is called from `ApprovalRequest.tsx` when user clicks Approve/Reject

**Write session request:**
Export `writeSessionRequest(projectId: string, sessionName: string)`:
- Write `<queueDir>/sessions/<timestamp>-<random4>.json` containing `{ projectId, sessionName, createdAt }`
- This is called from the modified OpenAiButton

The hook returns `{ writeApprovalDecision, writeSessionRequest }`.
