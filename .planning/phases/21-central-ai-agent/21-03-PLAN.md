**src/hooks/useAgentMcp.ts:**

Hook that manages the MCP server sidecar process lifecycle.

```typescript
import { useCallback, useRef } from "react";
import { Command } from "@tauri-apps/plugin-shell";

interface McpSidecarState {
  start: (dbPath: string) => Promise<void>;
  stop: () => Promise<void>;
  isRunning: () => boolean;
}
```

Implementation:
- Store child process reference in a `useRef`
- `start(dbPath)`: Spawn MCP server via `Command.create("node", [mcpServerPath, dbPath])` where `mcpServerPath` is resolved relative to the app's resource directory. For dev mode, use `mcp-server/dist/index.js`. Log stdout/stderr events to console for debugging.
- `stop()`: Kill the child process if running. Set ref to null.
- `isRunning()`: Check if child ref is non-null.
- On component unmount, kill the sidecar process (cleanup in parent).

Note: The MCP server's stdio is NOT piped to the agent -- the agent connects to the MCP server via its own MCP config (which specifies the server command). The sidecar here is launched by Element to ensure it's running, but the agent CLI tool spawns its own connection to it via the generated MCP config file.

Actually, correction based on MCP architecture: The MCP server runs as a subprocess of the agent CLI tool, not as a separate sidecar managed by Element. The agent CLI tool (e.g., `claude`) is configured with `--mcp-config` that tells it how to spawn the MCP server. Element only needs to:
1. Generate the MCP config JSON file pointing to `node mcp-server/dist/index.js <dbPath>`
2. Generate the agent system prompt file
3. Launch the agent CLI tool with the config

Revise `useAgentMcp.ts` to be a config generator hook instead:
- `generateMcpConfig(dbPath: string): Promise<string>` -- writes MCP config JSON to a temp file in the app data directory (`<appDataDir>/agent/mcp-config.json`) and returns the file path. Config content:
  ```json
  {
    "mcpServers": {
      "element": {
        "type": "stdio",
        "command": "node",
        "args": ["<absolute-path-to-mcp-server/dist/index.js>", "<dbPath>"]
      }
    }
  }
  ```
- `generateSystemPrompt(): Promise<string>` -- writes the agent system prompt to `<appDataDir>/agent/AGENT.md` and returns the file path. Prompt content is the system prompt from research (Pattern: Agent System Prompt Generation) -- describes available tools, operating mode (approve-only), and rules.
- Uses `@tauri-apps/api/path` for `appDataDir()` resolution
- Uses `@tauri-apps/plugin-fs` or Tauri `invoke` to write files

**src/hooks/useAgentLifecycle.ts:**

Hook that manages the agent CLI process lifecycle -- auto-start, restart with backoff, crash handling.

```typescript
import { useCallback, useEffect, useRef } from "react";
import { useAgentStore } from "@/stores/useAgentStore";
import { api } from "@/lib/tauri";
// import config generator from useAgentMcp

const BACKOFF_MS = [2000, 4000, 8000]; // D-04: 2s, 4s, 8s
const MAX_RETRIES = 3;
```

Implementation:
- `startAgent()`:
  1. Set status to "starting" via store
  2. Read CLI settings: `api.getAppSetting("cli_command")` and `api.getAppSetting("cli_args")`
  3. If no CLI command configured, set status to "stopped", show toast "No AI tool configured. Set one in Settings > AI.", return
  4. Validate CLI tool: `api.validateCliTool(command)`. If invalid, set status to "stopped", show toast, return
  5. Get DB path via Tauri `appDataDir()` + `/element.db`
  6. Generate MCP config file via `generateMcpConfig(dbPath)` -- returns config file path
  7. Generate system prompt file via `generateSystemPrompt()` -- returns prompt file path
  8. Build agent launch args: start with user-configured args (sanitize em-dashes like OpenAiButton does), then add `--mcp-config <configPath>` and `@<promptPath>`
  9. Store the command and args for the agent terminal to use (via a ref or store field)
  10. Set status to "running"
  11. Reset restart count

- `handleAgentExit(exitCode: number)`:
  1. If exit code is 0, set status to "idle" (graceful exit)
  2. If restartCount >= MAX_RETRIES, set status to "stopped", show toast "Agent failed to start after 3 attempts. Click Restart to try again." (per UI-SPEC copywriting), return
  3. Set status to "error"
  4. Increment restart count
  5. Schedule restart with `setTimeout(startAgent, BACKOFF_MS[restartCount])` where restartCount is the value BEFORE increment

- `restartAgent()`: Reset restart count, call startAgent(). Used by manual "Restart Agent" button.

- Auto-start: `useEffect` on mount calls `startAgent()`. Cleanup kills agent process.

- Return: `{ startAgent, restartAgent, handleAgentExit, agentCommand, agentArgs }` where agentCommand/agentArgs are the resolved command for the terminal to spawn.

The hook stores the resolved command/args in refs so `AgentTerminalTab` can access them to spawn the PTY.
