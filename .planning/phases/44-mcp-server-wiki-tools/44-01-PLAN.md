    Create `mcp-server/src/plugin-loader.ts` exporting:

    ```typescript
    export interface PluginMcpTool {
      name: string;           // "pluginName:toolName" (namespace-prefixed)
      description: string;
      inputSchema: Record<string, unknown>;
      handlerModule: string;  // Absolute path to handler .js file
      handlerFunction: string; // Export name in that module
    }

    export function loadPluginTools(pluginsDir: string): PluginMcpTool[]
    ```

    Implementation per D-01 and D-03:
    1. Check if `pluginsDir` exists, return `[]` if not
    2. Read each subdirectory looking for `plugin.json`
    3. Parse manifest JSON, look for `mcp_tools` array field
    4. For each mcp_tool entry, split `handler` string on "#" to get file and function name
    5. Resolve handler module path as absolute: `resolve(pluginsDir, pluginDirName, fileName)`
    6. Prefix tool name with `${manifest.name}:` for namespace enforcement (prevents collision with hardcoded tools)
    7. Return array of `PluginMcpTool` objects

    Expected manifest mcp_tools schema (Claude's discretion area):
    ```json
    {
      "name": "knowledge",
      "mcp_tools": [
        {
          "name": "wiki_query",
          "description": "Query the knowledge wiki for articles matching a topic",
          "input_schema": { "type": "object", "properties": { "query": { "type": "string" } }, "required": ["query"] },
          "handler": "mcp-handlers.js#handleWikiQuery"
        }
      ]
    }
    ```

    Use `node:fs` readFileSync, readdirSync, existsSync. Use `node:path` join, resolve.

  <action>
    Create `mcp-server/src/tools/wiki-tools.ts` exporting two handler functions:

    ```typescript
    export function handleWikiQuery(
      dbPath: string,
      args: { query: string }
    ): { content: Array<{ type: "text"; text: string }>; isError?: boolean }

    export function handleWikiIngest(
      dbPath: string,
      args: { filePath: string }
    ): { content: Array<{ type: "text"; text: string }>; isError?: boolean }
    ```

    Note: These handlers do NOT take a `db` parameter because they only interact with the filesystem (wiki files + agent queue), not SQLite. This differs from most existing handlers. The `dbPath` is used only to derive the `.knowledge/` and `agent-queue/` directory locations via `join(dirname(dbPath), ".knowledge")` and `join(dirname(dbPath), "agent-queue")`.

    **handleWikiQuery implementation:**
    1. Derive knowledgeDir as `join(dirname(dbPath), ".knowledge")`
    2. If knowledgeDir does not exist: return `{ content: [{ type: "text", text: "Wiki not initialized. Enable the knowledge plugin first." }], isError: true }`
    3. Read `join(knowledgeDir, "index.md")` content
    4. Parse index.md lines to find entries matching `args.query` (case-insensitive substring match against each line)
    5. Extract file paths from matching index entries. Index format assumption: lines containing wiki page references like `- [Page Title](wiki/page-name.md)` -- extract the path from the markdown link
    6. If no matches: return `{ content: [{ type: "text", text: "No wiki articles matched your query" }] }` (no isError flag)
    7. For each match, read the article file from `join(knowledgeDir, relativePath)` and build `{ path: relativePath, content: fileContent }`
    8. Return `{ content: [{ type: "text", text: JSON.stringify(articles, null, 2) }] }`

    **handleWikiIngest implementation:**
    1. Derive knowledgeDir as `join(dirname(dbPath), ".knowledge")`
    2. If knowledgeDir does not exist: return error "Wiki not initialized. Enable the knowledge plugin first."
    3. Check `args.filePath` exists with `existsSync` -- if not: return `{ content: [{ type: "text", text: "Error: File not found: {filePath}" }], isError: true }`
    4. Check file is readable with `accessSync(args.filePath, constants.R_OK)` in try/catch -- if not: return error "Error: File not readable: {filePath}"
    5. Derive queueDir as `join(dirname(dbPath), "agent-queue")`, operationsDir as `join(queueDir, "operations")`
    6. Ensure operationsDir exists with `mkdirSync(operationsDir, { recursive: true })`
    7. Generate operation ID with same pattern as orchestration-tools: `wiki-ingest-${Date.now()}-${Math.random().toString(36).substring(2, 6)}`
    8. Write JSON payload to `join(operationsDir, `${operationId}.json`)`:
       ```json
       { "id": "wiki-ingest-...", "type": "wiki_ingest", "filePath": "/abs/path/to/source.md", "status": "accepted", "createdAt": "2026-..." }
       ```
    9. Return `{ content: [{ type: "text", text: JSON.stringify({ operationId, status: "accepted" }) }] }`

    - Test all behavior cases listed above
    - Clean up temp dirs in afterEach
