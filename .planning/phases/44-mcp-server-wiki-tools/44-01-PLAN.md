<objective>
Replace filesystem-based plugin-loader with DB-based plugin-tools that reads the `plugin_mcp_tools` SQLite table for tool discovery and uses dual dispatch (built-in handler map for core plugins, dynamic import for user plugins). Wire into index.ts so wiki tools appear in MCP ListTools/CallTools when the knowledge plugin is enabled.

Purpose: External agents (Claude Code) need wiki_query and wiki_ingest MCP tools. The old plugin-loader.ts read plugin.json files from disk, but core plugins (like knowledge) have no plugin.json -- their tools are in the DB table populated by Rust PluginHost.

Output: Working MCP server that dynamically loads plugin tools from DB and dispatches to correct handlers.
</objective>

    1. **PluginToolDef interface:**

       export interface PluginToolDef {
         prefixedName: string;    // "core-knowledge:wiki_query"
         pluginName: string;      // "core-knowledge"
         toolName: string;        // "wiki_query"
         description: string;
         inputSchema: Record<string, unknown>;
       }

    2. **loadPluginToolsFromDb(db: DatabaseType): PluginToolDef[]**
       - Run SQL: `SELECT prefixed_name, plugin_name, description, input_schema FROM plugin_mcp_tools WHERE enabled = 1`
       - Map rows to PluginToolDef objects
       - Derive `toolName` from `prefixedName` by splitting on first ":" and taking everything after it: `row.prefixed_name.substring(row.prefixed_name.indexOf(":") + 1)`
       - Parse `input_schema` from JSON string to object via `JSON.parse(row.input_schema)`
       - Return empty array if no rows

    3. **McpResult type** (same as wiki-tools.ts):
       ```typescript
       type McpResult = {
         content: Array<{ type: "text"; text: string }>;
         isError?: boolean;
       };
       ```

    4. **CORE_HANDLERS map:**
       ```typescript
       import { handleWikiQuery, handleWikiIngest } from "./tools/wiki-tools.js";

       type PluginHandler = (dbPath: string, args: Record<string, unknown>) => McpResult;

       const CORE_HANDLERS: Record<string, PluginHandler> = {
         "core-knowledge:wiki_query": (dbPath, args) =>
           handleWikiQuery(dbPath, args as { query: string }),
         "core-knowledge:wiki_ingest": (dbPath, args) =>
           handleWikiIngest(dbPath, args as { filePath: string }),
       };
       ```

    5. **dispatchPluginTool(prefixedName, dbPath, pluginsDir, args): Promise<McpResult>**
       - Check CORE_HANDLERS[prefixedName] first -- if found, call it and return

       - If no handler found anywhere, throw Error(`No handler for tool "${prefixedName}"`)
       - Per D-03 (adapted for rework): core plugin handlers in mcp-server/src/tools/, user plugin handlers via dynamic import from plugin directory
