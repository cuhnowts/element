## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, color dot margin, inline padding |
| sm | 8px | Compact element spacing, sidebar item padding-x, accordion child indent |
| md | 16px | Default element spacing, sidebar section padding-x |
| lg | 24px | Section padding, dialog body padding |
| xl | 32px | Layout gaps, dialog width gutters |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Not used in this phase |

Exceptions: Drag handle hit area uses 24px width for comfortable grab targets. Color palette dot buttons use 20px diameter (closest multiple of 4 that fits 10 dots in a row within dialog).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 500 (medium) | 1.33 |
| Heading | 16px | 600 (semibold) | 1.25 |

Notes:
- Body (14px/400): Sidebar item text (project names, task titles, theme names in expanded view).
- Label (12px/500): Section headers ("PROJECTS", "UNCATEGORIZED"), uppercase tracking-wide, uses `text-muted-foreground`. Matches existing `ProjectList` header pattern.
- Heading (16px/600): Dialog titles ("Create Theme", "Edit Theme"), confirmation dialog headings.

Source: Body size and line-height from existing `app.css` (line 42-43). Label pattern from existing `ProjectList.tsx` (line 34).

## Interaction Contract

### Sidebar Layout (top to bottom)

1. **CalendarToggle** + MiniCalendar (conditional) -- unchanged (D-06)
2. **Border divider** -- `border-b border-border`
3. **Theme sections header row**: label "THEMES" (12px/500 uppercase tracking-wide text-muted-foreground) + "+" button (size-6 ghost)
4. **Theme accordion sections** -- one per theme, sorted by `sort_order`, draggable for reorder (D-08)
5. **Uncategorized section** -- always last, non-draggable, contains items with null theme_id (D-16)
6. **Border divider** -- `border-b border-border`
7. **WorkflowList** -- unchanged (D-07)
8. **Settings button** -- unchanged

### Theme Section (accordion)

- **Collapsed state**: Color dot (8px) + theme name (14px/400) + chevron-right (size-4) + hover actions
- **Expanded state**: Color dot (8px) + theme name (14px/400) + chevron-down (size-4) + hover actions, then indented children:
  - Projects listed first, each with expand arrow (chevron-right/down, size-3.5) for inline task reveal (D-10)
  - Standalone tasks listed after projects, each with circle icon (size-3.5) (D-12)
- **Children indent**: `pl-2` (8px) from parent

### Theme Header Interactions

| Trigger | Action | Component |
|---------|--------|-----------|
| Click chevron or theme name | Toggle collapse/expand | ThemeHeader |
| Hover on theme header | Show edit (Pencil, size-3.5) and delete (Trash2, size-3.5) icons | ThemeHeader |
| Click edit icon | Open CreateThemeDialog in edit mode (pre-filled name + color) | CreateThemeDialog |
| Click delete icon | If theme has items: open confirmation dialog. If empty: delete immediately. | Dialog (confirmation) |
| Right-click theme header | Full context menu: Rename, Change Color, Delete, Move Up, Move Down | DropdownMenu |
| Drag theme header | Reorder among themes (DnD via @dnd-kit/sortable) | ThemeSidebar |

### Project Item Interactions (within theme)

| Trigger | Action |
|---------|--------|
| Click project name | Navigate to project detail view in center panel (D-10) |
| Click expand arrow | Toggle inline task list below project in sidebar (D-10) |
| Right-click project | Context menu: Move to theme (submenu), Delete Project |

### Standalone Task Interactions (within theme)

| Trigger | Action |
|---------|--------|
| Click task | Select task, show in output panel |
| Right-click task | Context menu: Move to theme (submenu), Delete Task |

### Theme Creation Flow

1. User clicks "+" button next to "THEMES" header
2. CreateThemeDialog opens with: name input (autofocused), color palette grid (10 dots, default: indigo #6366f1)
3. User types name, optionally picks color
4. User clicks "Create Theme" button or presses Enter
5. Dialog closes, new theme appears in sidebar at bottom of theme list (max sort_order + 1)
6. Theme section is expanded by default

### Theme Deletion Flow (D-04)

1. User clicks delete icon or selects "Delete" from context menu
2. **If theme has items**: Confirmation dialog appears with message showing item count
3. **If theme is empty**: Delete immediately, no confirmation
4. On confirm: theme deleted, items become uncategorized (theme_id set to NULL, appear in Uncategorized bucket)

### Move to Theme Flow (D-09)

1. User right-clicks project or task
2. Context menu shows "Move to theme" with submenu
3. Submenu lists all themes (color dot + name) plus "Uncategorized" option
4. Current theme shown with checkmark
5. User selects target theme
6. Item moves immediately (optimistic update)

## States Inventory

### Theme Section States

| State | Visual Treatment |
|-------|-----------------|
| Collapsed | Chevron-right, children hidden |
| Expanded | Chevron-down, children visible with pl-2 indent |
| Dragging | Slight opacity reduction (opacity-50), drag overlay shows theme header |
| Drop target | 2px accent border-top indicator between themes |
| Empty (no items in theme) | Section renders header only when expanded, no empty message |

### Theme Header States

| State | Visual Treatment |
|-------|-----------------|
| Default | Color dot + name + chevron, action icons hidden |
| Hover | Action icons (edit, delete) fade in at right side |
| Selected/Active | No special selected state for themes (themes are containers, not selectable) |

### Sidebar Item States (projects, tasks within theme)

| State | Visual Treatment |
|-------|-----------------|
| Default | Normal text color, transparent background |
| Hover | `hover:bg-muted` background |
| Selected | `text-primary font-medium` (matches existing ProjectList pattern) |
| Context menu open | `bg-muted` background persists while menu is open |

### Dialog States

| State | Visual Treatment |
|-------|-----------------|
| Create mode | Title "Create Theme", empty name input, indigo pre-selected, CTA "Create Theme" |
| Edit mode | Title "Edit Theme", pre-filled name + current color selected, CTA "Save Changes" |
| Validation error | Red text below input (text-destructive-foreground, 12px) |
| Submitting | CTA button shows loading state (disabled + spinner or "Creating...") |
