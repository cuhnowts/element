## Phase Classification

**This phase is backend-dominant infrastructure.** The CONTEXT.md explicitly states: "This is backend infrastructure -- no new UI views." No new pages, panels, or visual components are introduced.

Frontend changes are limited to:
- TypeScript type extensions for new manifest fields (skills, mcp_tools, owned_directories, lifecycle hooks, manifest_version)
- New Tauri API bindings (dispatch_plugin_skill invoke)
- Zustand pluginSlice additions for skill and MCP tool registries
- Reuse of existing Dialog component for destructive skill confirmation (pattern already established)

The full UI integration of plugin skills into hub chat occurs in Phase 43.

## Interaction Contract

### Destructive Plugin Skill Confirmation

- **Trigger:** Hub chat dispatches a plugin skill where manifest declares `destructive: true`
- **Pattern:** Modal Dialog with destructive action description
- **Confirm button:** destructive variant, label matches the skill action verb
- **Cancel button:** secondary variant, label "Cancel"
- **Behavior:** Dialog blocks dispatch until user confirms or cancels
- **Note:** This pattern already exists for built-in destructive actions. Phase 41 extends it to plugin skills.

### Plugin Enable/Disable Feedback

- **Enable success:** Toast: "Plugin '{name}' enabled. {N} skills registered."
- **Enable failure (collision):** Toast (error): namespace collision copy from above
- **Disable success:** Toast: "Plugin '{name}' disabled. Skills and tools removed."
- **No restart required:** D-05 from CONTEXT.md -- disabling removes skills/tools from runtime registries without app restart.
