## Component Inventory

Phase 15 has a minimal UI surface. The work is primarily backend (Rust watcher, parser, sync logic). The frontend changes are:

### 1. Sync Badge on PhaseRow

A small inline indicator distinguishing synced phases from user-created ones.

| Property | Value |
|----------|-------|
| Component | `Badge` (existing shadcn component) |
| Variant | `outline` |
| Position | Inline after phase name, 8px gap |
| Icon | `RefreshCw` from lucide, 12px (size-3) |
| Text | "GSD" |
| Font | 10px, weight 600 (inherits from Badge `text-xs font-semibold`) |
| Visibility | Only shown when `phase.source === "gsd-sync"` (source field added in Phase 12) |
| Interactive | Not interactive — display only |

```
[Phase Name] [🔄 GSD]
```

### 2. Sync Badge on TaskRow

Same pattern as PhaseRow badge for consistency.

| Property | Value |
|----------|-------|
| Component | `Badge` (existing shadcn component) |
| Variant | `outline` |
| Position | Inline after task title, 8px gap |
| Icon | None (too small, task rows are compact) |
| Text | "GSD" |
| Font | Inherits Badge defaults (text-xs font-semibold) |
| Visibility | Only shown when `task.source === "gsd-sync"` |

### 3. Sync Toast Notification

Feedback when ROADMAP.md sync completes or fails.

| Property | Value |
|----------|-------|
| Component | `sonner` toast (existing, via `toast()` from sonner) |
| Trigger | Backend emits `planning-sync-complete` or `planning-sync-error` Tauri event |
| Position | Bottom-right (existing Toaster position) |
| Duration | 4000ms (default) |

**Success toast:**
```tsx
toast.success("Synced N phases, M tasks from .planning/")
```

**Error toast:**
```tsx
toast.warning("Could not parse .planning/ROADMAP.md — check file format")
```

**No-change toast:** Do not show a toast when content hash matches (silent skip).

### 4. Phase List — No Structural Changes

Synced phases appear in the existing `ProjectDetail` phase list using the same `PhaseRow` component. They are sorted by `sortOrder` alongside user-created phases. The only visual distinction is the sync badge.

Synced phases:
- ARE draggable (user may want to reorder)
- ARE collapsible (same interaction as user-created phases)
- ARE NOT editable inline (rename disabled when `source === "gsd-sync"`)
- ARE NOT deletable via context menu (delete option hidden when `source === "gsd-sync"`)

### 5. Auto-Import — No Explicit Button

Per D-11: When a GSD-tier project is selected and `.planning/ROADMAP.md` exists, auto-import if no synced phases exist yet. No import button is rendered in the UI.

## Interaction Contracts

### Sync Lifecycle (User Perspective)

1. User opens a GSD-tier project with a linked directory
2. Backend detects `.planning/ROADMAP.md`, auto-imports if no synced phases exist
3. Toast appears: "Synced N phases, M tasks from .planning/"
4. Phases appear in the phase list with "GSD" badge
5. User continues working; AI updates ROADMAP.md via terminal
6. File watcher detects change, re-syncs (replace strategy per D-10)
7. Phase list updates reactively via existing Tauri event listeners (`phase-created`, `phase-deleted`)
8. Toast appears with updated counts

### Synced Phase Restrictions

| Action | User-Created Phase | Synced Phase |
|--------|-------------------|--------------|
| Rename | Allowed (inline edit) | Disabled — context menu rename hidden |
| Delete | Allowed (context menu) | Disabled — context menu delete hidden |
| Reorder | Allowed (drag) | Allowed (drag) |
| Collapse/expand | Allowed | Allowed |
| Add task manually | Allowed | Disabled — add task button hidden |

### Synced Task Restrictions

| Action | User-Created Task | Synced Task |
|--------|------------------|-------------|
| Toggle status | Allowed (checkbox) | Disabled — checkbox is display-only |
| Click to select | Allowed | Allowed (read-only detail view) |
| Move to phase | Allowed (drag or menu) | Disabled |
| Edit title | Allowed (in detail panel) | Disabled |

### Error Recovery

Parse errors are non-destructive (D-09). Existing DB state is preserved. User sees a warning toast and can check the ROADMAP.md file format manually.

## States Matrix

| State | What User Sees |
|-------|---------------|
| No linked directory | Existing "Link a directory" UI — no sync behavior |
| Linked dir, no .planning/ | No change — standard project view |
| Linked dir, .planning/ exists, non-GSD tier | No sync — tier must be "gsd" |
| GSD tier, first open | Auto-import fires, toast confirms, phases appear |
| GSD tier, ROADMAP.md unchanged | Watcher fires, hash matches, silent skip (no toast) |
| GSD tier, ROADMAP.md updated | Watcher fires, re-sync, toast with new counts |
| GSD tier, ROADMAP.md malformed | Warning toast, existing phases preserved |
| GSD tier, .planning/ deleted | Watcher detects removal, synced phases remain in DB (no auto-delete) |
